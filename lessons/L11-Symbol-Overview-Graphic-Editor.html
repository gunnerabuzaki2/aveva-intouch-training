Lesson 11: Symbol Overview + Graphic Editor

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 3 (pages 99–112)

This lesson introduces symbols and the Graphic Editor — the two foundational tools for building process visualizations in System Platform.

11.1 Symbol Overview

Reference: Module 3, Section 1 (pages 101–104)

Symbols are the building blocks of process visualization in AVEVA System Platform. A symbol is a graphic element that can be embedded in or linked to automation objects, providing an efficient way to visualize object-specific information such as processing status, alarms, and history. System Platform ships with two prebuilt symbol libraries — the Industrial Graphic Library and the Situational Awareness Library — and you can also create your own symbols using the Graphic Editor.

Key Concept: Symbols can be created in three places depending on your needs: on the Graphics tab for reusable, standard symbols; in an object template for symbols shared by derived objects; or in an object instance for a symbol used only in that specific instance.

Graphics Tab

The Graphics tab in the System Platform IDE hosts libraries of symbols, controls, and other graphic-related content available to your Galaxy. The items shown depend on the Galaxy template used to create the Galaxy. Items on the Graphics tab are organized into a hierarchy of toolsets. Some toolsets are included by default, and you can create your own toolsets and sub-toolsets to organize content as needed.

To create a toolset, right-click the Galaxy name or an existing toolset and select New → Graphic Toolset. To create a symbol, right-click the Galaxy name or a toolset and select New → Symbol. The symbol is created with a default name and can be renamed, then double-clicked to open it for editing in the Graphic Editor.

Symbol Libraries

Two symbol libraries ship with System Platform for all Galaxy types:

  • Industrial Graphic Library — Contains pre-built, realistic-looking symbols of standard industrial objects such as valves, motors, pumps, and tanks.
  • Situational Awareness Library — Contains pre-built symbols with a simplified look and minimum visual detail, designed to enhance an operator's situational awareness of current process conditions.

11.2 Situational Awareness Visualization

Reference: Module 3, Section 1 (pages 102–103)

Situational awareness visualization provides operators with contextual information so they can figure out what actions to take. In broad terms, it operates on three cognitive levels: perception of new information, comprehension of the information's meaning and importance, and projection of the impact on goals. Achieving proper situational awareness helps operators make effective decisions and deliver overall business success.

Key Concept: In an HMI/SCADA application, situational awareness is a visual design philosophy that presents contextualized, easily interpreted information while filtering out visual noise. This enables operators to move from perception to comprehension to projection for safer operating conditions.

The situational awareness approach also encourages standardization, which helps eliminate the confusion caused by inconsistent use of color and other visual elements. Benefits include faster problem identification, maximized uptime, accelerated alarm recognition, increased operational consistency, and safer conditions.

The Situational Awareness cognitive model — perception, comprehension, and projection form the foundation of effective operator decision-making (page 102)

Situational Awareness Library Symbols

The Situational Awareness Library contains symbols created with situational awareness principles in mind. These symbols have a simplified look and provide a minimum of visual detail to efficiently convey their functional purpose and status without showing extraneous information that could distract operators.

By default, SA Library symbols use a color palette that increases contrast between normal and abnormal operating conditions. Normal conditions are shown in largely monochromatic, low-saturation colors, while abnormal conditions appear in vivid, high-contrast colors. Most SA Library symbols are designed as symbol wizards with multiple visual and functional configurations. For example, the SA_Meters symbol can function as a flow meter, temperature meter, level meter, or other meter type simply by selecting different Wizard Options.

SA Library symbols also come with an extensive set of default custom properties that control show/hide behavior, alarm conditions, and displayed values. They incorporate animations particularly suited for situational awareness, including Point, Alarm Border, Polar Star, and Sweep Angle animations.

Element Styles

An element style defines a set of visual properties that determine the appearance of text, lines, graphic outlines, and interior fill in graphics. When an Element Style is applied, the preconfigured values take precedence over a graphic's native visual properties. Element Styles are covered in more detail in the Galaxy Styles lesson.

11.3 The Graphic Editor Interface

Reference: Module 3, Section 2 (pages 105–107)

The Graphic Editor is the tool you use to create, configure, and modify symbols. It provides a drawing area called the canvas, on which you build graphics by embedding basic elements (lines, rectangles, curves) or by embedding symbols from the Toolbox, Assets, or Templates tabs. You can modify the appearance of elements through properties, toolbars, and menus, and configure them with animations, scripts, and custom properties.

The Graphic Editor interface is organized into five main areas:

Key Concept — Graphic Editor Layout:
(1) Toolbars and Menus — Buttons and menu commands for saving, zooming, line color, and other functions.
(2) Drawing Canvas — The central area where elements are placed and arranged to create a symbol.
(3) Elements List — A hierarchical list of all elements on the canvas.
(4) Element Tools — Tools for drawing basic shapes like lines, rectangles, ellipses, and text.
(5) Tabbed Area — Properties, Toolbox, Assets, and Templates tabs for configuring elements and accessing symbol libraries.
The Graphic Editor interface with all five labeled areas visible (page 105)

Canvas Properties

When the canvas is selected in the Graphic Editor, the Properties tab displays properties for the entire graphic, organized into four categories: Graphic (description and sizing), Appearance (anchor and smoothing), Runtime Behavior (content type, faceplate mode, scripts, zoom percent), and Custom Properties (user-defined properties).

By default, the canvas Size property is set to Auto, which automatically sizes the graphic by cropping all white space outside runtime elements. When set to Fixed, the FixedWidth and FixedHeight properties determine the absolute runtime size.

Elements List

The Elements list shows a hierarchical view of all graphic elements and embedded symbols by name. New items appear at the top by default, and the z-order can be manually adjusted. The Elements list is particularly useful for selecting elements that are visually hidden behind other elements on the canvas.

The Elements list showing the hierarchical structure of elements on a canvas (page 106)

Z-Order and Grouping

The z-order of items in the Elements list determines which element appears on top when elements overlap on the canvas. It also determines how elements of a path graphic connect. To change the z-order, right-click an element, select Order, and choose the desired position (Send To Back, Send Backward, Bring To Front, or Bring Forward).

You can also group elements to manage them as a single unit. Select the elements, right-click, and use the Grouping commands. Groups are assigned default names but can be renamed, and they can have properties different from their individual elements.

The right-click context menu with the Order submenu for managing z-order (page 107)
Warning: Renaming an element, group, or embedded symbol does not automatically update references in animations or scripts. If you rename an element that is referenced in an animation or script, you must manually update all references to the old name.

11.4 Element Properties and Selection

Reference: Module 3, Section 2 (pages 108–109)

You can control the appearance of elements using toolbar functions or the Properties tab. Read-only properties appear in gray, and non-default (modified) values appear in bold. Properties are organized into five categories: Graphic (name, identifiers), Wizard Options (visual/functional customization), Appearance (location, size, orientation, transparency), Runtime Behavior (visibility, tab order), and Custom Properties (user-defined).

Elements can be selected by clicking on them with the mouse, selecting from the Elements list, or dragging a lasso around them. When multiple elements are selected, the last-selected element is the primary element — it serves as the reference point for alignment and spacing operations. The other selected elements are secondary elements.

11.5 Embedding Symbols

Reference: Module 3, Section 2 (pages 109–110)

Embedding a symbol creates a linked copy of the original. If the original symbol is modified, changes propagate to all embedded locations. Symbols can be embedded from the Industrial Graphic Library, the Situational Awareness Library, and from templates and instances in the Galaxy.

There are two main methods for embedding symbols:

  1. Galaxy Browser — Click the Embed Industrial Graphic button from the toolbar, select it from the Edit menu, or right-click the canvas and choose Embed Industrial Graphic from the context menu.
  2. Drag-and-Drop — Find the symbol in the Toolbox, Assets, or Templates tab, then drag it from the preview area onto the canvas. For example, navigate to the Situational Awareness Library from the Toolbox tab, select the Meters toolset, and drag a symbol to the canvas.
A selected element on the canvas showing the selection handles used for resizing and rotation (page 109)
Multiple elements selected together, showing how primary and secondary elements are distinguished (page 109)

Symbol Search

The Toolbox, Assets, and Templates tabs include a search feature that lets you find symbols by name. For example, entering meters in the search box on the Toolbox tab will show all items with names containing that string.

Searching for "meters" in the Toolbox tab reveals matching symbols available for embedding (page 110)

11.6 Wizard Options, Custom Properties, and String Substitution

Reference: Module 3, Section 2 (pages 110–112)

Wizard Options

When a symbol with configured Wizard Options is selected on the canvas, its Wizard Options appear on the Properties tab. Many Situational Awareness Library symbols come with pre-configured Wizard Options that provide a simplified configuration menu for common settings — for example, you can configure a meter type without needing to modify low-level parameters.

Custom Properties

You can configure custom properties via the Edit Custom Properties dialog box, accessed by right-clicking an embedded symbol and selecting Custom Properties. Each property has a data type (such as Float), a default value, a visibility setting (public or private), and a description. The default value can be overridden with a value, reference, or expression.

The Edit Custom Properties dialog showing 33 properties with their data types and tag references (page 111)

Substitute Strings

You can search for and replace text strings in any element on the canvas that has a Text property. To do this, select the embedded symbol and choose Special → Substitute Strings from the menu, or right-click and select Substitute → Substitute Strings. The Substitute Strings dialog lets you replace text directly in the list or use Find & Replace functions like Match Case and Use Wildcards.

The Substitute Strings dialog — replacing "Level" with "Temperature" across all text elements (page 112)

Substitute References

Similarly, you can search for and replace data references used by elements. Select the embedded symbol and choose Special → Substitute References from the menu. This opens the Substitute Reference dialog, where you can replace reference paths directly or use Find & Replace. Note that while the Galaxy Browser can be used to select new references, auto-complete is not available in this dialog.

The Substitute Reference dialog — replacing Me.PV references with Me.Speed.PV paths (page 112)

Knowledge Check

1. What are the two symbol libraries that ship with System Platform, and how do they differ in their design philosophy?

Click to reveal answer

The Industrial Graphic Library provides realistic-looking symbols of standard industrial objects, while the Situational Awareness Library provides simplified, minimal-detail symbols designed to enhance operator situational awareness through color-coded normal/abnormal states.

2. In the Graphic Editor's Elements list, what happens if you rename an element that is referenced in an animation or script?

Click to reveal answer

Renaming does not automatically update references. Animations or scripts referencing the old name will break and must be manually updated to reference the new name.

3. What is the difference between the primary element and secondary elements when multiple items are selected on the canvas?

Click to reveal answer

The primary element (last selected) is the active element that serves as the reference point for alignment and spacing operations. Secondary elements are inactive and follow edits made to the primary element.

4. How does the Substitute Strings feature differ from Substitute References?

Click to reveal answer

Substitute Strings replaces visible text labels in elements (like "Level" → "Temperature"), while Substitute References replaces data binding paths (like Me.PV → Me.Speed.PV) that link graphics to runtime data.

5. What does the canvas Size property set to "Auto" do at runtime compared to "Fixed"?

Click to reveal answer

Auto automatically crops all white space outside runtime graphics, while Fixed uses FixedWidth and FixedHeight to define the exact pixel dimensions of the graphic at runtime.
