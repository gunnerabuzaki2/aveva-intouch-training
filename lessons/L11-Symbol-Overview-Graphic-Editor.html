Lesson 11: Symbol Overview and the Graphic Editor

Module 3 — Symbols | Pages 99–112

Collaborative workspace — a reminder that symbol design is both a technical and creative discipline.

This lesson marks the beginning of Module 3, where the focus shifts from deployment and runtime configuration to the visual heart of any HMI: symbols. A symbol is a reusable graphic object that can represent anything from a simple valve to a complex multi-attribute meter. Understanding how symbols are structured, created, and managed is essential for building effective operator displays. This lesson provides a comprehensive overview of symbols, situational awareness principles, the Graphic Editor interface, element management, embedding techniques, custom properties, and string/reference substitution.

What Are Symbols?

Symbols are the building blocks of InTouch visualization. At their simplest, a symbol is a graphic that can be embedded in or linked to an automation object, providing a visual representation of that object's data — its processing status, alarm state, historical trends, and more. System Platform ships with prebuilt symbol libraries, including the Industrial Graphic Library and the Situational Awareness Library, but you can also create your own symbols from scratch using the Graphic Editor.

The decision of where to create a symbol determines its scope and reusability:

  • On the Graphics tab — creates a reusable, standard symbol available throughout the Galaxy. This is the preferred approach for symbols that will be used by multiple automation objects.
  • In an object template — creates a symbol that is available to all objects derived from that template. Use this when a symbol is specific to a particular object type.
  • In an object instance — creates a symbol that exists only within that single instance. Use this for one-off graphics that will not be reused.
Key Concept — Symbol Scope

The Graphics tab is your Galaxy's symbol library. Symbols created there are global resources, accessible from any automation object's Graphic tab. When you create a symbol on the Graphics tab, you are building a reusable component — think of it like a library function in programming. Symbols created inside object templates or instances are more like local variables: useful, but limited in scope.

The Graphics Tab

The Graphics tab in the System Platform IDE serves as the central repository for all graphic-related content in your Galaxy. By default, the items available on this tab depend on which Galaxy template was used when the Galaxy was created. The tab supports a hierarchical organization of toolsets — folders that group symbols, controls, and other graphic assets.

Some toolsets are included by default (such as those containing the Industrial Graphic Library), but you can create your own to organize project-specific symbols. To create a new toolset, right-click the Galaxy name or an existing toolset and select New > Graphic Toolset. To add a symbol, right-click the toolset and select New > Symbol — the symbol is created with a default name and can be renamed and edited by double-clicking it.

Tip

Organize your Graphics tab toolsets to mirror your plant structure or functional areas (e.g., "Mixer Graphics," "Navigation Elements," "Alarm Displays"). A well-organized Graphics tab saves significant time during development, especially when multiple engineers are working on the same Galaxy.

Situational Awareness Visualization

Before diving into the Graphic Editor, it is worth understanding the design philosophy behind effective HMI graphics: situational awareness. Situational awareness visualization provides operators with contextual information so they can determine what actions to take. It operates on three cognitive levels:

  1. Perception — the operator notices new information (a value changing, an alarm appearing).
  2. Comprehension — the operator understands what the information means in context (is this normal? is this trending up?).
  3. Projection — the operator anticipates the impact on goals (will this exceed a limit? should I intervene now?).
The three-level cognitive model of situational awareness: Perception → Comprehension → Projection.

In an HMI/SCADA application, situational awareness means presenting operators with the most relevant information at the right time while filtering out visual noise. Operators see the current situation in full context — what is happening now, how it relates to recent events, and where the process might be headed. This approach encourages standardization in color, animation, and layout, eliminating the confusion that comes from inconsistent visual design.

The benefits of applying situational awareness principles are measurable: faster problem identification, maximized uptime, accelerated alarm recognition, increased operational consistency, and safer operating conditions.

Situational Awareness Library Symbols

The Situational Awareness Library provided with System Platform contains symbols designed according to these principles. They have a simplified, clean appearance and provide minimal visual detail to convey information efficiently. Most are implemented as symbol wizards — symbols that incorporate multiple visual and functional configurations. For example, the SA_Meters symbol can be configured as a flow meter, temperature meter, level meter, or other meter types simply by selecting options from its Wizard Options panel.

These symbols come with an extensive set of default custom properties that control visibility, alarm conditions, and displayed values. They also incorporate animations particularly suited for situational awareness, including Point, Alarm Border, Polar Star, and Sweep Angle animations.

Element Styles

An element style defines a set of visual properties — text appearance, line color, outline style, and interior fill — that can be applied to graphics. When an element style is applied to a graphic, its preconfigured values take precedence over the graphic's native visual properties. Element styles are closely related to Galaxy Styles and are covered in detail in the Galaxy Styles lesson (Lesson 22).

Key Concept — Element Styles vs. Galaxy Styles

Element styles control the visual appearance of individual graphic elements (lines, fills, text). Galaxy styles provide a broader layer of theming that can be applied across multiple symbols. Together, they ensure visual consistency across your entire HMI without requiring manual property changes on every element.

The Graphic Editor Interface

The Graphic Editor is the primary tool for creating, configuring, and modifying symbols. It provides a canvas-based drawing environment where you build graphics from basic elements, embedded symbols, and controls. The editor interface consists of five main areas:

The Graphic Editor interface with its five main components labeled.
  1. Toolbars and Menus — buttons and menu commands for saving, zooming, selecting colors, and other common operations.
  2. Drawing Canvas — the workspace where elements are placed, arranged, and configured to create your symbol.
  3. Elements List — a hierarchical tree showing every element and embedded symbol on the canvas, by name.
  4. Element Tools — a palette of basic graphic elements (lines, rectangles, ellipses, text, buttons, and more) that you drag onto the canvas.
  5. Tabbed Area — contains the Properties tab (element and symbol properties), Assets tab (graphics from instances), Toolbox tab (graphics from symbol libraries), and Templates tab (graphics from templates).

The Canvas

The canvas is where the creative work happens. When the canvas itself is selected (with no element selected), the Properties tab shows properties for the entire graphic, organized into categories: Graphic (description, sizing), Appearance (anchor, smoothing), Runtime Behavior (content type, faceplate mode, scripts, zoom), and Custom Properties.

By default, the canvas size is set to Auto, which automatically crops all white space outside the graphics that appear at runtime. If you need a fixed size, set the Size property to Fixed and specify FixedWidth and FixedHeight values.

The Elements List

The Elements list is a powerful management tool that shows every element, group, and embedded symbol on the canvas in a hierarchical tree. When you add a new item, it appears at the top of the list. The order in this list determines the z-order — which element appears on top when graphics overlap on the canvas.

The Elements list showing a hierarchical tree of canvas objects.

You can use the Elements list to select, rename, group, reorder, and manage elements. It is especially useful for selecting elements that are visually hidden behind other objects on the canvas — you can always find and select them from the list even if you cannot click them directly.

Watch Out — Renaming Breaks References

If you rename an element, group, or embedded symbol that is referenced in an animation or script, the reference will break. Animations and scripts reference elements using the format <Element name>.<Property name>. Renaming does not automatically update these references — you must find and update every animation or script that uses the old name manually. Plan your element names carefully before adding animations.

Adjusting Z-Order

The z-order of items in the Elements list determines which element appears on top when they overlap. To change an element's position in the stack, right-click it in the list, select Order, and choose the desired position. You can also access Order commands from the Arrange menu. Moving an element forward moves it higher in the list (and visually on top); moving it backward moves it lower (and visually behind other elements).

The right-click context menu with Order and Grouping commands.

Grouping Elements

Grouping lets you manage multiple elements as a single unit. Select the elements you want to group, right-click, and use the Grouping commands. Groups receive a default name that you can rename, and they have their own properties that can differ from the properties of the individual elements within them. Grouping is essential for complex symbols where related elements (e.g., a valve body, handle, and flow indicator) need to move, animate, or be managed together.

Element Properties

You control an element's appearance through the toolbar at the top of the Graphic Editor or through properties on the Properties tab. Properties common to all selected elements are displayed, with read-only properties shown in gray and non-default values shown in bold. Properties are organized into categories:

  • Graphic — element name and identifiers.
  • Wizard Options — visual and functional configuration options (available on symbols that support them).
  • Appearance — location, size, orientation, and transparency.
  • Runtime Behavior — visibility, tab order, and other runtime behaviors.
  • Custom Properties — user-defined properties you associate with elements.

Selecting Elements

Elements can be selected by clicking them on the canvas, selecting them from the Elements list, or dragging a lasso around multiple elements. When multiple elements are selected, the last element you selected becomes the primary element — it appears with color-filled handles and serves as the reference point for alignment and spacing operations. All other selected elements are secondary elements (white handles) and follow the edits applied to the primary. To multi-select, hold the Shift key and click individual elements, or drag a lasso around the desired area.

A selected element with resize handles (blue squares) and rotation handle (black circle).
Another example of a selected element ready for geometry editing.

Embedding Symbols

Embedding a symbol creates a linked copy of the original. This is a critical concept: if the original symbol is modified, the changes propagate automatically to every location where it is embedded. This makes embedding the primary mechanism for reusing symbols across your application.

There are several ways to embed symbols onto the canvas:

  • Click the Embed Industrial Graphic button on the toolbar, or select it from the Edit menu, or right-click the canvas and choose Embed Industrial Graphic from the context menu. The Galaxy Browser opens for you to select a graphic.
  • Navigate to the Toolbox, Assets, or Templates tab on the right side of the Graphic Editor, find the symbol you need, and drag it onto the canvas.
Combining embedded symbols and basic shapes to create a custom HMI graphic.

The Toolbox, Assets, and Templates tabs include a search feature. For example, entering "meters" in the Toolbox search box will filter the preview area to show only symbols whose names contain that string, making it easy to find specific symbols in large libraries.

The Toolbox search feature filtering for "meters" symbols.

Wizard Options

When a symbol with preconfigured Wizard Options is selected on the canvas, its Wizard Options appear on the Properties tab. Many Situational Awareness Library symbols come with Wizard Options already configured, allowing you to quickly change the symbol's functional type (e.g., switching an SA_Meters symbol from a flow meter to a temperature meter) without manually editing individual properties.

Custom Properties

Custom properties let you define data interfaces for your symbols. Right-click an embedded symbol and select Custom Properties to open the Edit Custom Properties dialog. Each custom property has a defined data type, default value, visibility setting, and description. The default value is the initial value defined in the source symbol, which you can override with a literal value, a tag reference, or an expression.

The Edit Custom Properties dialog — here the default value is overridden with a tag reference.

Custom properties are also listed and editable on the Properties tab in the Graphic Editor, providing quick access during symbol development.

Substituting Strings and References

When you embed a symbol that contains text graphics or tag references, you often need to customize those values for the specific automation object using the symbol. System Platform provides two substitution tools for this purpose.

Substitute Strings

The Substitute Strings dialog lets you search for and replace text strings in any element on the canvas that has the Text property. To use it, select the embedded symbol and choose Special > Substitute Strings from the menu, or right-click and select Substitute > Substitute Strings. The dialog provides basic find-and-replace functionality as well as advanced options like Match Case and Use Wildcards.

Substituting "Level" with "Temperature" across all text elements in an embedded symbol.

Substitute References

The Substitute References dialog performs the same function but for tag references instead of text strings. Select the embedded symbol, choose Special > Substitute References, and the dialog displays all references used by elements on the canvas. You can replace them manually or use Find & Replace with Match Case and Wildcard support. Note that the Galaxy Browser is available for selecting new references, but auto-complete is not supported within this dialog.

Substituting tag references — changing Me.PV to Me.Speed.PV across the symbol.
Tip

Use Substitute Strings and Substitute References when embedding symbols into automation objects. These tools let you take a single symbol definition and customize it for dozens of different process points — a flow meter for Line 1, a flow meter for Line 2, a flow meter for Line 3 — all from the same source symbol. This is the core efficiency advantage of the symbol-based workflow.

Quiz — Test Your Understanding

Q1: What are the three levels of situational awareness, and why do they matter for HMI design?

The three levels are Perception (noticing new information), Comprehension (understanding what it means), and Projection (anticipating its impact on goals). They matter because effective HMI design should support operators in moving through all three levels quickly — from seeing a change to understanding its significance to deciding on the right action. Graphics designed with situational awareness principles reduce cognitive load and enable faster, safer decision-making.

Q2: What happens if you rename an element that is referenced in an animation?

The animation reference breaks. Animations and scripts use the format <Element name>.<Property name> to reference elements. When you rename an element, the reference is not automatically updated — you must manually find and update every animation or script that uses the old name. This is why it is important to plan element names carefully before adding animations.

Q3: What is the difference between creating a symbol on the Graphics tab versus inside an object instance?

A symbol created on the Graphics tab is a reusable, Galaxy-wide resource available to any automation object. A symbol created inside an object instance exists only within that single instance and cannot be reused elsewhere. For standard or commonly-used symbols, always create them on the Graphics tab to maximize reusability.

Q4: What is the advantage of embedding symbols rather than copying graphics directly?

Embedding creates a linked copy — when the original symbol is modified, changes propagate automatically to every embedded location. Copying graphics directly creates independent copies that must be updated individually. Embedding ensures consistency and dramatically reduces maintenance effort when symbols evolve over time.

Q5: When should you use Substitute References instead of Substitute Strings?

Use Substitute References when you need to change tag bindings (e.g., changing Me.PV to Me.Speed.PV) in an embedded symbol. Use Substitute Strings when you need to change displayed text (e.g., changing the word "Level" to "Temperature"). They target different types of content — references are data connections to the process, while strings are visible labels on the screen.
