Lesson 40: Lab 23 — Using the Web Client

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 8 — Web Client, Lab 23 (pages 537–550)

In this lab, you will enable the Web Client, launch it from WindowMaker, navigate and interact with InTouch graphics in a browser, and acknowledge alarms — all without RDP or IIS.

40.1 Lab Objectives

Reference: Module 8, Lab 23 (page 537)

Upon completion of this lab, you will be able to:

  • Enable Web Client in Application Manager
  • Use the Web Client fast switch to open the Web Client
  • Pan and zoom a graphic in a web browser using a mouse and keyboard
  • Navigate and view Application windows using the Web Client
  • Use Navigation Search to find specific graphics
  • Visualize and acknowledge alarms in Web Client

40.2 Disable Galaxy Security

Reference: Module 8, Lab 23 (pages 538–539)

Lab Step: Because the Web Client relies on Windows authentication for user access, you must first disable Galaxy security for this lab exercise.

Follow these steps to disable security:

  1. Close WindowMaker and check in $MixerView.
  2. In the IDE, on the Galaxy menu, select Configure > Security.
  3. In Authentication Mode, select None.
  4. Click Save. A Galaxy security confirmation message appears.
  5. Click OK to shut down the IDE.
Galaxy security confirmation dialog — the IDE must restart after changing the authentication mode to None (page 538)

40.3 Enable the Web Client

Reference: Module 8, Lab 23 (pages 539–541)

With security disabled, you now enable the Web Client through Application Manager. The Web Client uses local Windows security groups to manage user access, so you will also need to disable a security feature used in the Mixer graphic.

  1. From the Windows Start menu, open Application Manager.
Application Manager main window showing the InTouch tab and ribbon toolbar (page 539)
  1. Select the Web Client tab.
  2. On the toolbar, click the Web Client button. Notice the small clock on the icon — this indicates the Web Server has not started yet.
The Web Client tab showing configuration options: refresh rates, header/navbar toggles, and website name (page 540)
The Web Client button with clock icon and tooltip — click to start the Web Server (page 540)

After a moment, the clock disappears from the icon, indicating the Web Server is running.

  1. Close the Application Manager.
  2. Open the System Platform IDE.
  3. In the Home menu, on the Recent tab, double-click TrainingGalaxy.
The IDE Home tab with TrainingGalaxy listed under Recent projects (page 541)

40.4 Disable MixerDropDown for Web Client

Reference: Module 8, Lab 23 (page 542)

Before launching the Web Client, you need to disable the MixerDropDown combo box in the Mixer template. This is because the Web Client uses local security groups to manage user access, and this control relies on a security feature that must be disabled for browser-based access.

  1. In the IDE, on the Templates tab, in the Training\Project toolset, double-click $Mixer to open it for editing.
  2. In Content, double-click SwitchedMixingProcess to open it for editing.
  3. On the canvas, double-click MixerDropDown to open Edit Animations.
  4. In Animations, for Disable, click the drop-down list and select Disabled.
  5. Click OK to close Edit Animations.
  6. Save and close the graphic.
  7. Save, close, and check in $Mixer.
Edit Animations dialog showing the Disable interaction property set to Disabled for Web Client compatibility (page 542)

40.5 Launch the Web Client

Reference: Module 8, Lab 23 (pages 542–543)

Now you will start WindowViewer to provide live data for the Web Client, then use the Web Client fast switch to open the browser interface.

  1. On the Templates tab, in the Training\ViewApps toolset, double-click $MixerView to open WindowMaker.
  2. Click Open to open the selected windows.
  3. Click Runtime to start WindowViewer.
Key Concept: WindowViewer must be running to provide data and animations for the Web Client. The Web Client is a browser-based viewer — it does not supply its own data.
  1. Click Development! to return to WindowMaker.
  2. In WindowMaker, in the top-right corner, click the Web Client fast switch.

After a few moments, the default web browser opens, and the application windows appear in the Web Client.

  1. Maximize the web browser.
The Web Client in Chrome showing $MixerView with alarm table, navigation bar, and Mixer100 process view (page 543)

40.6 Navigate and Zoom

Reference: Module 8, Lab 23 (pages 543–545)

With the Web Client open, you can interact with the application just as you would in WindowViewer. Start by switching between mixers and then practicing the zoom and pan controls.

  1. In the ContentDisplay window, switch to a different mixer using the dropdown selector and verify the components of the selected mixer are displayed.
  2. Press and hold the Ctrl key, hover the cursor over the Web Client, and then scroll the mouse wheel up and down to zoom in and out.
  3. Release the Ctrl key.
After switching mixers, the ContentDisplay updates to show the selected mixer's process diagram and trend graphs (page 543)
Zoomed-in view showing pump icons, inlet valves, mixer vessel, and Level/Temperature trend graphs with alarm thresholds (page 544)

To pan the view, you must first be zoomed in:

  1. In the Web Client, click and hold down the middle mouse button (scroll wheel), then move up, down, left, and right to pan across the graphic.
  2. On the right side of the Web Client header, click the Home button to restore the Home view.
Zoomed and panned view demonstrating Ctrl+scroll zoom and middle-click pan functionality (page 544)

40.7 View Alarm Aggregation

Reference: Module 8, Lab 23 (pages 545–546)

The Web Client provides full access to alarm displays, including the hierarchical alarm aggregation view that groups alarms by production line and equipment unit.

  1. In the Navigation window, click Alarm Overview and observe the alarms.
  2. Close the AlarmAggregationOverview window.
The Home button location in the right side of the Web Client header bar (page 545)
AlarmAggregationOverview with hierarchical tree: Production level, Line1/Line2 sub-levels, and individual Mixer units with severity counts (page 545)

40.8 Navigate the Web Client Menu and Search

Reference: Module 8, Lab 23 (pages 546–547)

The hamburger menu provides access to all folders and symbols in the Web Client root folder. You can also use the search field to quickly locate specific graphics.

  1. In the top-left corner, click the Web Client menu (hamburger icon).
  2. In the menu, expand Training and select Dashboard.
The hamburger menu icon in the top-left corner of the Web Client (page 546)
Navigation menu with the Training folder expanded and Dashboard highlighted (page 546)
The Dashboard view with four mixer monitoring panels, each showing Level/Temperature trends and equipment status (page 547)

Now use the search field to find a specific library graphic:

  1. In the Web Client menu, in the Search field, enter clock, and then select ClockDigital12.
Search results for "clock" — ClockDigital12 is highlighted for selection (page 547)
ClockDigital12 displayed in the browser — note the LABEL placeholder has not been configured (page 548)
Key Concept: Most library graphics require further configuration (such as setting labels or data references) before they display correctly in the Web Client. The clock graphic shows a LABEL placeholder because it has not been configured.

40.9 Acknowledge Alarms

Reference: Module 8, Lab 23 (pages 548–550)

Finally, you will use the Web Client to acknowledge alarms, demonstrating that the browser-based interface supports interactive alarm management when a ReadWrite license is available.

  1. Click Home to return to the main display.
  2. In the AlarmDisplay window, in the bottom-right corner, click Ack All. The Ack Comment dialog box appears.
The Ack All option in the alarm display context menu (page 548)
  1. In Ack Comment, enter ticket issued.
  2. Click OK to acknowledge all alarms. The alarms now display as acknowledged.
Ack Comment dialog with the comment "ticket issued" entered (page 549)
Alarm table showing all alarms acknowledged with the "ticket issued" comment and ACK state (page 549)
Lab Complete: You have successfully enabled the Web Client, navigated application graphics in a browser, used pan and zoom, searched for symbols, viewed alarm aggregation, and acknowledged alarms — all through the browser-based Web Client interface.

Knowledge Check

1. Why must Galaxy security be disabled before enabling the Web Client in this lab?

Click to reveal answer

The Web Client relies on Windows authentication and local security groups for user access. Galaxy security conflicts with this authentication model, so it must be set to None for the Web Client to function properly.

2. What must be running in WindowViewer before the Web Client can display live data?

Click to reveal answer

WindowViewer must be running to provide data and animations for the Web Client. The Web Client is a viewer — it does not supply its own data.

3. What happens if you use ShowGraphic() without the InTouch: prefix in the Web Client?

Click to reveal answer

Without the InTouch: prefix, the Web Client creates a new popup window instead of replacing the current window. The InTouch:WindowName syntax is required for proper replace behavior.

4. How do you zoom in and pan within the Web Client?

Click to reveal answer

Hold Ctrl and scroll the mouse wheel to zoom. Hold the middle mouse button (scroll wheel) and move the mouse to pan. You must be zoomed in before panning works.

5. What does the LABEL placeholder on the ClockDigital12 graphic indicate?

Click to reveal answer

The LABEL placeholder indicates that the graphic has not been fully configured. Most library graphics require additional configuration of labels and data references before they display correctly in the Web Client.
