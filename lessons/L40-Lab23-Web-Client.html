Lesson 40: Lab 23 – Using the Web Client

Module 8 — Web Client | Pages 537–550

40.1 Introduction

In this final lab, you will enable the Web Client, configure it for your Mixer application, and interact with your InTouch graphics in a web browser. You will navigate between windows, zoom and pan graphics, search for symbols, view alarms, and perform alarm acknowledgment — all through a standard HTML5 browser.

Objectives — Upon completion, you will be able to:
  • Enable Web Client in Application Manager
  • Use the Web Client fast switch to open the Web Client
  • Pan and zoom graphics in a web browser
  • Navigate and view application windows
  • Use Navigation Search
  • Visualize and acknowledge alarms in Web Client

40.2 Disabling Security (Training Only)

Because Web Client relies on Windows authentication, you must disable Galaxy security for this lab. Close WindowMaker, check in $MixerView, then navigate to Galaxy > Configure > Security and set Authentication Mode to None. Click Save — the IDE shuts down and must be restarted.

Training only: Disabling security is done here solely for training purposes. In a production environment, Web Client security is managed through Windows user groups (aaInTouchUsers and aaInTouchRWUsers).

40.3 Enabling the Web Client

Open the AVEVA Application Manager from the Windows Start menu. Select the Web Client tab and click the Web Client button on the toolbar. Watch for the small clock icon on the button — when it disappears, the Web Client has started. Close Application Manager.

AVEVA Application Manager Web Client tab — Web Client button with tooltip indicating the server has not yet started

Open the System Platform IDE and open TrainingGalaxy. On the Templates tab, open $Mixer for editing. Navigate to Content > SwitchedMixingProcess, double-click MixerDropDown, and set the Disable animation to Disabled. This disables the mixer dropdown control for Web Client compatibility. Save, close, and check in $Mixer.

Edit Animations dialog — Disable interaction property set to "Disabled" for MixerDropDown control

40.4 Launching the Web Client

Open $MixerView in WindowMaker. Click Open to open the selected windows, then click Runtime to start WindowViewer. After a few seconds, click Development! to return to WindowMaker.

In WindowMaker, click the Web Client fast switch in the top-right corner. Your default web browser opens, displaying the $MixerView application. Maximize the browser.

Web Client in Chrome showing $MixerView — navigation bar, alarm table, and Mixer100 process view with trend graphs

40.5 Navigating and Interacting

In the ContentDisplay area, switch to a different mixer using the dropdown (which is now disabled for Web Client, so you may need to use navigation buttons). Verify the mixer's components display correctly.

40.5.1 Pan and Zoom

Press and hold the Ctrl key, hover over the Web Client, and scroll the mouse wheel up to zoom in and down to zoom out. Release Ctrl. To pan (when zoomed in), press and hold the mouse wheel and move the cursor in any direction. Click the Home button in the header to restore the default view.

Web Client zoomed-in view — Mixer100 process diagram with pump icons, inlet valves, and trend graphs with alarm thresholds

40.5.2 Viewing Alarms

In the Navigation window, click Alarm Overview to view the alarm aggregation. Close it when done.

Web Client — AlarmAggregationOverview with hierarchical alarm aggregation tree showing Production, Line1, Line2, and individual Mixer nodes

40.6 Using Navigation Search

Click the hamburger menu (three horizontal lines) in the top-left corner. The navigation side menu shows all folders and symbols under the Web Client root folder. Expand Training and select Dashboard to view the dashboard symbol.

Web Client navigation side menu — Training folder expanded with Dashboard highlighted
Web Client showing $MixerView: Dashboard with 2x2 grid of mixer monitoring panels showing real-time trend graphs and controls

Open the navigation menu again and type "clock" in the search field. Select ClockDigital12 from the results. The clock graphic fills the browser — note that most library graphics require further configuration for full Web Client functionality.

Web Client showing ClockDigital12 — retro-style green seven-segment digital clock filling the browser window

40.7 Acknowledging Alarms

Click Home to return to the main view. In the AlarmDisplay window, click Ack All in the bottom-right corner. In the Ack Comment dialog, enter "ticket issued" and click OK. The alarms now display as acknowledged.

Ack Comment dialog with "ticket issued" comment entered
Web Client alarm table showing acknowledged (ACK) alarms with "ticket issued" comments

40.8 Quiz — Test Your Understanding

Quiz

Q1: Why must WindowViewer be running before using the Web Client?

WindowViewer provides the real-time data and animations that the Web Client displays. The Web Client is essentially a browser-based viewer that reads data from the running WindowViewer instance.

Q2: How do you zoom and pan in the Web Client?

Hold Ctrl and scroll the mouse wheel to zoom (up to 500%). Press and hold the mouse wheel and move the cursor to pan. Click the Home button to restore the default view.

Q3: Why was the MixerDropDown control disabled for Web Client?

The MixerDropDown uses an ActiveX or .NET control that is not supported by the Web Client. Disabling it prevents errors when viewing the graphics in the browser. Navigation must be done through the Web Client's own menu system instead.

Q4: What is the difference between Home Windows and Home Symbol behavior in the Web Client?

If Home Windows are configured, they always take precedence and are displayed by default — the Home Symbol is ignored. If only a Home Symbol is configured, it appears by default. If neither is configured, no default graphic is shown.

Q5: How does the Web Server determine ReadWrite vs. Read-Only access?

It first tries to acquire a ReadWrite license. If the user belongs to the aaInTouchRWUsers group and the license is available, they get ReadWrite. Otherwise, it falls back to Read-Only. If no license is available and this is the first session, it operates in Single Session Mode.
