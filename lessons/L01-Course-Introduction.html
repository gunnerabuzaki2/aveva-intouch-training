Lesson 1: Course Introduction

Module 1 — Introduction | Pages 1–6

This section sets the stage for the entire course. It describes what you will learn, what you should already know, and how the four days of training are organized. If you are new to the AVEVA InTouch ecosystem, this lesson will give you a clear roadmap of what lies ahead.

1.1 What This Course Covers

The AVEVA InTouch for System Platform 2023 course is a four-day, instructor-led training program designed to give you a thorough overview of the features and capabilities of AVEVA InTouch when used as part of the System Platform ecosystem. Over the course of four days, you will learn how to build and deploy visualization applications, create graphics, visualize alarms and events, analyze trends and historical data, and implement security controls in an InTouch application. Hands-on labs are woven throughout the course to reinforce each concept with practical experience.

Key Concept: InTouch as a Visualization Client
InTouch for System Platform is not a standalone product — it is a visualization client that works together with AVEVA Application Server. Your displays, graphics, and operator interfaces are all powered by real-time data flowing from the Application Server Galaxy. Understanding this relationship is essential: Application Server provides the data engine, while InTouch provides the screens that operators interact with.

This course is intended for individuals who need to configure or modify AVEVA InTouch for System Platform applications. If you are a system integrator, control engineer, or application developer working with AVEVA visualization tools, this course will give you the foundational knowledge you need.

1.2 Prerequisites

Before diving into the course material, you should have a working knowledge of industrial automation software concepts. More specifically, familiarity with the following Application Server features and technologies will help you get the most out of this training:

  • System Platform IDE — the Integrated Development Environment used to design and configure applications
  • AutomationObjects and their role in the Galaxy — the building blocks of every System Platform application
  • Alarms and how they are configured on attributes — critical for process safety and operator awareness
  • Historization of attributes and data logging — recording process data over time for analysis
  • Security models and permissions — controlling who can do what within the application
  • The deployment model for pushing objects to runtime — moving from development to live operation
  • The plant model and how areas are organized — structuring your application to mirror the physical plant
  • QuickScript.NET scripting language — extending application logic with custom scripts
Tip: If you are new to System Platform, consider reviewing the AVEVA Application Server fundamentals before starting this course. The InTouch visualization layer builds directly on concepts from Application Server — particularly AutomationObjects, the Galaxy, and the deployment model.

1.3 Course Objectives

Upon completion of this course, you will be able to accomplish the following:

  • Describe the visualization components of AVEVA InTouch for System Platform and how they relate to Application Server
  • Create and run visualization applications using WindowMaker and WindowViewer
  • Understand and apply situational awareness concepts to design effective operator displays
  • Create and use graphics effectively, including symbols, animations, and custom properties
  • Implement navigation for visualization applications — building a logical flow between screens
  • Implement alarm visualization in your displays — live alarms, historical alarms, and alarm popups
  • Implement both real-time and historical trending — from simple trend pens to Historian Client controls
  • Implement security controls in your visualization application — authentication, permissions, and runtime lockdown
  • View graphics in a web browser using AVEVA InTouch Access Anywhere
