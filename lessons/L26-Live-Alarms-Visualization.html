Lesson 26: Live Alarms Visualization

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 5 – Alarms and Events Visualization, Section 2 (pages 333–350)

This lesson introduces the Alarm Client control — the primary tool for visualizing live alarms in InTouch. You will learn how to embed, configure, and customize the Alarm Client, and then begin building a current alarms display in Lab 14.

26.1 The Alarm Client Control

Reference: Module 5, Section 2 (pages 333–335)

The Alarm Client control is a graphic element delivered as part of the Graphic Editor. It replaces the older Alarm Viewer and Alarm DB View controls and extends alarm visualization directly into the graphics environment. You place it on a canvas just like any other tool, and then customize it with additional graphics, interactions, and scripts.

Key Concept: The Alarm Client control has five client modes that can be grouped by data source: it supports the InTouch Database (WWALMDB), the ArchestrA Database (A2ALMDB), and Historian History Blocks. This flexibility allows a single control to display current alarms, recent alarms and events, historical alarms, historical events, or a combination of both.

At runtime, operators can interact with the Alarm Client in several ways:

  • Refresh the grid to show the most current alarms
  • Use the status bar to view alarm record information
  • Acknowledge, hide, filter, or sort alarms
  • Freeze the grid to examine a specific snapshot
  • Switch between client modes
  • Switch between languages

Embedding the Alarm Client

To embed the Alarm Client control, open the Graphic Editor, select the Alarm Client tool from the Tools pane, and click the canvas to place it. You can resize the control and double-click it to access configuration options.

26.2 Configuration Options

Reference: Module 5, Section 2 (pages 335–341)

Alarm Mode

The Client Mode property determines which type of alarms to display. You can set this at design time or switch it at runtime using the ClientMode integer property in scripting. The available modes are:

Client ModeDescriptionValue
Current AlarmsDisplays real-time active alarms from the alarm provider1
Recent Alarms and EventsShows recently occurred alarms and events2
Historical AlarmsRetrieves alarm records from the alarm database or History Blocks3
Historical EventsRetrieves event records from the alarm database or History Blocks4
Historical Alarms and EventsRetrieves both alarm and event records5

The Alarm Query field specifies which alarms to retrieve using the query syntax covered in Lesson 24. Use Ctrl + Enter to create new lines in the query field.

Colors

Different alarm types can be displayed in different colors using priority breakpoints. The Alarm Client supports four priority ranges, each with separate colors for acknowledged and unacknowledged states. You can also customize the control background, grid, and heading colors.

Column Details

Column headers can be renamed, resized, and reordered. Changes are reflected in the grid preview, where you can also use the pointer to adjust column widths and order.

Queries and Filters

Query filters allow you to narrow the displayed alarms using logical constructs. For example, you can filter to show only alarms with priorities between 500 and 750. Filters created at design time can be reused at runtime, and vice versa.

Important: When using TimeLCT, TimeOAT, or TimeLCTOAT as filter criteria for historical alarm modes, ensure the TimeSelector.StartDate and TimeSelector.EndDate properties do not inadvertently limit your query. Set StartDate earlier and EndDate later than any time filtering requirement.

Run-Time Behavior

The Alarm Client's runtime behavior is highly configurable: you can show or hide the heading, grid, and status bar; control whether the grid queries on startup; enable auto-scrolling to new alarms; hide warnings and errors; restrict operator access; and customize the context menu.

Events

The Event configuration lets you write scripts that execute when specific client control events occur. Each event provides parameters with defined data types and names.

26.3 Lab 14 – Building a Current Alarms Display (Part 1)

Reference: Module 5, Lab 14 (pages 343–350)

Lab Objectives:
  • Use and configure the Alarm Client to visualize current alarms
  • Configure and select alarm filters in the Alarm Client

In this lab, you will create a symbol with the Alarm Client, configure it to query alarms from the Production area, and add filters for Line1, Line2, and Production.

Create the Alarm Display

  1. In the IDE, on the Graphics tab, in the Training toolset, create a new symbol named AlarmLiveDisplay and open it for editing.
  2. In Tools, click Alarm Client and place it on the canvas.
  3. Double-click the Alarm Client to configure it.
  4. With Alarm Mode selected, ensure Client Mode is set to Current Alarms.
  5. In Alarm Query, enter \\<NodeName>\Galaxy!Production.
Tip: Your instructor will provide the node name. In a real-world scenario, this would point to a production node. For this course, the GR node provides alarms.
Configuring the Alarm Client with Current Alarms mode and the Production area alarm query (page 344)

Add Filters

  1. In Animations, select Queries and Filters.
  2. In Query and Filter Favorites, click Add new filter.
  3. Name the filter Line1. Select Group from the filter criteria list and click the right arrow to add it.
  4. In the filter definition, select Group, verify the operator is =, enter Line1 in Value, and click Set.
  5. Click OK to close Add Query or Filter.
  6. Click Add new filter again. Name it Line2 and configure Group = Line2.
  7. Add a third filter named Production with Group = Production.
  8. Click OK to close Edit Animations.

Test at Runtime

  1. Click AlarmClient and configure its size: Width 1920, Height 200.
  2. Save and close, then check in the graphic.
  3. In WindowMaker, embed AlarmLiveDisplay on the AlarmDisplay window.
  4. In Properties, uncheck Maintain aspect ratio so the display stretches to fit.
  5. Click Runtime and verify that current alarms are displayed in the AlarmDisplay window.
Live alarms displayed in the AlarmDisplay window at runtime, showing current alarms from the Production area (page 350)

Knowledge Check

1. What are the five client modes available in the Alarm Client control?

Click to reveal answer

Current Alarms (1), Recent Alarms and Events (2), Historical Alarms (3), Historical Events (4), and Historical Alarms and Events (5). These modes determine what data the Alarm Client displays.

2. How do you switch the Alarm Client between Current Alarms and Historical Alarms at runtime?

Click to reveal answer

Use the ClientMode integer property in scripting. Set it to 3 for Historical Alarms, 4 for Historical Events, or 5 for Historical Alarms and Events.

3. What must you do to the AlarmLiveDisplay after embedding it in the AlarmDisplay window to ensure it fills the available space?

Click to reveal answer

Uncheck the Maintain aspect ratio property. This allows the embedded symbol to stretch and fill the entire window rather than maintaining its original proportions.

4. What is the purpose of the three filters (Line1, Line2, Production) created in this lab?

Click to reveal answer

These filters allow operators to narrow the displayed alarms to a specific area at runtime. By right-clicking and selecting Queries and Filters, an operator can switch between viewing all Production alarms, Line1-only alarms, or Line2-only alarms.
