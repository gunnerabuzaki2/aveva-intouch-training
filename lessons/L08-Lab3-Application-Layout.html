Lesson 8: Lab 3 – Building the Application Layout

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 2, Lab 3 (pages 61–80)

In this lab, you will create five windows for the Managed InTouch application: Navigation, Logon, AlarmDisplay, OverviewDisplay, and ContentDisplay. You will then embed a clock symbol and test the application in runtime.

Lab Objectives:
  • Create InTouch windows with specific properties and dimensions
  • Configure InTouch window properties (type, location, size, color, title bar)
Tip: The steps and screenshots in this lab are designed for a resolution of 1920 × 1080. Adjust coordinates and sizes if your development resolution differs.

8.1 Create the Navigation Window

Reference: Module 2, Lab 3 (pages 62–65)

The Navigation window sits at the top of the application and will contain buttons that allow operators to navigate between other windows at runtime. It uses the Frame window template and is configured as an Overlay type.

  1. In the top-left corner of WindowMaker, click the New (+) button to open the backstage view.
Click the New (+) button in the top-left corner of WindowMaker (page 62)
The backstage New view displays available window templates (page 62)
  1. In Available templates, select Frame window.
Select the Frame window template (page 63)

Now configure the Navigation window's properties:

  1. In Name, enter Navigation.
  2. In Comment, enter Display system navigation graphics.
  3. In the Window type drop-down, select Overlay.
  4. Set Location to X: 0, Y: 0 and Size to Width: 1300, Height: 120.
  5. Uncheck Title bar and Size controls.
Configure the Navigation window properties as shown — note the Overlay type and dimensions (page 64)
  1. Click Create. The Navigation window appears in the canvas.
The Navigation window now appears in the Base Windows tree and on the design canvas (page 65)

8.2 Create the Logon Window

Reference: Module 2, Lab 3 (pages 65–67)

The Logon window displays a clock and will later contain a symbol that allows users to log in to the application. It sits to the right of the Navigation window.

  1. Click the New (+) button again.
  2. Ensure Frame window is selected.
  3. In Name, enter Logon.
  4. In Comment, enter Display logon and clock graphics.
  5. Ensure the Window type is Overlay.
  6. Set Location to X: 1300, Y: 0 and Size to Width: 620, Height: 120.
  7. Click Create.
Configure the Logon window properties — positioned immediately to the right of the Navigation window (page 66)

You can scroll the canvas to the right to see the Logon window, then scroll back to reset the view.

The canvas now shows both the Navigation and Logon windows. Use the horizontal scrollbar to navigate between them (page 67)

8.3 Create the AlarmDisplay Window

Reference: Module 2, Lab 3 (pages 68–69)

The AlarmDisplay window spans the full width of the screen below the Navigation and Logon windows. It will later host the Alarm Client symbol for displaying live alarms.

  1. Click the New (+) button.
  2. Select Frame window.
  3. Name: AlarmDisplay, Comment: Display live alarms.
  4. Window type: Overlay.
  5. Set Location to X: 0, Y: 120 and Size to Width: 1920, Height: 200.
  6. Click Create.
Configure the AlarmDisplay window — note it starts at Y=120 (directly below Navigation and Logon) and spans the full 1920 width (page 68)
The AlarmDisplay window appears in the canvas as a large horizontal panel (page 69)

8.4 Create the OverviewDisplay Window

Reference: Module 2, Lab 3 (pages 70–72)

The OverviewDisplay window occupies the left side of the screen below the alarm area. It has a distinctive wheat-colored background and will display dashboard graphics for the Mixer application.

  1. Click the New (+) button.
  2. Select Frame window.
  3. Name: OverviewDisplay, Comment: Display system overview graphics.
  4. Window type: Overlay.
  5. Set Location to X: 0, Y: 320 and Size to Width: 420, Height: 680.
  6. Click the ellipsis button next to Window color.
Configure the OverviewDisplay properties and click the ellipsis to change the background color (page 70)
  1. In the Color Dialog, select the Web tab and scroll to the bottom of the list.
  2. Select Wheat.
The Color Dialog — Professional tab with color picker (page 71)
Switch to the Web tab and select Wheat for the OverviewDisplay background (page 71)
  1. Click OK to close the Color Dialog.
  2. Click Create. The OverviewDisplay window appears with its wheat-colored background.
The properties dialog now shows Wheat as the window color — click Create (page 72)
The OverviewDisplay window appears with its distinctive wheat-colored background (page 72)

8.5 Create the ContentDisplay Window

Reference: Module 2, Lab 3 (pages 73–74)

The ContentDisplay window is the largest window and occupies the main area to the right of OverviewDisplay. Unlike the other windows, it uses the Replace window type and has a white background with a title bar enabled.

  1. Click the New (+) button.
  2. Select Frame window.
  3. Name: ContentDisplay, Comment: Display system graphics.
  4. Window type: Replace.
  5. Set Location to X: 420, Y: 320 and Size to Width: 1500, Height: 680.
  6. Click the ellipsis next to Window color, select the Web tab, scroll to the bottom, and select White.
  7. Click OK to close the Color Dialog.
  8. Check the Title bar checkbox.
  9. Click Create.
Select White from the Web tab for the ContentDisplay window background (page 73)
Enable the Title bar for ContentDisplay and click Create (page 74)
All five windows are now created: Navigation, Logon, AlarmDisplay, OverviewDisplay, and ContentDisplay (page 74)
Key Concept: The ContentDisplay window uses the Replace type, meaning it will close any overlapping windows when it appears. It also has a title bar enabled, unlike the other overlay windows.

8.6 Embed a Clock Symbol

Reference: Module 2, Lab 3 (pages 75–76)

Now you will add a clock graphic to the Logon window so that operators can see the current time at runtime. The clock symbol is available from the Industrial Graphic Library in the Graphics panel.

  1. In Canvas, scroll to the right to show the Logon window.
  2. In Industrial Graphics, expand Industrial Graphic Library > Clocks.
  3. Drag the ClockAnalogWall symbol from Industrial Graphics onto the Logon window in the canvas.
Scroll right in the canvas to reveal the Logon window area (page 75)
Expand Industrial Graphic Library > Clocks to find the ClockAnalogWall symbol (page 75)
Drag the ClockAnalogWall symbol from the Industrial Graphics panel onto the Logon window (page 75)
  1. The clock appears in the Logon window. Position and resize it to fit. Scroll back to the left to reset the view.
The ClockAnalogWall symbol is now placed in the Logon window (page 76)

8.7 Test in Runtime

Reference: Module 2, Lab 3 (pages 76–77)

To verify that everything works, switch to WindowViewer (the runtime environment) using the Runtime fast switch. WindowMaker automatically saves any open windows before switching.

  1. In the top-right corner, click the Runtime button to display the windows in the runtime environment.
Click the Runtime button in the top-right corner to switch to WindowViewer (page 76)

After a moment, WindowViewer opens and all five windows appear. The clock in the Logon window displays the current time. You can identify each window by its position and size:

  • Navigation — Top-left wide panel for navigation controls
  • Logon — Top-right panel with the analog clock
  • AlarmDisplay — Full-width panel below the top row for alarms
  • OverviewDisplay — Left vertical panel with wheat-colored background
  • ContentDisplay — Large central/right panel for process graphics
Runtime view showing all five windows laid out correctly. The clock in the Logon window displays the current time (page 77)
  1. Click Development! in the upper-right corner to return to WindowMaker.
Click the Development! button to return to WindowMaker from runtime (page 77)

8.8 Lock Window Size

Reference: Module 2, Lab 3 (pages 78–79)

Finally, enable the Lock window size option to prevent windows from changing their scale when viewed at different resolutions. This is an important production setting.

  1. In WindowMaker, open the backstage view and select Configure > WindowMaker.
  2. In the WindowMaker configuration, check Lock window size.
  3. Click Save.
Open WindowMaker configuration from the backstage menu (page 78)
Check Lock window size and click Save (page 78)

A notification appears indicating that WindowMaker must be restarted for the changes to take effect.

  1. Close WindowMaker (this also closes WindowViewer).
  2. In the Check-in dialog, enter the comment: Created windows and configured Lock Window Size.
  3. Click Check-in to check in the $MixerView template.
  4. When the check-in progress completes, click Close.
WindowMaker notifies you that a restart is required for the Lock window size change (page 79)
Enter a comment and click Check-in to save your changes to the Galaxy (page 79)
Key Concept: Always check in your InTouch application when closing WindowMaker. This saves your changes to the Galaxy repository and allows other team members to work with the latest version.

Knowledge Check

1. What window type should you use for the Navigation and Logon windows, and why?

Click to reveal answer

Both use the Overlay type. Overlay windows appear on top of currently displayed windows but allow click-through to underlying windows — ideal for navigation bars that should remain visible while operators interact with content below.

2. Why does the ContentDisplay window use the Replace type instead of Overlay?

Click to reveal answer

The ContentDisplay uses Replace because it is the main content area — when a new content window appears, it should close any overlapping Replace windows to avoid clutter. The Replace type automatically closes any windows it intersects.

3. What are the Y-coordinate positions for each window in the layout, and what do they tell you about the screen layout?

Click to reveal answer

Navigation and Logon are at Y=0 (top row), AlarmDisplay at Y=120 (below top row, height 200), and OverviewDisplay/ContentDisplay at Y=320 (main content area). This reveals a three-tier layout: navigation bar, alarm strip, and content area.

4. What happens when you click the Runtime button, and what does WindowMaker do before switching?

Click to reveal answer

Clicking Runtime switches from WindowMaker to WindowViewer (the runtime environment). WindowMaker automatically saves all open windows before displaying them in runtime. In runtime, you see the actual operator view with live animations.

5. Why is the Lock window size setting important for production deployments?

Click to reveal answer

Lock window size prevents windows from changing their scale when viewed at a different resolution than they were designed for. Without it, windows could appear stretched, compressed, or distorted on monitors with different resolutions, making the interface difficult to use.
