Lesson 2: System Platform Overview

Module 1 — Introduction | Pages 7–10

This section describes fundamental concepts about AVEVA System Platform and AVEVA System Platform Enterprise. It also introduces ArchestrA technology — the distributed architecture that makes everything possible. If you understand the layers described here, the rest of the course will make much more sense.

2.1 What is AVEVA System Platform?

AVEVA System Platform, formerly known as Wonderware System Platform, is an industrial software platform built on ArchestrA technology. It provides an integrated set of services and an extensible data model for managing plant control and information management systems. What makes System Platform particularly powerful is that it supports both the supervisory control layer and the manufacturing execution system layer, presenting them as a unified information source.

Key Concept: Components vs. Clients
System Platform has a clear separation between components (which access and process external data from controllers and other sources) and clients (which access information from System Platform for visualization and reporting). Understanding this distinction is fundamental to working with the architecture.

The System Platform software is built on industry standards and Microsoft technologies including Windows, .NET, SQL Server, and IIS. ArchestrA provides the foundational technology and services that make the multi-user, object-oriented platform possible. Together, System Platform and its clients provide the framework and tools for developing, executing, monitoring, and visualizing your industrial applications.

Think of System Platform as a layered system with three groups:

System Platform layered architecture — Supervisory Clients on top, System Platform Components in the middle, and Field Devices at the bottom (Module 1, page 1-7)

2.2 System Platform Components

System Platform includes three core components that handle the heavy lifting of data acquisition, processing, and storage:

AVEVA Application Server is the heart of System Platform. It provides an object-oriented framework with tools for developing and deploying applications. Everything in System Platform revolves around the Application Server and the Galaxy it manages. When you create AutomationObjects, configure attributes, set up alarms, and define deployment models, you are working within Application Server.

AVEVA Historian provides process data historization, along with alarm and event logging for Application Server. Data stored in Historian is exposed through SQL Server and/or an Open Data Protocol (oData) interface, making it accessible for reporting and analysis. Historian is what enables you to look back at what happened yesterday, last week, or last month.

AVEVA Communication Drivers provide the platform for communicating with controllers and other data sources. Beyond controller-specific protocols, they also support open connectivity protocols such as MQTT, OPC DA, and OPC UA. These drivers were formerly called OI Servers and also include legacy DA Servers and IO Servers. System Platform can also communicate directly with third-party drivers, such as third-party OPC Servers, giving you flexibility in how you connect to your existing control infrastructure.

2.3 System Platform Clients

System Platform clients are the visualization and reporting tools that sit on top of the core components. There are two categories of clients worth understanding:

Supervisory Clients run the operator interface and provide real-time access to Application Server data, alarms, and events. There are two supervisory client products, and both can coexist in the same System Platform solution:

  • AVEVA Operations Management Interface (OMI) has a rapid-design visualization framework with tools and provides out-of-the-box navigation based on the application's data model. OMI is the newer, model-driven approach to building operator displays.
  • AVEVA InTouch for System Platform uses traditional development tools to create displays and the corresponding navigation framework from scratch. This is the course you are taking — you will learn to build InTouch applications from the ground up. Note that InTouch is not included with AVEVA System Platform Enterprise.

Supervisory client applications can also be accessed remotely through a web browser using AVEVA InTouch Access Anywhere. With Access Anywhere, you simply enter a URL in a web browser and log on — no separate client application needs to be installed on the remote device. This is particularly useful for mobile operators or managers who need occasional access to process screens.

AVEVA Historian Client is a collection of tools for accessing and reporting on historized data. It includes a Trend application for plotting data graphically, a Query application for building SQL queries through a point-and-click interface, and add-ins for Microsoft Excel and Word for generating reports. AVEVA Historian Client Web provides similar capabilities through a browser-based interface.

2.4 Core Services Provided by System Platform

Together, System Platform and its clients deliver the core services needed to develop, implement, and deploy industrial automation applications:

  • Real-time data acquisition from field devices (PLCs, RTUs)
  • Scaling, statistics, and manipulation of data
  • Process control
  • Alarm triggering, logging, acknowledgment, and management
  • Historization, trending, and analysis of process data
  • Visualization of real-time process data
  • Reporting

Applications built on System Platform are extensible through scripting within the product itself or through available toolkits. For example, a script can access a third-party web service to integrate external data into your application.

2.5 ArchestrA Technology

ArchestrA is the distributed architecture that underpins System Platform. It is an open and extensible technology based on a distributed, object-oriented design, leveraging the Microsoft .NET Framework for the industrial automation world. ArchestrA provides several critical system services:

  • Object management for creating object-oriented applications
  • A component object framework for modeling plants, factories, and equipment
  • A common, global namespace for all application types, from single-node to distributed networked applications
  • Inter-process communications with system maintenance and diagnostic information
  • Centralized security services supporting multi-user environments for development and runtime
  • Version management for every object, including logging of development operations

Most of these services are exposed, configured, and managed through Application Server. When you work in the IDE and deploy objects to runtime, you are using ArchestrA services under the hood.

Key Concept: System Platform vs. System Platform Enterprise
System Platform Enterprise builds on the core technology to add enhanced user accessibility and exclusive visualization extensions for enterprise users building AVEVA Unified Operations Center (UOC) solutions. Enterprise includes the OMI Web Client and additional widgets (ScatterChart, NavigationTree, Graphic Repeater, MapApp, etc.) and OMI Apps (Power BI, PI Vision, Sankey). However, Enterprise does not include InTouch HMI (WindowMaker and WindowViewer), the InTouch Web Client, or InTouch Workspaces. This is an important distinction — if you are working with InTouch, you need the standard System Platform, not Enterprise.

Quiz — Test Your Understanding

Q1: What are the three core components of AVEVA System Platform?

AVEVA Application Server, AVEVA Historian, and AVEVA Communication Drivers.

Q2: What technology does ArchestrA leverage to provide its services?

ArchestrA leverages the Microsoft .NET Framework, providing an open and extensible distributed object-oriented platform.

Q3: What is the key difference between AVEVA System Platform and System Platform Enterprise?

System Platform Enterprise adds exclusive features like the OMI Web Client, additional web widgets, and OMI Apps (Power BI, PI Vision, Sankey), but it does NOT include InTouch HMI (WindowMaker/WindowViewer), InTouch Web Client, or InTouch Workspaces.

Q4: How can supervisory client applications be accessed remotely?

Through AVEVA InTouch Access Anywhere, which allows remote access via a web browser on a desktop or mobile device without requiring a separate client application to be installed.
