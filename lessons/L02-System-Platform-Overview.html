Lesson 2: System Platform Overview

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 1, Section 2 (pages 7–10)

Understanding AVEVA System Platform, its layered architecture, ArchestrA technology, and the role of Application Server as the foundation for InTouch visualization.

2.1 AVEVA System Platform — The Big Picture

Reference: Module 1, Section 2 (pages 7–8)

AVEVA System Platform (formerly Wonderware System Platform) is an industrial software platform built on ArchestrA technology. It provides an integrated set of services and an extensible data model to manage plant control and information management systems. System Platform supports both the supervisory control layer and the manufacturing execution system layer, presenting them as a single information source.

Key Concept: System Platform provides the core framework and tools for developing, executing, monitoring, and visualizing industrial automation applications. Core services (data acquisition, historization, alarming) come from System Platform components, while client services (visualization, trending) come from its clients.

The System Platform software is built on industry standards and Microsoft technologies, including Windows, .NET, SQL Server, and IIS. ArchestrA provides the fundamental technology and services for the multi-user, object-oriented platform.

System Platform Components

System Platform components access and process external data from controllers, software applications, and other data sources. The three main components are:

ComponentRole
AVEVA Application ServerThe heart of System Platform. Provides an object-oriented framework with tools for developing and deploying applications.
AVEVA HistorianProvides process data historization and alarm/event logging. Data is exposed through SQL Server and/or OData interfaces.
AVEVA Communication DriversPlatform for communication with controllers and data sources. Supports MQTT, OPC DA, OPC UA, and controller-specific protocols.

2.2 System Platform Clients

Reference: Module 1, Section 2 (pages 8–9)

System Platform clients access information from the core platform and provide the operator-facing interfaces. There are several client products designed for different use cases:

Supervisory Clients

Supervisory clients run the operator interface and provide real-time access to Application Server data, alarms, and events. Two supervisory client products can coexist in the same System Platform solution:

  • AVEVA Operations Management Interface (OMI) — a rapid-design visualization framework with out-of-the-box navigation based on the application’s data model.
  • AVEVA InTouch for System Platform — based on traditional development tools to create displays and navigation from scratch. This is the focus of this training course.

Remote Access

Supervisory client applications can be accessed remotely through a web browser using AVEVA InTouch Access Anywhere. Simply enter a URL in a web browser on a desktop or mobile device, log on, and interact with the application — no separate client software installation required.

Historian Clients

  • AVEVA Historian Client — a collection of tools for accessing historized data: a Trend application for graphing, a Query application for SQL queries, and add-ins for Microsoft Excel and Word.
  • AVEVA Historian Client Web — a browser-based tool for quick data query, trending, and analysis.

Core Services Provided

  • Real-time data acquisition from field devices
  • Scaling, statistics, and data manipulation
  • Process control
  • Alarm triggering, logging, acknowledgment, and management
  • Historization, trending, and analysis of process data
  • Visualization of real-time process data
  • Reporting

2.3 ArchestrA Technology

Reference: Module 1, Section 2 (page 1-9)

ArchestrA is a distributed architecture developed for supervisory control and manufacturing information systems. It is an open, extensible technology based on a distributed, object-oriented design that leverages the Microsoft .NET Framework for industrial automation.

Key Concept: ArchestrA provides the foundational technology layer for System Platform, including object management, a component framework, global namespace, inter-process communications, centralized security, and version management.

ArchestrA provides the following system services:

  • Object management for creating object-oriented applications
  • Component object framework for modeling plants, factories, and equipment
  • Common global namespace for all application types, from single-node to distributed
  • Inter-process communications with system maintenance and diagnostic information
  • Centralized security with support for multi-user environments
  • Version management for every object, including development operation logging

2.4 System Platform Enterprise

Reference: Module 1, Section 2 (page 1-9)

AVEVA System Platform Enterprise builds on the core technology to bring enhanced accessibility and exclusive visualization extensions for enterprise users building AVEVA Unified Operations Center (UOC) solutions. It is an AVEVA Flex subscription-activated product.

Exclusive features of System Platform Enterprise include:

  • A web client for Operations Management Interface (OMI)
  • Additional web widgets: ScatterChart, Breadcrumb, NavigationTree, Hamburger, TitleBar, Graphic Repeater, MapApp
  • Additional OMI Apps: Power BI, PI Vision, Sankey
  • UOC documentation
Important: System Platform Enterprise does not include InTouch HMI (WindowMaker and WindowViewer), InTouch Web Client, or InTouch Workspaces. These are part of the standard AVEVA InTouch for System Platform product.
