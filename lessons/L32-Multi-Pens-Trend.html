Lesson 32: Multi Pens Trend

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 6, Section 3 (pages 401–422)

This lesson covers the Multi Pens Trend control — a built-in tool for displaying multiple process values on a single chart — and walks through the complete configuration of a three-pen trend popup.

32.1 Introduction to Multi Pens Trend

Reference: Module 6, Section 3 (pages 401–402)

The Graphic Editor offers two built-in trend controls. The first is the Trend Client control, which can chart real-time and historical data from Application Server attributes or InTouch tags. The second — and the focus of this course — is the Multi Pens Trend, which displays a succession of process values with multiple trend lines on a single chart.

The Multi Pens Trend is available from the Tools pane in the Graphic Editor. Once placed on the canvas, the Edit Animations dialog lets you configure up to eight pens, each with its own data reference, color, weight, pattern, and plot type. A pen's reference can be an object attribute, an InTouch tag, or a custom property — constants and expressions are not allowed.

Key Concept: The Multi Pens Trend supports up to eight pens per chart. Each pen needs a valid data reference (object attribute, InTouch tag, or custom property), a min/max range, and a display style.

32.2 Multi Pens Trend Configuration

Reference: Module 6, Section 3 (pages 401)

The Multi Pens Trend configuration has two tabs: Trend Pens and Options. The Trend Pens tab lets you add, remove, and configure individual pens. The Options tab controls the overall chart appearance and data source.

OptionDescription
Fixed TimeDetermines whether a Date Time Picker is available at runtime
Time IntervalSets the default chart duration (e.g., 5 Min)
Scale LabelControls whether scale labels appear on the left, right, both sides, or neither

Options tab: Fixed Time unchecked, 5-minute duration, Scale Label on left, Historian source with Auto Detect (page 401)

32.3 Multi Pens Trend at Runtime

Reference: Module 6, Section 3 (pages 402)

At runtime, the Multi Pens Trend displays the chart with a legend at the top showing each pen's name, reference, current value, range, and engineering units. You can right-click the legend to show, hide, add, or delete pens. The bottom control bar provides play/pause buttons and a time-interval selector.

Pitfall: Changes made at runtime (adding/removing pens, changing time intervals) do not persist beyond the current session. Once the control is reloaded, all runtime changes are lost and the design-time configuration returns.

32.4 Knowledge Check

Knowledge Check

1. What is the maximum number of pens the Multi Pens Trend control supports?

Click to reveal answer

Eight pens per chart. Each pen requires a unique data reference, min/max range, and display style.

2. What happens to runtime changes (such as adding a pen or changing the time interval) when the Multi Pens Trend is reloaded?

Click to reveal answer

All runtime changes are lost. The control reverts to its design-time configuration. Runtime changes do not persist beyond the current session.

3. Why does the Reference column in the TrendPopup legend not update when switching between mixers at runtime?

Click to reveal answer

The default in-memory caching setting in WindowViewer causes the last state of an Industrial Graphic to persist. Disabling "Cache Industrial Graphics not embedded in InTouch windows" in WindowViewer's Memory settings resolves this.

4. What is the purpose of the Auto Detect option in the Multi Pens Trend's Options tab?

Click to reveal answer

Auto Detect automatically finds the Historian node for AppServer references, enabling backfill so operators don't have to wait for the full time period to see data on the chart.

5. What type of data reference is required for a Multi Pens Trend pen?

Click to reveal answer

A valid external reference such as an object attribute, an InTouch tag, or a custom property. Constants and expressions are not allowed.
