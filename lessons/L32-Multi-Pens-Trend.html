Lesson 32: Multi Pens Trend

Module 6 — Trend Visualization | Pages 401–422

32.1 Multi Pens Trend Overview

The Multi Pens Trend is the primary trend control for displaying multiple process values on a single chart. Available from the Tools pane in the Graphic Editor, it supports up to eight pens — each with its own reference, color, weight, pattern, and plot type. Unlike the SA_Trend_MultiPen (which is limited to four pens with predefined styles), the Multi Pens Trend gives you full control over every aspect of the visualization.

When you place the Multi Pens Trend on the canvas, the Edit Animations dialog opens with two tabs: Trend Pens (for configuring individual pens) and Options (for chart-wide settings). Each pen requires a reference to a data source — this can be an object attribute, an InTouch tag, or a custom property. Constants and expressions are not allowed as pen references.

Options tab — Fixed Time, scale labels, grid colors, and the default Historian source with Auto Detect are configured here.
Key Concept: Historian Backfill with Auto Detect
The Multi Pens Trend defaults to using Historian as its data source with Auto Detect enabled. This means at runtime, the chart is immediately backfilled with historical data from the configured Historian node — operators don't have to wait for the full time period to elapse before seeing data. The Auto Detect feature automatically discovers the Historian node for Application Server references.

32.2 Multi Pens Trend at Runtime

At runtime, the Multi Pens Trend displays a legend table at the top showing each pen's name, reference, current value, range, and engineering units. Below the legend is the trend chart itself, plotting all configured pens simultaneously. The bottom control bar provides:

  • Time period selector — Change the chart duration (1 Min, 5 Min, 10 Min, 15 Min, 30 Min, 1 Hour)
  • Pause/Resume — Stop or restart the scrolling trend
  • Fixed Time — When enabled, shows a Date Time Picker to select a specific start time

Operators can right-click in the legend to show, hide, add, or delete pens at runtime. A tooltip appears when hovering over the chart, showing each pen's value, quality, engineering units, and timestamp at that point.

Runtime — three pens (Temperature, Level, Agitator) plotted together with live values in the legend and a 5-minute time window.
Warning: All changes made at runtime — adding pens, changing time periods, pausing — are temporary. They do not persist beyond the current session and do not affect the design-time configuration. Once the Multi Pens Trend control is reloaded, all runtime modifications are lost.

32.6 Disabling In-Memory Caching

You may notice that when you switch to a different mixer and reopen the TrendPopup, the pen references still point to the previous mixer. This is caused by WindowViewer's in-memory caching setting, which preserves the last state of Industrial Graphics when they are reopened. To fix this:

  1. Go to File > Configure > WindowViewer
  2. Select the Memory tab
  3. Uncheck "Cache Industrial Graphics not embedded in InTouch windows"
  4. Click Save, then close and restart WindowViewer

After restarting, the TrendPopup will correctly update its references when you switch between mixers.

WindowViewer Memory settings — uncheck the caching option to ensure trend references update when switching between mixers.

Quiz — Test Your Understanding

Q1: What is the maximum number of pens the Multi Pens Trend control supports?

The Multi Pens Trend supports up to eight pens. This is more than the SA_Trend_MultiPen, which is limited to four.

Q2: Why do runtime changes to the Multi Pens Trend (adding pens, changing time period) not persist?

Runtime changes are stored in memory only and do not write back to the design-time configuration. Once the control is reloaded or the session ends, all modifications are lost. This is by design — it allows operators to temporarily explore data without permanently altering the HMI.

Q3: What causes trend pen references to not update when switching between different mixer instances?

WindowViewer's in-memory caching setting preserves the last state of Industrial Graphics when they are reopened. Disabling "Cache Industrial Graphics not embedded in InTouch windows" in the Memory tab forces the graphic to reload fresh references each time.

Q4: What are the two categories of Historian Client .NET controls, and how do they differ?

Application controls (aaHistClientTrend, aaHistClientQuery) run as standalone applications within the container and require minimal scripting. Building block controls (aaHistClientTagPicker, aaHistClientTimeRangePicker, etc.) provide specific functionality but require scripting to connect and operate.

Q5: What happens when Auto Detect is enabled for the Historian source in the Multi Pens Trend?

Auto Detect automatically discovers the Historian node associated with Application Server references. At runtime, the trend immediately backfills with historical data from that Historian, so operators see a populated chart without waiting for the full time period to elapse.
