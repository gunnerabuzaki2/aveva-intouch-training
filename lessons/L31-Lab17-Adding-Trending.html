Lab 17 — Adding Trending to Graphics

In this lab, you will display a real-time data plot using the SA_Trend_SinglePen symbol from the Situational Awareness Library. By embedding Level and Temperature trend charts directly inside the mixing process graphic, operators gain an immediate visual sense of how key process variables evolve over time — without navigating to a separate trend window.

Objectives:
  • Configure the SA_Trend_SinglePen symbol to display real-time and historical data
  • Use Substitute References and Substitute Strings to redirect generic symbol bindings to specific object attributes
  • Embed trend graphics into a process display using Relative References so that each mixer instance shows its own data

31.1 Creating the Level Trend

The process begins inside the $Mixer template, where you add a new symbol called LevelTrend_Detailed. Once opened for editing, you search the Toolbox tab for "single" and drag the SA_Trend_SinglePen onto the canvas. This pre-built symbol from the Situational Awareness Library includes a trend chart, a vertical meter, alarm limit indicators, and a floating tooltip — all wired together and ready to be configured through its wizard options.

The wizard exposes a rich set of properties that control how the trend looks and behaves. For the Level trend, you configure it as a smooth Line plot with an alarm border, a vertical Level-type meter, and all four alarm limit indicators (HiHi, Hi, Lo, LoLo) enabled. The floating tooltip is also turned on so that operators can hover over the chart to see the current PV, maximum, minimum, and average values at any point in time.

31.2 Reconfiguring References with Substitute Tools

At this point, the SA_Trend_SinglePen still contains its generic references — everything points to Me.PV, which is a placeholder. To connect the trend to the actual Level attribute of the mixer, you use the Substitute References tool. Right-clicking the trend and selecting Substitute > Substitute References opens a dialog where you can find and replace all occurrences of Me.PV with Me.Level.PV in a single operation. This bulk replacement updates the data source, alarm references, and engineering unit references across thirteen tag bindings simultaneously — a significant time-saver compared to editing each one individually.

Substitute References — 13 references updated from Me.PV to Me.Level.PV in a single operation.
Warning: The bulk replacement does not catch everything. The generic SA_Trend_SinglePen uses EngUnitsRangeMax and EngUnitsRangeMin as property names, but the actual Level attribute properties are named EngUnitsMax and EngUnitsMin. Because the find-and-replace only changes the prefix (Me.PV → Me.Level.PV), these two suffixes must be corrected manually after the bulk operation — otherwise the engineering unit range will not resolve correctly at runtime.

Next, the Substitute Strings tool is used to change the text label from "Label" to "Level" throughout the symbol. Finally, on the Properties tab, the custom properties EnableFill is set to True and FillColor is set to Gray. This fills the area under the trend line with a shaded region, which in this course visually represents the volumetric flow of liquid measured in liters per second through the mixing process.

Custom properties — EnableFill=True and FillColor=Gray add a shaded area under the level trend line.
Tip: The FillColor property only recognizes eight named colors: Red, Green, Blue, Yellow, Orange, Brown, White, and Gray. If you enter any other value, the fill silently falls back to no fill — no error is shown, but the shaded area disappears.

31.3 Creating the Temperature Trend by Duplication

Rather than building a second trend from scratch, you duplicate the LevelTrend_Detailed symbol and rename the copy TemperatureTrend_Detailed. This approach ensures consistency and saves time. Once the duplicate is opened for editing, three property changes are made: the Name becomes TemperatureTrend, the MeterType changes to Temperature, and EnableFill is set to False — because the Temperature trend uses a line-only display for visual clarity.

After the property changes, Substitute Strings replaces the word "Level" with "Temperature" throughout the symbol, and Substitute References redirects all Me.Level.PV references to Me.Temperature.PV. This completes the repurposing of the duplicated symbol.

Substitute Strings — changing the Level label to Temperature in the duplicated trend symbol.

31.4 Embedding Trends into the Mixer Process Graphic

With both trend symbols created, the next step is to embed them inside the MixingProcess_Detailed graphic. You open that graphic for editing and delete the original Level and Temperature meter objects — they are now superseded by the new trend displays. Using Relative References, you embed Me.TemperatureTrend_Detailed in the bottom half of the mixer tank, and Me.LevelTrend_Detailed in the top half. The agitator graphic is moved to the right of the tank to make room, and the tank itself is resized to accommodate both trend widgets.

Both trend charts embedded in the mixer vessel — Level on top, Temperature on bottom, with equipment symbols on either side.
Key Concept: Relative References for Embedding
When you embed a symbol using Relative References (e.g., Me.LevelTrend_Detailed), InTouch automatically resolves the references to the correct instance at runtime. This is what allows the same $Mixer template to display different trend data for each mixer instance — Mixer100 shows Mixer100's data, Mixer200 shows Mixer200's data. Without Relative References, you would need to manually reconfigure every reference for each mixer, which is impractical in any real deployment.

31.5 Testing in Runtime

Switching to Runtime reveals the payoff of all this configuration work. Inside the ContentDisplay window, the Level and Temperature trends update in real time, scrolling smoothly as new data arrives. Hovering the mouse over a trend reveals the floating tooltip, which displays the current PV, maximum, minimum, and average values — all computed over the visible time window. The vertical meter on the right shows alarm limit indicators for HiHi, Hi, Lo, and LoLo, and the alarm border activates whenever the process value enters an alarm state, giving operators an immediate visual cue that something requires attention.

Runtime — Level (99.49) and Temperature (119.29°C) trends update live inside the mixer vessel display.

Quiz — Test Your Understanding

Q1: Why do you need to manually change EngUnitsRangeMax to EngUnitsMax after using Substitute References?

The generic SA_Trend_SinglePen uses EngUnitsRangeMax/Min as property names, but the actual object attribute properties are named EngUnitsMax/Min. The bulk find-and-replace of Me.PV to Me.Level.PV doesn't change the property suffix, so these two names must be corrected manually.

Q2: What does the EnableFill property do, and why is it set to False for the Temperature trend?

EnableFill adds a shaded area under the trend line, filled with the color specified in FillColor. It is set to True for Level (to visually represent volumetric flow) and False for Temperature (which uses a line-only display for clarity).

Q3: What is the purpose of using Relative References when embedding trends into the mixer graphic?

Relative References (e.g., Me.LevelTrend_Detailed) allow InTouch to automatically resolve references to the correct instance at runtime. This means the same $Mixer template can display different data for each mixer instance (Mixer100, Mixer200, etc.) without manual reconfiguration.

Q4: What information does the floating tooltip display when you hover over a trend at runtime?

The floating tooltip displays the current PV (process value), Max (maximum in the window), Min (minimum in the window), and Avg (average, computed as Max minus Min). The meter beside the trend also shows alarm limit indicators for HiHi, Hi, Lo, and LoLo.
