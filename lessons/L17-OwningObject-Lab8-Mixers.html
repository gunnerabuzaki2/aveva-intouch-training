Lesson 17: The OwningObject Property + Lab 8

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 3, Section 5 + Lab 8 (pages 205–213)

This lesson introduces the OwningObject property for dynamically switching object references at runtime, covers Windows common controls, and walks through Lab 8 — building a mixer selector using a Combo Box.

17.1 The OwningObject Property

Reference: Module 3, Section 5 (page 203)

The OwningObject is a runtime property that allows you to dynamically change which object a graphic references — without redesigning the graphic. When you set the OwningObject property on an embedded graphic, it replaces all Me. references in that graphic's expressions and scripts with the specified object. This means a single graphic can display data from different instances at runtime simply by changing the OwningObject value.

Key Concept: The OwningObject property replaces all "Me." references in a graphic with a specified object instance, enabling one graphic to display data from multiple objects at runtime by simply changing the OwningObject value.

The OwningObject value can be set using the tagname or hierarchical name of an automation object. For example, if a graphic contains the expression Me.PV and you set OwningObject to Mixer500, the expression resolves to Mixer500.PV at runtime.

You can link the OwningObject property to a Combo Box animation, allowing operators to select which object instance the graphic should display from a dropdown list.

17.2 Windows Common Controls

Reference: Module 3, Section 5 (page 203)

In addition to standard drawing elements, the Graphic Editor provides several Windows common controls that you can add to your graphics:

ControlDescription
Radio Button GroupAllows selection of one option from a group
Check BoxToggles a Boolean value on or off
Edit BoxAllows users to type text input
Combo BoxProvides a dropdown list for selection
Calendar ControlDisplays a calendar for date selection
Date-Time PickerProvides date and time selection
List BoxDisplays a scrollable list of items

Place these controls by selecting them from the Tools pane and clicking (or dragging) on the canvas. After placement, you can configure background and text colors (except DateTime Picker), control-specific properties, and control-specific animations.

17.3 Lab 8 Introduction — Switching Between Mixers

Reference: Module 3, Lab 8 (pages 205–213)

Lab 8 Objectives: Upon completion of this lab, you will be able to:
• Use the OwningObject property to dynamically change instance references of a graphic
• Add and configure a Combo Box

In this lab, you will add a Combo Box to the Mixer display that lets operators select which of the eight mixers (Mixer100 through Mixer800) they want to view at runtime. The Combo Box is linked to the OwningObject property of the MixingProcess graphic, so selecting a different mixer from the dropdown causes the graphic to display data from that mixer instance.

17.5 Adding and Configuring the Combo Box

Reference: Module 3, Lab 8 (pages 208–211)

Select the Combo Box tool from the Tools pane, then click and drag below the Select Mixer text to draw the Combo Box on the canvas. Name it MixerDropDown on the Properties tab.

The Combo Box tool in the Tools pane (page 208)
The Combo Box drawn on the canvas below the Select Mixer label (page 208)
Setting the Combo Box name to MixerDropDown (page 208)

Double-click the Combo Box to open Edit Animations. In the Reference field, enter MixingProcess.OwningObject. This links the Combo Box selection to the OwningObject property of the MixingProcess graphic — when the operator selects a mixer from the dropdown, the OwningObject updates and the graphic switches to display that mixer's data.

Linking the Combo Box to MixingProcess.OwningObject (page 209)
Key Concept: The Reference field MixingProcess.OwningObject connects the Combo Box selection to the OwningObject property. When a user selects a mixer name, the OwningObject changes, causing all "Me." references in MixingProcess to resolve to the selected mixer instance.

In the Static Values and Captions table, click the Add a row button to create eight rows (one for each mixer). Check Use Values as Captions so the caption automatically mirrors the value text. Uncheck AllowDuplicates and set the Type to DropDown.

Reference: Module 3, Lab 8 (pages 212–213)

Switch to WindowMaker and replace the existing symbol in the ContentDisplay window with the new SwitchedMixingProcess. Right-click the ContentDisplay window, select Embed Industrial Graphic, navigate to Mixer100 in Instances, select SwitchedMixingProcess, click OK, and confirm the replacement.

Tip: The OwningObject property is a powerful pattern for creating reusable graphics. Instead of creating separate graphics for each mixer, you build one graphic and let the OwningObject property switch which data it displays at runtime.

Knowledge Check

1. What does the OwningObject property do when set on an embedded graphic?

Click to reveal answer

It replaces all "Me." references in the graphic's expressions and scripts with the specified object instance. For example, setting OwningObject to Mixer500 causes Me.PV to resolve to Mixer500.PV at runtime.

2. What type of animation is used to configure a Combo Box control, and what makes it unique?

Click to reveal answer

A Combo Box animation is used, and it is unique to the Combo Box element type — no other element type supports this animation. It defines the dropdown list items through Static Values and Captions.

3. In the Combo Box configuration, what does the "Use Values as Captions" checkbox do?

Click to reveal answer

When checked, the Caption column automatically mirrors whatever is entered in the Value column. This saves time because you only need to type the value once instead of entering both the display text and the underlying value separately.

4. Why do you create a SwitchedMixingProcess symbol instead of modifying MixingProcess_Detailed directly?

Click to reveal answer

The SwitchedMixingProcess symbol wraps MixingProcess_Detailed and adds the Combo Box for mixer selection. Keeping MixingProcess_Detailed unchanged preserves it as a reusable component, while SwitchedMixingProcess adds the runtime switching capability as a separate layer.

5. What are the three types of Combo Box behavior available in AVEVA InTouch?

Click to reveal answer

Simple (type or use arrow buttons), DropDown (type or select from dropdown list), and DropDownList (only select from predefined list — no typing allowed).
