Module 3 — Symbols | Pages 173–180

15.1 Connectors and Connection Points

When building complex graphics like piping diagrams, flow charts, or wiring schematics, you need elements that stay connected even when you move or resize them. That's exactly what connectors do. A connector is a line drawn between two graphic elements that starts at a connection point on one element and ends at a connection point on another. Unlike a plain line, a connector automatically adjusts its length and orientation when the connected elements are moved, resized, or rotated.

Connection points are the anchor locations on a graphic element where connectors can attach. Every element has a default set of connection points on its bounding rectangle. You can also add custom connection points using the Connection Point tool — just click on the element where you want a new point to appear (shown as a small green rectangle).

Connectors link graphic elements at connection points. When you move either element, the connector automatically adjusts to maintain the link.
Connectors linking industrial symbols together — the line automatically adjusts when either symbol is moved or resized.

To draw a connector, select the Connector tool, click a connection point on the first element, drag to a connection point on the second element, and release. If you need to draw multiple connectors, double-click the Connector tool to keep it active — click another tool or press Esc to deactivate it.

Key Concept: Connectors vs. Static Lines
A static line stays in one place even when you move the elements it appears to connect. A connector is anchored to connection points and automatically repositions when elements move. Always use connectors for flow diagrams, piping, and any graphic where elements might be repositioned during design or runtime.

15.2 Drawing Tools Overview

The Graphic Editor provides a comprehensive set of drawing tools in the Tools pane. These include:

The Graphic Editor Tools pane provides a full set of drawing tools for creating custom graphics from basic shapes.
  • Basic shapes — lines, rectangles, rounded rectangles, ellipses, and polygons for building custom graphics from scratch.
  • Complex shapes — polylines, curves, arcs (2-point and 3-point), pies, and chords for more detailed drawings.
  • Text tools — plain Text elements (no border, no wrap) and Text Box elements (with border, background, and text wrapping).
  • Pointer tool — for selecting and moving elements on the canvas.
  • Windows common controls — combo boxes, radio button groups, calendar controls, and other interactive UI elements.
  • Status tool — for showing quality and status of selected attributes at runtime.

You can double-click any drawing tool to keep it active for placing multiple elements of the same type, which is a significant time saver when building complex graphics.

15.3 Visualization Animations

Animations are what transform static graphics into dynamic, data-driven displays. You configure animations through the Edit Animations dialog, which you access by double-clicking an element or selecting it and choosing Edit Animations from the context menu. There are two broad categories: visualization animations and interaction animations.

Visualization animations change an element's appearance based on data:

  • Visibility — show or hide an element based on a Boolean expression.
  • Fill Style — change the fill color or pattern based on a condition.
  • Line Style — change line color, style, or width.
  • Text Style — change font, color, or size of text elements.
  • Blink — make an element flash to draw attention.
  • Alarm Border — add a colored border that reflects the alarm state of a linked attribute.
  • Value Display — display real-time data values in a text element.
The Edit Animations dialog provides access to all visualization and interaction animations available for the selected element type.

15.4 Value Display Animations

The Value Display animation is one of the most commonly used — it lets a text element display real-time data from your automation objects. You can configure it to show:

  • A Boolean value as a message (e.g., "Running" / "Stopped")
  • An Analog value with configurable decimal places
  • A String value
  • A Time or Date value
  • The tag name, hierarchical name, or contained name of the hosting object
  • A value based on Truth Table conditions (multiple expressions evaluated in order)
  • A value based on Bit State conditions (individual bits of an integer value)
Key Concept: Truth Table vs. Bit State Value Display
Use Truth Table when you need to evaluate multiple independent expressions and display different text for each. Use Bit State when a single integer value encodes multiple status bits (common with equipment status registers where different bits represent different states).

15.5 Interaction Animations

Interaction animations let operators interact with the display at runtime:

  • Pushbutton — makes an element clickable to trigger a command or script.
  • Show Symbol — opens a popup symbol when an element is clicked.
  • Slider Horizontal/Vertical — creates a slider control for adjusting setpoints.
  • Action Scripts — runs custom script code when an element is clicked.
  • Disable — disables an element (and its animations) based on a condition, without losing its configuration.

You can have both visualization and interaction animations on the same element. For example, a button might have a Fill Style animation (to change color based on state) and a Pushbutton animation (to trigger an action when clicked). You can also cut, copy, and paste animations between elements — only compatible animation types will be pasted.

15.6 Element Style Animations

Element Style animations apply predefined visual styles to elements based on conditions. Instead of manually configuring colors, fonts, and borders for every state, you define an Element Style once and apply it through an animation. Element Style animations support three evaluation modes:

  • Boolean — applies one style when True, another when False.
  • Truth Table — applies different styles based on which expression evaluates to True.
  • Bit State — applies styles based on individual bit states of an integer value.
An Element Style animation applies predefined visual styles based on a Boolean, Truth Table, or Bit State expression — ensuring consistent appearance across your application.

15.7 Containment Relationships and Lab Introduction

Automation objects can be organized in containment relationships, where one object is nested inside another. When working with contained objects, symbols can use relative references (like Me.PV) that automatically resolve through the containment hierarchy. This means a symbol defined in a contained object (like a valve) can be embedded into a symbol of the container object (like a mixer), and all the Me. references will resolve correctly at runtime.

Tip: Template-Level Containment Requires Explicit Embedding
At the template level, if the container template needs symbols from its contained templates, you must explicitly add a symbol to the container template and embed the contained symbols using relative references. The containment resolution only works at runtime, not at design time in the IDE.

This concept is the foundation for Lab 7, where you'll create a MixingProcess_Detailed symbol in the $Mixer template that combines the agitator, level, pump, temperature, and valve graphics you created in the previous lab — all organized to represent the complete mixing process for a single mixer instance.

The end goal: a complete mixer process mimic showing all equipment components working together in a single composite display.

Quiz — Test Your Understanding

Q1: What is the key difference between a connector and a static line?

A static line is just a fixed graphic element that doesn't respond to changes in surrounding elements. A connector is anchored to connection points on two elements and automatically adjusts its length, position, and orientation when either element is moved, resized, or rotated. This makes connectors essential for flow diagrams and piping layouts.

Q2: Name three types of visualization animations and explain what each does.

Visibility shows or hides an element based on a Boolean condition. Fill Style changes an element's fill color or pattern based on data. Value Display makes a text element show real-time data values from automation objects. Other examples include Alarm Border (colored border reflecting alarm state), Blink (flashing for attention), and Text Style (changing font properties).

Q3: When would you use a Truth Table Value Display instead of a simple Analog display?

Use a Truth Table when you need to display different text based on multiple independent conditions — for example, showing "Low" when a value is below 25, "Normal" when between 25 and 75, and "High" when above 75. A simple Analog display just shows the raw number; a Truth Table lets you map ranges or conditions to meaningful text labels.

Q4: What are the two categories of animations, and how do they differ?

Visualization animations change an element's appearance (color, visibility, text content) based on data — they're passive displays. Interaction animations let operators take action (clicking a button, moving a slider, triggering a script) — they're active controls. Many elements can have both types simultaneously.

Q5: Why are relative Me. references important for containment relationships?

Relative Me. references allow a symbol defined in one object to be embedded in a container object's symbol, and the references automatically resolve to the correct instance at runtime. Without relative references, you'd need to hardcode specific tag names for each equipment instance, making the symbol non-reusable across different mixers.
