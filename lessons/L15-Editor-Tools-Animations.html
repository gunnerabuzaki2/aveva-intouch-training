Lesson 15: Graphic Editor Tools and Animations

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 3, Section 4 (pages 173–180)

This lesson covers connectors, connection points, animation types, value displays, element styles, containment relationships, and introduces Lab 7.

15.1 Connectors and Connection Points

Reference: Module 3, Section 4 (pages 173–175)

When building complex process graphics — such as flow diagrams, piping layouts, or electrical wiring schematics — you often need lines that link one graphic element to another. In the Graphic Editor, these linking lines are called connectors, and they attach to elements through connection points.

A connector is a special line drawn between two graphic elements. It starts at a connection point on one element and ends at a connection point on another. The key advantage of connectors over regular lines is that they are smart: when you move, resize, or rotate the connected elements, the connector automatically adjusts its length and orientation to stay linked. This makes connectors ideal for dynamic layouts where elements may shift position during editing or at runtime.

Key Concept: Connectors automatically adjust when connected elements are moved, resized, or rotated — unlike static lines which must be manually repositioned.

Connection points are the specific locations on a graphic element where connectors can attach. By default, each element has connection points placed around its bounding rectangle. However, you can add custom connection points anywhere within the bounding rectangle to give yourself more flexibility in how connectors attach.

15.2 Drawing Connectors and Adding Connection Points

Reference: Module 3, Section 4 (pages 174–175)

To draw a connector between two elements, select the Connector tool from the Tools pane, then click a connection point on the first element, hold the mouse button and drag to the connection point on the second element, then release. A line appears connecting the two elements.

Tip: Double-click the Connector tool to lock it for drawing multiple connectors in succession. This avoids having to reselect the tool after each connector. Click another tool or press Esc to cancel the locked mode.

To add a custom connection point, select the Connection Point tool and click anywhere within the bounding rectangle of a graphic element. A small green rectangle appears at the clicked location, marking the new connection point. You can also double-click the Connection Point tool to lock it for adding multiple points.

15.3 Animations Overview

Reference: Module 3, Section 4 (pages 176–177)

Animations are what bring your static graphics to life at runtime. They allow graphic elements to change appearance based on data, respond to user input, and display dynamic values. You configure animations through the Edit Animations dialog box, which you access by double-clicking an element on the canvas.

AVEVA InTouch supports two broad categories of animations:

CategoryAnimation TypesPurpose
VisualizationVisibility, Fill Style, Line Style, Text Style, Element Style, Blink, Alarm Border, Value Display, % Fill, Location, Orientation, TooltipChange how elements look based on data conditions
InteractionDisable, User Input, Slider, Pushbutton, Action Scripts, Show/Hide Symbol, HyperlinkLet users interact with the graphic at runtime

A single element can have multiple animations applied to it. You can enable or disable individual animations without losing their configuration. Animations can also be cut, copied, and pasted between elements — though only animations that are valid for the target element type will be pasted.

Common Mistake: Not all animations work with all element types. For example, applying a Line Style animation to a Text element has no effect and is not a valid combination. The system prevents you from selecting or copying invalid animation combinations.
The Edit Animations dialog showing all available animation types organized by category (page 176)

15.4 Value Display and Element Style Animations

Reference: Module 3, Section 4 (pages 177–178)

The Value Display animation is one of the most commonly used visualization animations. Applied to text elements, it allows the text to show dynamic values at runtime. You can configure it to display several types of data:

  • Boolean — a True/False value shown as a message label
  • Analog — a numeric value from a tag or expression
  • String — a text string value
  • Time — a time or date value
  • Name — the tag name, hierarchical name, or contained name of the hosting object
  • Truth Table — a value selected based on evaluated conditions
  • Bit State — a value selected based on bit conditions

The Element Style animation lets you apply predefined visual styles (colors, fonts, borders) to elements based on conditions. It supports Boolean, Truth Table, and Bit State evaluation modes. When a condition is true, the corresponding style is applied; when false, a different style (or no style) is used.

Key Concept: The Value Display animation transforms static text into dynamic data readouts, while the Element Style animation transforms element appearance based on process conditions — together they form the backbone of interactive HMI graphics.

You can also disable individual animations for an element without deleting them. Disabled animations retain their configuration and can be re-enabled later. The Animation Summary pane in the bottom-right corner of the Graphic Editor provides a quick way to toggle animations on and off.

15.5 Containment Relationships and Element Styles

Reference: Module 3, Section 4 (pages 178–179)

In AVEVA System Platform, automation objects can exist within a containment relationship — where one object (the container) holds references to other objects (the contained). When symbols within these objects use relative references like Me.PV, those references resolve automatically when the symbols are embedded in the container's graphics. This means you don't need to manually rewrite references when embedding a contained object's graphic into the container.

Key Concept: Relative references (such as Me.PV) in contained object symbols automatically resolve when embedded in the container's graphics, eliminating the need for manual reference rewriting.

Element Styles define a set of visual properties — text formatting, line colors, fill colors, and outline styles — that can be applied consistently across multiple elements in your graphics. When an Element Style is applied to a graphic, its preconfigured visual property values override the graphic's native visual properties. This is particularly useful for maintaining consistent visual standards across a large application. You access Element Styles through the ElementStyle property on the Properties tab.

Knowledge Check

1. What is the key advantage of using connectors instead of regular lines when linking graphic elements?

Click to reveal answer

Connectors automatically adjust their length and orientation when the connected elements are moved, resized, or rotated, whereas regular static lines require manual repositioning.

2. What are the two main categories of animations in the Graphic Editor, and what is one example from each category?

Click to reveal answer

The two categories are Visualization animations (e.g., Value Display, Element Style, Visibility) which change how elements look based on data, and Interaction animations (e.g., Pushbutton, Show Symbol, User Input) which let users interact with the graphic at runtime.

3. When you embed a symbol from a contained object that uses relative references like Me.PV, what happens to those references?

Click to reveal answer

Relative references automatically resolve when the symbol is embedded in the container's graphics — you do not need to manually rewrite them.

4. How can you lock a tool (such as Connector or Connection Point) for repeated use without reselecting it each time?

Click to reveal answer

Double-click the tool in the Tools pane to lock it. This keeps the tool active for drawing multiple elements of the same type. Click another tool or press Esc to cancel the locked mode.

5. What determines which animation types are available when you open the Edit Animations dialog for a specific element?

Click to reveal answer

The element type determines which animations are valid. For example, Line Style is not valid for a Text element, and only animations applicable to the selected element type will appear in the dialog.
