Lesson 19: Lab 9 – Creating a Custom Symbol

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 3 — Symbols (pages 223–248)

This lab walks you through building a reusable CommandPanel symbol with Open/Close buttons, quality-status indicators, and a speed setpoint entry — then wiring it to agitators, pumps, and valves so operators can control equipment from runtime popup windows.

19.1 Defining Custom Properties for the Command Panel

Reference: Module 3 (pages 223–224)

In the previous lesson you learned how to create and use custom properties on symbols. In this lab, you will put that knowledge into practice by building a complete CommandPanel symbol. The symbol needs three custom properties that will drive its buttons and captions at runtime: a Boolean command tag and two String properties for the Open and Close button labels.

Lab Steps: Open the CommandPanel symbol for editing. On the Custom Properties tab, add the following properties:
Property NameData TypeDefault ValueDescription
CmdBooleanMe.CmdCommand for button
CmdOffMsgStringMe.Cmd.OffMsgButton caption
CmdOnMsgStringMe.Cmd.OnMsgButton caption

When you switch the String properties to Expression or Reference mode, the Default Value field is treated as a tag reference rather than literal text. This lets each instance of the symbol inherit the correct caption from its owning object.

Key Concept: Custom properties with Expression or Reference defaults (like Me.Cmd) resolve relative to the object that owns the symbol. This is what makes the CommandPanel truly reusable — each instance automatically references the correct tag.

19.2 Adding the Open Button with Animations

Reference: Module 3 (pages 225–227)

With the custom properties in place, the next step is to draw the Open button on the canvas and wire it up with three types of animations: a Pushbutton interaction that sets the command, a Value Display that shows the on-caption, and an Element Style animation that highlights the button when the command is active.

Lab Steps:
  1. From the Tools palette, select the Button tool and draw a button on the canvas.
  2. Set the button text to Open. Configure its properties: ElementStyle = Intensity2, Width = 100, Height = 40.
  3. Double-click the button to open Edit Animations and add the three animations described below.
Animation TypeStatesReferenceConfiguration
PushbuttonBooleanCmdAction = Set
Value DisplayStringCmdOnMsgShows the Open caption text
Element StyleBooleanCmdTrue → Intensity4, False → default (unchecked)

When the operator presses the button, the Pushbutton animation writes 1 to the Cmd tag. The Element Style animation makes the button appear brighter (Intensity4) while the command is active, giving the operator clear visual feedback.

Tip: For the Element Style animation's False/Off state, leave the checkbox unchecked. This tells the button to revert to its default ElementStyle property (Intensity2) rather than applying a different style.

19.3 Creating the Close Button by Duplication

Reference: Module 3 (pages 228–230)

Rather than building the Close button from scratch, you can save time by duplicating the Open button and modifying its animations. The key trick is using the not operator on the reference to invert the Element Style animation without having to reconfigure it from scratch.

Lab Steps:
  1. Right-click the Open button and select Duplicate. Place the copy to the right.
  2. Change the duplicate's Text property to Close.
  3. Double-click the Close button and modify the animations:
    • Element Style: Change the reference to not Cmd — this inverts the highlighting so the Close button lights up when the command is off.
    • Value Display: Change the reference to CmdOffMsg.
    • Pushbutton: Change the Action from Set to Reset.
Key Concept: Using not Cmd as a reference is a powerful time-saver. Instead of manually swapping the True/False style values, the not operator inverts the logic, so the same Intensity4-on-True configuration now highlights the button when Cmd is False.

19.4 Adding a Quality & Status Indicator

Reference: Module 3 (pages 230–231)

A quality and status indicator between the two buttons gives operators feedback about the communication state of the underlying tag. The Status tool creates a small element that monitors the selected graphic elements and displays quality icons at runtime.

Lab Steps:
  1. From the Tools palette, select the Status tool.
  2. Draw a small rectangle between the Open and Close buttons on the canvas. The Edit Animations dialog opens automatically.
  3. On the Graphics tab, move both Button1 and Button2 to the Selected Graphic Elements list using the >> button.
  4. Click OK to close.

At runtime, this indicator will briefly show an hourglass icon whenever the quality state transitions between commands — a useful visual cue that the system is processing the operator's request.

19.5 Fixing the Canvas Size and Checking In

Reference: Module 3 (page 232)

Before checking in the symbol, you should set the canvas to a fixed size so the CommandPanel always appears at consistent dimensions when embedded in other graphics.

Lab Steps: Click the canvas (to deselect all objects), then set: Size = Fixed, FixedWidth = 320, FixedHeight = 230. Save, close, and check in the graphic.

19.6 Creating the Speed Setpoint Panel

Reference: Module 3 (pages 233–235)

The agitator needs a speed setpoint entry in addition to the Open/Close buttons. Rather than modifying the base CommandPanel, you will duplicate it and add a numerical entry widget from the Situation Awareness library.

Lab Steps:
  1. Duplicate the CommandPanel symbol and name the copy CommandPanelwithSetpoint. Open it for editing.
  2. From the Toolbox tab, search for "numerical" and drag SA_NumericalEntry onto the canvas below the buttons.
  3. Configure the wizard options: LabelType = CustomPropertyLabel, EngUnitsType = CustomPropertyLabel, Outline = False.
  4. Right-click the numerical entry and select Substitute > Substitute References. Use Find & Replace to change all Me.PV references to Me.Speed.SP.
  5. Open Custom Properties for the numerical entry and set the Label property's Default Value to Me.Speed.SP.Description.
  6. Save, close, and check in.

19.7 Wiring the Show Symbol Animation

Reference: Module 3 (pages 236–240)

Now comes the critical step: connecting the command panels to the actual equipment graphics so that clicking an agitator, pump, or valve opens the correct popup. This is done with the Show Symbol interaction animation, which opens a symbol as a popup window at runtime.

Lab Steps:
  1. Open Agitator_Detailed for editing. Select the Agitator graphic and set TreatAsIcon = True on the Properties tab. This enables interaction animations on the embedded graphic.
  2. Double-click the Agitator graphic and add a Show Symbol animation.
  3. For the Reference, click Browse Galaxy and select CommandPanelwithSetpoint from the Training folder.
  4. Configure the animation:
    • Title: Uncheck "Use Symbol Name" and switch to Expression mode. Enter OwningObject.
    • Position: Below / Parent Symbol.
  5. Copy the animation from the Agitator graphic (right-click > Animations > Copy).
  6. Open Pump_Detailed. Set TreatAsIcon = True on the Pump graphic. Paste the animation. Open it and change the Reference to CommandPanel (without the setpoint).
  7. Copy the animation from the Pump graphic.
  8. Open Valve_Detailed. Set TreatAsIcon = True. Paste the animation. Save and check in.
Key Concept: Setting TreatAsIcon = True is essential — without it, the Show Symbol animation cannot be attached to an embedded graphic. The title uses OwningObject so the popup window displays the equipment name (e.g., "Inlet1_001") rather than the symbol name.

19.8 Testing the Command Panels in Runtime

Reference: Module 3 (pages 241–245)

With all the wiring complete, it is time to switch to WindowViewer and test each command panel. You will click on a valve, pump, and agitator to verify that the correct popup appears with the right buttons and captions.

Lab Steps:
  1. In WindowMaker, click Runtime to switch to WindowViewer.
  2. Click Inlet1_001 (a valve) to open its command panel. The title shows "Inlet1_001" and the buttons display "Open" and "Close."
  3. Click the lighter button (Intensity2 — the active command) to toggle the valve. Watch the Quality & Status hourglass appear briefly between commands.
  4. Click the X at the top of the command panel to close it. You must close the popup before performing other actions in WindowViewer.
  5. Click Pump1_001. Its command panel shows "Start" and "Stop" buttons.
  6. Click Agitator_001. Its command panel includes Start/Stop buttons plus the Speed setpoint entry.
  7. Enter 45 in the Speed setpoint field and press Enter. The agitator should begin running at 45 RPM.
  8. Verify the running speed on the main process display, then click Development! to return to WindowMaker.
Tip: To keep the simulator working correctly, make sure the pump and its associated valve are in the same state. For example, if Inlet 1 is Open, Pump 1 should also be Running. When both match, the tank level will change.

Knowledge Check

1. Why must you set TreatAsIcon to True before adding a Show Symbol animation to an embedded graphic?

Click to reveal answer

TreatAsIcon = True enables interaction animations on an embedded graphic. Without it, the Show Symbol animation will not function at runtime even if it appears to be configured correctly.

2. What happens when you use not Cmd as the Element Style reference on the Close button?

Click to reveal answer

The not operator inverts the Boolean logic, so the Intensity4 style is applied when Cmd is False (Off) rather than True. This makes the Close button highlight when the command is not active, without needing to manually reconfigure the True/False style values.

3. What is the purpose of the Substitute References feature when configuring the SA_NumericalEntry widget?

Click to reveal answer

Substitute References performs a bulk find-and-replace on all tag references inside a symbol. In this lab, it was used to change all Me.PV references to Me.Speed.SP in a single operation, saving the effort of manually editing each animation binding.

4. Why does the command panel title use the OwningObject expression instead of the symbol name?

Click to reveal answer

Using OwningObject as the title expression makes the popup window display the equipment name (e.g., "Inlet1_001", "Pump1_001") rather than the generic symbol name "CommandPanel." This gives operators clear context about which piece of equipment they are commanding.

5. What is the significance of closing the command panel popup by clicking the X before performing other actions in WindowViewer?

Click to reveal answer

The command panel opens as a modal popup window. If you do not close it by clicking the X, you will not be able to interact with any other elements in WindowViewer — the modal window blocks all other input.
