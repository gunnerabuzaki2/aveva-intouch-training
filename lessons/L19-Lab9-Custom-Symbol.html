Lesson 19: Lab 9 — Creating a Custom Symbol

Module 3 — Symbols | Pages 223–248

In the previous lesson, you learned the theory behind custom properties, buttons, and animations. Now it is time to put those concepts into practice by building something real. In this lab, you will construct command panel symbols that operators can use to open and close valves, start and stop pumps and agitators, and change the agitator speed setpoint. Along the way, you will apply custom properties for configuration, Pushbutton animations for equipment control, Show Symbol animations for popup panels, and the TreatAsIcon property to make embedded graphics interactive. By the end, you will have a complete set of reusable command panels that adapt automatically to whichever equipment they control.

Creating the CommandPanel Symbol

Lab Objective: Build a reusable CommandPanel symbol that provides Open/Close or Start/Stop buttons for valves and pumps. The panel will use custom properties to receive configuration data from the hosting equipment graphic, ensuring that the same panel works for any equipment type.

Begin by creating a new symbol called CommandPanel in the Training toolset on the Graphics tab. Open it for editing. This symbol will serve as the basic command panel for valves and pumps — it provides two buttons without any additional controls.

The first step is to define the custom properties that will make this symbol configurable. Right-click the canvas and select Custom Properties (or press Ctrl+M) to open the Edit Custom Properties dialog. You will add three properties, each serving a distinct purpose in the panel's operation.

Defining the Custom Properties

The three properties you need are: a Boolean Cmd property that holds the command state, a String CmdOffMsg property for the "off" button caption, and a String CmdOnMsg property for the "on" button caption. The key to making this panel reusable lies in how you set their default values.

For the Cmd property, set the Data Type to Boolean, the Default Value to Me.Cmd (using Expression or Reference mode), the Visibility to Public, and the Description to "Command for button." The Me.Cmd reference is critical: it creates a forwarding chain so that when the CommandPanel is embedded inside a valve symbol, the property automatically resolves to that valve's Cmd attribute. When the same panel is embedded in a pump, it resolves to the pump's Cmd attribute instead.

The CmdOffMsg property follows a similar pattern. Set it to String type with the default value Me.Cmd.OffMsg in Expression or Reference mode. This pulls the off-message label (such as "Close" or "Stop") from whatever equipment hosts the panel. The CmdOnMsg property uses Me.Cmd.OnMsg for the same purpose, pulling the on-message label ("Open" or "Start").

Property Forwarding with Me. References — Setting a custom property's default value to Me.Cmd creates a forwarding chain: when the CommandPanel is embedded inside a valve or pump symbol, the custom property automatically resolves to that equipment's Cmd attribute. This is what makes the CommandPanel truly reusable — the same panel works for any equipment type without modification.
Edit Custom Properties dialog with the + Add custom property button and an empty property grid, ready for creating the first custom property.
Edit Custom Properties showing all three properties configured: Cmd (Boolean), CmdOffMsg (String), and CmdOnMsg (String).

Adding Buttons with Animations

With the custom properties defined, the next step is to add the interactive buttons that will control the equipment. Use the Button tool from the Tools pane to draw a button on the canvas. Type "Open" as the label and press Enter. Then configure its properties through the Properties tab: set ElementStyle to Intensity2, Width to 100, and Height to 40.

Double-click the Open button to open Edit Animations, where you will layer three separate animations that together create a complete, responsive button:

First, add a Pushbutton animation with the Boolean Reference set to Cmd and the Action set to Set. When the operator presses this button at runtime, it writes True to the command attribute, signalling an "open" or "start" command to the equipment.

Second, add a Value Display animation using the String state, with the Expression Or Reference set to CmdOnMsg. This dynamically displays the "on" caption pulled from the custom property — so the button label adapts automatically depending on whether it controls a valve ("Open") or a pump ("Start").

Third, add an Element Style animation with the Boolean Reference set to Cmd. For the True/On state, enable Element Style and select Intensity4 — a brighter style indicating the active state. For the False/Off state, leave Element Style unchecked so the button reverts to its default Intensity2 appearance. This gives the operator immediate visual feedback about the equipment's current state.

Tip: The Element Style animation provides visual feedback that operators rely on. When the command is active (True), the button glows brighter (Intensity4). When inactive (False), it dims to its default style (Intensity2). This instant visual cue tells the operator which state the equipment is in without requiring a separate status indicator.

Now duplicate the Open button (right-click > Duplicate) and place the copy to its right. Change its Text property to "Close" and adjust its three animations: change the Pushbutton action to Reset (which writes False to the command attribute), update the Value Display expression to CmdOffMsg, and invert the Element Style expression to not Cmd so the Close button appears bright when the command is Off and dim when On.

Value Display animation for the Close button showing the String state with Expression or Reference set to CmdOffMsg for the button caption.

Between the two buttons, add a Status element to display quality and status information. Click the Status tool, draw a small rectangle between the Open and Close buttons, and in the Edit Animations dialog, move both Button1 and Button2 from the Available list to the Selected Graphic Elements list. This links the status indicator to both buttons so it shows the data quality of the command attribute — a brief hourglass icon appears while the write is in progress.

Finally, set the canvas Size to Fixed with FixedWidth: 320 and FixedHeight: 230. Save, close, and check in the CommandPanel graphic.

Canvas resized to its fixed dimensions (320×230) showing the Open button, Close button, and quality status indicator.

Creating the Speed Setpoint Panel

The agitator requires more than just Start/Stop buttons — it also needs a speed setpoint input so operators can adjust the agitator speed at runtime. Rather than building a new symbol from scratch, duplicate the CommandPanel symbol and name the copy CommandPanelwithSetpoint. Open it for editing and add the speed input field.

Search the Toolbox for numerical and drag SA_NumericalEntry onto the canvas below the existing buttons. Configure its wizard options: set LabelType and EngUnitsType to CustomPropertyLabel, and disable the Outline. The numerical entry widget is a pre-built symbol that includes its own custom properties — you are now composing symbols inside symbols, building up complexity in a modular way.

The numerical entry widget is configured by default to reference a process variable (Me.PV), but you need it to control the speed setpoint instead. Use Substitute References (right-click > Substitute > Substitute References) to find all instances of Me.PV and replace them with Me.Speed.SP. This bulk operation updates every internal reference — the PV value, engineering units, range maximum, and range minimum — in a single step, avoiding the tedium of editing each one individually.

Substitute Reference dialog showing Find and Replace of Me.PV with Me.Speed.SP, with 4 replacements made in one operation.

To set the label text dynamically, open the SA_NumericalEntry's Custom Properties, select the Label property, switch to Expression or Reference mode, and enter Me.Speed.SP.Description as the default value. This pulls the label text from the speed attribute's description, ensuring it updates automatically when the panel is embedded in different equipment.

Custom Properties for SA_NumericalEntry1 showing the Label property configured as String type with default value Me.Speed.SP.Description.

Save, close, and check in the CommandPanelwithSetpoint graphic.

Configuring Equipment to Show Command Panels

The command panels are complete, but they will not appear unless the equipment graphics know how to display them. You need to make each equipment graphic clickable so that clicking it opens the appropriate command panel as a popup. This involves the same three steps for each equipment type: enabling TreatAsIcon, adding a Show Symbol animation, and copying the animation for reuse on other equipment.

Warning: Always set TreatAsIcon = True before adding the Show Symbol animation. If you add the animation first and then set TreatAsIcon, the animation may not activate properly. The correct order is: open the graphic, select the embedded element, set TreatAsIcon = True, and only then add the Show Symbol animation.

Step 1: Agitator. Open Agitator_Detailed for editing. Select the agitator graphic on the canvas and set TreatAsIcon = True on the Properties tab under Runtime Behavior. With the gate now open, double-click the agitator graphic and add a Show Symbol animation. Use the Galaxy Browser to select CommandPanelwithSetpoint from the Training toolset — this panel includes the speed setpoint field that the agitator needs. Uncheck "Use Symbol Name" for the title bar and enter OwningObject as the title. This makes the popup window dynamically display the name of whichever equipment was clicked, so the operator always knows which agitator they are commanding. Set the Position to Below and Parent Symbol so the popup appears directly under the agitator graphic.

Show Symbol animation for the Agitator fully configured: Reference = CommandPanelwithSetpoint, title using OwningObject, Position = Below Parent Symbol, modal with close button.

Step 2: Pumps. Copy the agitator's animation (right-click > Animations > Copy), then open Pump_Detailed. Set TreatAsIcon = True on the pump symbol, paste the animation, and change the Reference to CommandPanel — pumps do not need the speed setpoint panel. Copy this animation for reuse, save, close, and check in.

Step 3: Valves. Open Valve_Detailed, set TreatAsIcon = True, paste the animation (which already references CommandPanel), save, close, and check in. The valve inherits the same command panel behaviour with no additional configuration needed.

Testing in Runtime

Click Runtime to test the command panels. Click on Inlet1_001 (the first valve) — a command panel popup appears with the title "Inlet1_001" and buttons labelled "Open" and "Close." Click Open to command the valve open, then click Close to close it. Notice the Quality & Status hourglass icon briefly appearing between commands, indicating the write is in progress.

Inlet1_001 command panel with the Open button appearing lighter (Intensity2 style) and the yellow hourglass quality indicator showing the write is in progress.

Close the command panel by clicking the X on its title bar. Then click on Pump1_001 — its command panel shows "Start" and "Stop" buttons, automatically adapted from the pump's CmdOnMsg and CmdOffMsg attributes. Operate the pump and observe the results.

Warning: You must close the command panel popup (click the X) before performing any other actions in WindowViewer. If you leave the popup open, it blocks interaction with other elements because it is a modal window.

Finally, click on Agitator_001 — its command panel includes Start/Stop buttons plus the Speed setpoint field. Enter a speed value (for example, 45) and press Enter. The agitator should begin running at the specified speed, and the mixer monitoring screen should show the agitator's speed updating in real time.

Agitator_001 command panel with the Speed setpoint changed to 45.00 rpm after operator input.
Mixer monitoring screen showing Agitator_001 running at 45.03 rpm in real time.

Click Development! to return to WindowMaker when you are finished testing. You have now built a complete set of reusable command panels that adapt automatically to valves, pumps, and agitators — all through the power of custom properties, property forwarding with Me. references, and the TreatAsIcon gatekeeper.

Quiz — Test Your Understanding

Q1: Why do the CommandPanel's custom properties use Me.Cmd, Me.Cmd.OffMsg, and Me.Cmd.OnMsg as their default values?

These relative references create a forwarding chain. When the CommandPanel is embedded inside a valve or pump, Me.Cmd resolves to that equipment's Cmd attribute, and Me.Cmd.OffMsg / Me.Cmd.OnMsg resolve to its off/on message strings. This makes the panel automatically adapt to whichever equipment it is embedded in.

Q2: What is the purpose of the Element Style animation on the buttons?

The Element Style animation provides visual feedback to the operator. When the command is active (True), the active button appears with a brighter style (Intensity4), while the other button reverts to its default. This immediately communicates which state the equipment is in — Open vs. Close, or Start vs. Stop.

Q3: Why is the Pushbutton animation's Action set to "Set" on the Open button and "Reset" on the Close button?

Set writes True (1) to the Boolean command attribute, signalling an "on" command. Reset writes False (0), signalling an "off" command. Together they implement the Open/Close or Start/Stop toggle behaviour.

Q4: How does the Substitute References feature simplify configuring the SA_NumericalEntry for the agitator's speed setpoint?

Substitute References performs a bulk find-and-replace across all references inside the symbol. By replacing all Me.PV with Me.Speed.SP, every reference (PV value, engineering units, range max, range min) is updated in one operation, avoiding the need to manually edit each reference individually.

Q5: Why does the Show Symbol animation use "OwningObject" as the popup window's title?

Using OwningObject as the title makes the popup window dynamically display the name of the equipment that was clicked — "Inlet1_001", "Pump1_001", or "Agitator_001". This helps operators immediately identify which piece of equipment they are commanding without needing to cross-reference the display.
