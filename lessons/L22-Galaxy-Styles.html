Lesson 22: Galaxy Styles

Module 3 — Symbols | Pages 277–280

22.1 What Are Galaxy Styles?

In a well-designed HMI, visual consistency is not optional — it is essential. When operators scan a screen full of graphics, they need to instantly recognize what is normal, what is alarming, and what needs their attention. Galaxy Styles provide the mechanism for enforcing this consistency across your entire InTouch application.

Key Concept: Galaxy Styles
A Galaxy Style is a collection of visual overrides that define how graphics appear at runtime. Styles control everything from text fonts and colors to alarm border patterns, ensuring that every symbol in your Galaxy follows the same visual language.

Think of Galaxy Styles as a centralized theme system. Instead of manually setting colors and fonts on each individual graphic element, you define a style once and apply it across the application. This means that when your corporate visual standards change, you only need to update the style — not every symbol individually.

22.2 The Six Style Categories

Galaxy Styles are organized into six categories, each controlling a different aspect of visual presentation:

Style CategoryWhat It Controls
Quality & StatusVisual indicators for data quality states — communication errors, configuration errors, warnings, bad quality, security errors, and more
HMI ElementAppearance of text, fills, lines, and outlines on graphic elements — fonts, colors, blink settings, and line patterns
Alarm ElementVisual appearance of alarm severity, alarm states, and user acknowledgment indicators
Trend ElementVisual properties for the SA_SinglePen and SA_MultiTrendPen symbols from the Situation Awareness Library
User Defined24 customizable styles for your own visual needs
FormatVisual overrides for numerical values in InTouchViewApp, including value range, fixed width, and precision

22.3 Quality & Status Styles

The Quality & Status tab is where you define how data quality problems appear visually. When a sensor loses communication or a device enters an error state, the graphic needs to communicate this immediately. You can configure text, fill, outline, blink, and display icons for each quality state.

The default Standard_Style includes icons for all standard quality states: Communication Error, Configuration Error, Pending, Operational Error, Software Error, Security Error, Warning, Out Of Service, Device Failure, Bad, Uncertain, Initializing, and Live Value Playback. Each state has a configurable visual override that takes precedence over the native graphic properties.

The Styles Configuration interface showing the Quality & Status tab with icons for each data quality state

22.4 Alarm Element Styles

Alarm Element styles define the visual appearance of alarm severity and state at runtime. The system uses four severity levels, each color-coded:

SeverityColorMeaning
1RedCritical
2YellowHigh
3CyanMedium
4MagentaLow

Each severity level also has three states that affect the visual presentation:

  • ACK — The alarm has been acknowledged but is still in alarm condition
  • RTN — The alarm has returned to normal but has not been acknowledged
  • UNACK — The alarm is active and has not been acknowledged

Alarm Element styles also include alarm border styles in the form of visual outlines. These borders appear around graphics when alarms activate, providing a highly visible notification to operators.

22.5 HMI Element Styles

An HMI Element style defines the visual properties of text, fills, lines, and outlines on graphic elements. These styles override the native visual properties of elements, allowing you to standardize appearance across all symbols in your Galaxy.

The key visual properties you can override include:

  • Text: Font, size, style, color, and blink settings
  • Fill: Fill color and blink settings
  • Line: Line patterns, weight, color, and blink settings
  • Outline: Whether outlines are shown or hidden
Important: Predefined styles cannot be deleted, and new styles cannot be added to a Galaxy Style list. You can only modify the visual overrides within existing styles. If you need a completely new visual approach, you must duplicate an existing style card and modify the copy.

22.6 Managing Style Cards

The Styles area lists each style as a card. The Standard_Style is the default active style for every Galaxy. From the configuration area, you can:

  • Add a new style card by clicking the + button — this duplicates the selected card with an incremented name
  • Delete a style card by selecting it and clicking the − button
  • Make Default to activate a different style for the Galaxy (only one can be active at a time)
  • Rename by clicking the card title or pressing F2
  • Duplicate by pressing Ctrl+D
  • Export individual style cards as XML files

You can also import additional style cards from the installation path: C:\Program Files (x86)\ArchestrA\Framework\Bin\AdditionalElementStyles. This folder contains XML files with predefined styles that can be loaded into your Galaxy.

Tip: Use the Export All option to back up your entire style configuration. This is especially useful before making significant changes to your visual standards — you can always import the backup if something goes wrong.

22.7 Trend Element and Format Styles

Trend Element styles support the SA_SinglePen and SA_MultiTrendPen symbols found in the Situation Awareness Library. These styles control the visual appearance of trend pen displays.

Format styles provide visual overrides for numerical values displayed in InTouchViewApp. They allow you to standardize the format of value ranges, fixed widths, and decimal precision across your application. The list includes 25 user-defined format styles that can be customized to match your display requirements.

Quiz — Test Your Understanding

Q1: How many categories of styles are available in a Galaxy Style?

Six categories: Quality & Status, HMI Element, Alarm Element, Trend Element, User Defined, and Format.

Q2: What happens when you click the Add (+) button in the Styles area?

The currently selected style card is duplicated with an incremented name. The new card can then be renamed and edited.

Q3: What color is assigned to Critical severity alarms?

Red is assigned to Critical (severity 1) alarms.

Q4: Can you delete the default Standard_Style from a Galaxy?

No, predefined styles cannot be deleted. You can only modify their visual overrides or create new styles by duplicating existing ones.
