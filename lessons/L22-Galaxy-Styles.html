Lesson 22: Galaxy Styles

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 3 — Symbols, Section 8 (pages 277–280)

Galaxy Styles provide a centralized way to control the visual appearance of graphics across your entire application. This lesson covers the six types of style overrides, how to manage style cards, and how to import and export styles.

22.1 What Are Galaxy Styles?

Reference: Module 3, Section 8 (page 277)

A Galaxy Style applies visual and numerical standards to the presentation of graphics. Rather than configuring the appearance of every element individually, you define a style once and apply it across your entire Galaxy. This ensures consistent visual standards throughout your application — critical for large projects where multiple developers may be creating graphics.

You access Galaxy Styles from the menu: Galaxy > Configure > Galaxy > Styles. The configuration area displays each style as a card, and each card provides access to all available override types.

Key Concept: By default, the Standard_Style is provided and set as the active style for the Galaxy. You can add additional styles, designate a different active style, delete styles, and duplicate styles from the configuration area. Predefined styles cannot be deleted and new styles cannot be added to the style list itself — you work with the existing cards.

22.2 The Six Style Categories

Reference: Module 3, Section 8 (pages 277–278)

Each Galaxy Style card contains overrides for six categories of visual properties:

Style CategoryPurposeKey Overrides
Quality & StatusVisual indicators for tag quality and status statesIcons, colors, and text for communication errors, configuration errors, warnings, bad quality, security errors, and more
HMI ElementAppearance of graphic elements in symbolsText (font, size, style, color, blink), Fill (color, blink), Line (pattern, weight, color, blink), Outline (show/hide)
Alarm ElementAlarm severity and state indicatorsFour severity colors (Red=Critical, Yellow=High, Cyan=Medium, Magenta=Low) and three states (ACK, RTN, UNACK) plus border styles
Trend ElementVisual settings for trend symbolsSupports SA_SinglePen and SA_MultiTrendPen from the Situation Awareness Library

Key Concept: HMI Element style overrides take precedence over the native visual properties of elements. If a style specifies a red fill color for a certain state, that color will be used regardless of what the individual symbol defines.

22.3 Quality and Status Styles

Reference: Module 3, Section 8 (page 278)

The Quality & Status tab defines visual elements for runtime quality and status states. You can configure text, fill, outline, blink, and associated display icons for various conditions including:

  • Communication errors and configuration errors
  • Pending states and operational errors
  • Software errors and security errors
  • Warnings, out-of-service states, and device failures
  • Bad quality, uncertain quality, and initializing states
  • Live value playback indicators

Each status item has a configurable icon (typically a colored circle with a symbol), and you can customize the text, fill color, and outline for each. This level of control ensures that operators can immediately recognize the state of any data point at a glance.

22.4 HMI Element Styles

Reference: Module 3, Section 8 (page 278)

HMI Element styles define the visual properties of text, lines, outlines, and fills for elements shown in symbols. The four sub-categories are:

  • Text: Font, size, style, color, and blink settings for text elements
  • Fill: Fill color and blink settings for graphic fills
  • Line: Pattern, weight, color, and blink settings for lines
  • Outline: Whether outlines are shown or hidden

Each override specifies a preconfigured visual property that takes precedence over the native properties of an element. This means a single style change can update the appearance of every matching element across the entire Galaxy.

22.5 Managing Style Cards

Reference: Module 3, Section 8 (pages 278–280)

The Styles configuration area provides tools for creating, organizing, and managing style cards.

Adding and Deleting Cards

Click the Add button (+) to duplicate the currently selected card with an incremented name and a copy of all its styles. The new card can be renamed and edited independently. To delete a card, select it and click the Delete button (−).

Importing and Exporting

Click the vertical ellipsis (⋮) in the top-right of the Styles area to access import/export options:

  • Import: Loads one or more style cards from XML files. Additional styles are available at:
    C:\Program Files (x86)\ArchestrA\FrameWork\Bin\AdditionalElementStyles
  • Export All: Exports all style cards as an XML file for backup or reuse in other Galaxies.

Individual Card Management

Each card has its own vertical ellipsis menu with these options:

OptionEffectShortcut

DuplicateMakes a copy of the cardCtrl+D

The Import/Export menu — Import loads XML files from the AdditionalElementStyles folder, Export All saves all cards (page 279)
Each style card has its own context menu for management operations (page 279)
Tip: Use the Duplicate feature (Ctrl+D) to create a new style based on an existing one. This is faster than building a style from scratch — duplicate an existing card and then modify only the overrides that need to change.
Important: Only one style card can be set as the default (active) style for the Galaxy at a time. When you make a new card the default, the previous default is deactivated. Choose the active style carefully as it affects the visual appearance of all graphics that use it.

Knowledge Check

1. What are the six categories of overrides available in a Galaxy Style card?

Click to reveal answer

Quality & Status, HMI Element, Alarm Element, Trend Element, User Defined, and Format. Each category controls a different aspect of how graphics appear at runtime.

2. How do you access the Galaxy Styles configuration area?

Click to reveal answer

From the Galaxy menu, select Configure > Galaxy > Styles. This opens the Configure Galaxy Styles area where you can view, manage, and configure all style cards.

3. What happens when you click the Add button in the Styles area?

Click to reveal answer

The currently selected style card is duplicated with an incremented name and a complete copy of all its styles. The new card can then be renamed and edited independently.

4. Where can you find additional style XML files for import?

Click to reveal answer

Additional styles are available at C:\Program Files (x86)\ArchestrA\FrameWork\Bin\AdditionalElementStyles. This folder contains XML files that can be imported through the Import option in the vertical ellipsis menu.

5. What are the four alarm severity color codes defined in Alarm Element styles?

Click to reveal answer

Red for Critical, Yellow for High, Cyan for Medium, and Magenta for Low. Each severity also has three states: ACK (acknowledged), RTN (return to normal, unacknowledged), and UNACK (unacknowledged and in alarm).
