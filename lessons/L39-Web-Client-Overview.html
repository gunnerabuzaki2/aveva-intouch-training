Lesson 39: Web Client Overview

Module 8 — Web Client | Pages 527–536

39.1 Introduction

The InTouch Web Client allows you to view Industrial Graphics from an InTouch for System Platform application in any HTML5-supported web browser. A built-in web server eliminates the need for Remote Desktop Protocol (RDP) or Internet Information Services (IIS) — you simply enter a URL in a browser and access your application graphics directly.

Key capability: The Web Client works only with Managed InTouch applications. Standalone application windows must be converted to Industrial Graphics before they can be viewed in the browser.

39.2 Enabling the Web Client

By default, the Web Client is disabled to secure systems where no decision has been made to enable a web server. You need administrative privileges to enable or disable it.

Open the AVEVA Application Manager and navigate to the Web Client tab. Click the Web Client button on the toolbar — when it is not started, a small clock icon appears on the button. After a moment, the clock disappears, indicating the Web Client has started.

AVEVA Application Manager — Web Client button with clock icon indicating the server has not started
Note: WindowViewer must be running to provide data and animations for the Web Client. Always start WindowViewer first, then use the Web Client fast switch to open the browser view.

39.3 Configuring Graphics for Web Viewing

Only graphics stored in the folder set as the Web Client root folder are displayed in the browser. You configure this using the Industrial Graphic Toolbox in WindowMaker. A single symbol within the root folder can be set as the Home Symbol — it will be automatically displayed when the Web Client is accessed, and the home button in the Web Client interface returns to it.

IDE Industrial Graphics toolbox — TrainingGalaxy assets with Web Client and Runtime toggle buttons in the toolbar

39.4 Web Client Fast Switch

The Web Client fast switch in WindowMaker provides a quick way to open the Web Client in your default browser. Initially, this button may be disabled if the InTouch application path is not set to the current application. Starting WindowViewer via the Runtime fast switch resolves this by setting the application path, building the Web folder, and starting the Web Client fast switch.

39.6 Pan, Zoom, and Home Button

The Web Client supports pan and zoom gestures for all supported browsers and devices. Hold Ctrl and scroll to zoom (up to 500%), and press the mouse wheel to pan. The Home button restores the default view.

For InTouch applications, if Home Windows are configured, they take precedence over the Home Symbol. If only a Home Symbol is configured, it appears by default. If neither is configured, no default graphic is displayed.

39.7 Licensing and Authorization

Without a license, the Web Client operates in Single Session Mode — one browser can connect, possibly with multiple tabs. For full use, you need a Web Server license:

License TypeCapability
Unlimited Read-OnlyUnlimited sessions for viewing graphics
Unlimited ReadWriteUnlimited sessions for viewing and interacting with graphics
Authentication flow: The Web Server first authenticates the user (checking membership in aaInTouchUsers or aaInTouchRWUsers groups), then determines authorization. It attempts to acquire the ReadWrite license first; if unavailable, it falls back to Read-Only; if that's also unavailable, it uses Single Session Mode.

39.8 Known Limitations

Some features common to InTouch for System Platform are not yet supported by the Web Client, including ActiveX and .NET controls such as the Alarm Client. For a detailed list, consult the PDF "Viewing InTouch Application Graphics in a Web Browser."

39.9 Quiz — Test Your Understanding

Quiz

Q1: Why must WindowViewer be running before using the Web Client?

WindowViewer provides the real-time data and animations that the Web Client displays. Without WindowViewer, the browser would have no data source for the graphics.

Q2: What determines whether a user gets ReadWrite or Read-Only access in the Web Client?

The Web Server first checks if a ReadWrite license is available and if the user belongs to the aaInTouchRWUsers group. If both conditions are met, the user gets ReadWrite access. Otherwise, it falls back to Read-Only mode.

Q3: What is the difference between Home Windows and Home Symbol in the Web Client?

Home Windows take precedence — if configured, they always display by default regardless of any Home Symbol setting. If only a Home Symbol is configured, it appears by default. If neither is configured, no default graphic is shown.

Q4: Why is the Web Client disabled by default?

Enabling a web server on the installation machine has security implications. The Web Client is disabled by default to ensure that no web server is running until an administrator explicitly decides to enable it and configure appropriate security.
