Lesson 7: The InTouch Development Environment

Module 2 — Getting Started | Pages 55–60

Now that your Managed InTouch application exists, it is time to learn the tool you will use to build every display window inside it: WindowMaker. WindowMaker is the InTouch development environment — the place where you draw graphics, define animations, configure windows, and wire up navigation. This lesson introduces its interface, explains how the ribbon organizes design features, walks through the configuration options that shape your workflow, and covers the four types of windows you can create.

7.1 The WindowMaker Interface

WindowMaker's user interface follows standard Windows conventions, so it should feel familiar from the first moment you open it. You get right-click context menus for quick access to frequently used commands, floating and docking toolbars that you can rearrange to suit your workflow, pull-down menus for less common operations, and context-sensitive help that responds to whatever you have selected. A customizable color palette provides support for 16.7 million colors — limited only by your video display hardware.

When WindowMaker opens for the first time after application initialization, it displays its full default configuration. Every toolbar, every panel, every ribbon tab is visible, giving you an overview of everything available. The environment is configurable, however — you can show, hide, and rearrange panels as your comfort level grows. The figure below shows the default layout with its major components labeled.

The default WindowMaker layout: ribbons at the top, Windows and Scripts panels on the left, Industrial Graphics and Properties panels on the right, and the empty design canvas in the center.

The WindowMaker interface is organized into seven distinct views, each serving a specific purpose in your development workflow:

ViewPurpose
WindowsLists all user-created Native and Frame windows. This is where you navigate between windows during development.
SupertagsTemplate maker for constructing Supertags — a hierarchical tag structure that lets you define reusable tag groups.
ToolsQuick access to major WindowMaker utilities such as the Tag dictionary and Alarm groups.
ScriptsAccess to the global script type editors (Application, Key, Condition, Data change, and ActiveX event scripts).
CanvasThe central design workspace where you draw and arrange graphics.
Industrial GraphicsDisplays items from the Graphics tab of the IDE — the symbols, libraries, and other graphical assets available to your application.
PropertiesShows the editable properties of the currently selected element (or the Frame window if nothing is selected).
Key Concept: Canvas vs. Windows Panel
The Canvas is where you work — it is the blank drawing surface. The Windows panel lists all the windows in your application and lets you switch between them. Think of the Windows panel as a file explorer and the Canvas as the open document. You double-click a window in the panel to open it on the Canvas for editing.

7.2 The WindowMaker Ribbon

The ribbon at the top of WindowMaker contains all of the design features you need to build an InTouch application. It includes both legacy features for maintaining older InTouch applications and modern features for Managed InTouch applications. The ribbons are organized by the tasks and workflows that a designer would follow — from basic clipboard operations to advanced animation and scripting.

The WindowMaker Home ribbon, organized into functional groups for clipboard operations, inserting objects, aligning elements, and arranging layers.

Each ribbon tab serves a distinct role in the development workflow:

File (Backstage) — Opens the backstage view where you create and manage windows, import and export files, and configure the behavior of both WindowMaker and WindowViewer. This is the command center for application-level operations.

Home — The everyday workflow tab. It contains clipboard tools (copy, paste, duplicate), the Insert group (buttons, SmartSymbols, text, graphics, wizards), alignment and arrangement tools, and access to major utilities like the Tag dictionary and Alarm groups.

Draw (Legacy only) — Used to draw legacy visual elements: lines, ellipses, rectangles, and other basic shapes. Also handles font formatting and the placement of original InTouch objects such as SmartSymbols, real-time trends, and historical trends. This tab exists primarily for maintaining older applications; for new Managed InTouch applications, you will primarily use the industrial graphics approach.

Animation — Provides the tools for animating legacy visual elements with InTouch-specific animations. Also handles tag substitution and string substitution, which let you rewire an animation's data source without rebuilding the graphic.

Properties — Supports hierarchical symbol and instance selection from Application Server's model. This is how you navigate the parent-child relationships between objects and configure the properties of selected symbols. It also contains legacy features for the Symbol Factory and NAD notification.

View — Controls which WindowMaker panels are visible, provides access to AVEVA Connect (the cloud-based graphics repository), and lets you set the Dark mode theme for the editor.

Help — Opens web-based help topics for System Platform products in your default browser.

Tip: If you are working primarily with Managed InTouch applications, you will spend most of your time on the Home tab. The Draw and Animation tabs are essential for legacy applications but are used less frequently in modern System Platform deployments where industrial graphics (symbols) handle the visual representation.

7.3 WindowMaker Configuration

Before you begin creating windows, it is worth understanding the configuration options that shape how WindowMaker behaves. You can access these settings by clicking File > Configure > WindowMaker in the backstage view. The configuration screen lets you control several preferences that affect your day-to-day development experience.

The WindowMaker configuration screen, accessed from File > Configure, controls how the editor behaves during development.

Some of the most useful configuration options include:

  • Lock window size — Prevents graphic distortion when the resolution changes between development and production. When enabled, the window dimensions you set at design time are preserved regardless of the target display resolution.
  • Show tag count — Displays the total number of tags used in the application. Useful for monitoring complexity and staying within licensing or performance limits.
  • Disable script autocomplete — Prevents the autocomplete dropdown from appearing under your cursor while you are writing scripts. Some developers find autocomplete distracting when they are typing quickly.
  • Enable fast switch — Speeds up switching between open windows during development.
  • Show grid — Displays a grid on the canvas to help with alignment of graphics.
Tip: The Lock window size option is especially valuable when your development machine has a different resolution than the production operator stations. Enabling it ensures that a window designed at 1920×1080 on your laptop will display identically on a 1280×1024 operator panel — the window will not stretch or distort, though its visible area may be clipped.

7.4 InTouch Window Types

An InTouch application is a collection of windows. Each window can contain graphics, text, animations, and scripts. When you create a new window (by clicking the + button in the upper-left corner of WindowMaker), you must choose a window type. The type determines how the window interacts with other open windows at runtime. There are four types:

Replace — This is the most common window type. When a Replace window opens, it automatically closes any other windows it overlaps, including other Replace and Popup windows. Replace windows are ideal for primary display screens — operator stations typically have a single Replace window visible at a time, with navigation buttons switching between different screens.

Overlay — An Overlay window appears on top of whatever is currently displayed but does not close the underlying windows. It can be larger than the windows it covers. Clicking on any visible portion of a window behind the Overlay makes that window active. Overlay windows are useful for pop-up details or secondary information panels that appear alongside the main display.

Popup — Similar to an Overlay, but with a critical difference: a Popup window always stays on top of all other open windows, even if you click on a different window. This makes Popups ideal for alarm notifications, system messages, or any display that must remain visible regardless of what the operator is doing.

Frame — A Frame window hosts a single graphic and enables pan, zoom, and touch capabilities at runtime. Frame windows are useful for displaying large graphics (such as plant-wide overview maps) that an operator needs to navigate by scrolling and zooming.

Key Concept: Choosing the Right Window Type
The window type you choose has a direct impact on operator experience. Use Replace for main navigation screens, Overlay for supplementary panels that appear alongside the main display, Popup for critical alerts that must never be hidden, and Frame for large graphics that need pan and zoom. Mixing types within a single application is normal — most applications use a combination of all four.

7.5 Window Properties

Every window has a set of properties that control its appearance and behavior. When you create a new window or select an existing one, the Properties panel on the right side of WindowMaker shows the editable attributes. Key properties include:

  • Name — The unique identifier for the window (e.g., Navigation, OverviewDisplay).
  • Window type — Replace, Overlay, Popup, or Frame (see above).
  • Location — The X and Y coordinates of the window's top-left corner.
  • Size — The width and height in pixels.
  • Window color — The background color of the window.
  • Title bar — Whether the window has a visible title bar at runtime.
  • Close button — Whether the operator can close the window at runtime.
  • Size controls — Whether the window can be resized by the operator at runtime.
Example window properties for a Navigation window — a narrow Replace window at the top of the screen with no title bar and no resize handles.
Warning: Frame Window Limitations
Frame windows provide pan and zoom capabilities, but they come with significant restrictions. Frame windows do not support multiple graphics, native InTouch controls, ActiveX controls, or SmartSymbols. They also do not support undo/redo, flip/rotate, print, cut/copy/paste, cross-reference, window-to-symbol conversion, or instance editing. If you need any of these features, use a different window type. Frame windows are specifically designed for hosting a single graphic with pan/zoom navigation.

7.6 Template Windows and Duplication

As your application grows, you will find yourself creating windows that are variations of each other — the same navigation bar, the same layout structure, the same color scheme. Rather than rebuilding these elements from scratch each time, you can use template windows. A template window is a reusable starting point that lives in the Templates Windows folder in the Windows panel. When you mark a window as a template, it is listed separately, and you can use it as the basis for new windows without modifying the original.

An important detail: modifying a template window does not change windows that were previously built from it. The template is a snapshot at the moment of creation, not a live link. This means you can safely evolve a template over time while preserving the consistency of windows already derived from earlier versions.

When you want to create an exact copy of an existing window — rather than a variation — use the Save Window As command from the File menu. Open the window you want to duplicate, select Save Window As, and give the new window a different name. The copy is independent of the original from that point forward.

7.7 Creating a Button

To place a button in a window, select the Button tool from the Draw ribbon. Move your cursor over the window — it changes to a crosshair (+). Click and drag to define the button's size, then release the mouse button. The button appears with default text.

To change the button's label, right-click the button and select Substitute > Substitute Strings. This opens the string substitution dialog where you can set the text that appears on the button at runtime. To add animation — such as navigation behavior when the button is clicked — double-click the button to open the animation configuration dialog.

Tip: In a Managed InTouch application, you will often use industrial graphics (symbols) instead of raw buttons. However, understanding how raw buttons work is important for legacy applications and for quick prototyping. The Substitute Strings mechanism is also used extensively when customizing symbols at the instance level.

Quiz — Test Your Understanding

Q1: What are the four window types in InTouch, and how does each one behave at runtime?

Replace closes any overlapping windows when it opens. Overlay appears on top but lets you click through to underlying windows. Popup stays on top of all windows even when another is clicked. Frame hosts a single graphic with pan/zoom capabilities.

Q2: What is the difference between a template window and a regular window?

A template window is a reusable starting point that appears in the Templates Windows folder. You can create new windows from it, but modifying the template does not change windows that were previously built from it. Regular windows are standalone — they do not serve as starting points for other windows.

Q3: Why would you enable the "Lock window size" configuration option?

Lock window size preserves the window dimensions you set at design time, preventing graphic distortion when the production node has a different resolution than the development machine. Without it, windows may stretch or compress on displays with different aspect ratios.

Q4: What are the limitations of Frame windows compared to other window types?

Frame windows cannot host multiple graphics, native InTouch controls, ActiveX controls, or SmartSymbols. They do not support undo/redo, flip/rotate, print, cut/copy/paste, cross-reference, window-to-symbol conversion, or instance editing. They are designed specifically for single-graphic pan/zoom navigation.

Q5: How do you change the text label on a button you have drawn in WindowMaker?

Right-click the button and select Substitute > Substitute Strings. This opens the string substitution dialog where you can define the text that will appear on the button at runtime. You can also double-click the button to add animation behavior.
