Lesson 18: Custom Properties

Module 3 — Symbols | Pages 217–222

As your InTouch HMI graphics grow more sophisticated, a recurring design challenge emerges: how do you create a single symbol that can be reused across many different pieces of equipment without hard-coding every piece of data? Imagine building a command panel that must work equally well for a valve, a pump, and an agitator. Each of these requires different labels, different command references, and perhaps different data types. Copying and pasting the symbol for each equipment type would be tedious and error-prone. Custom properties provide the answer. They allow you to define named, configurable attributes on a symbol so that any engineer embedding that symbol can adjust its behaviour from the Properties tab — without ever opening the symbol for editing.

In this lesson, you will learn what custom properties are, how to create and configure them through the Edit Custom Properties dialog, and how they interact with the Button tool, Pushbutton animations, and Show Symbol animations. You will also encounter the TreatAsIcon property — a small but critical Boolean that must be set correctly before interaction animations will function on embedded graphics.

Understanding Custom Properties

Custom Properties are named, configurable attributes that you define on a graphic symbol to extend its functionality. When a graphic containing custom properties is embedded inside another graphic, those properties appear on the Properties tab in the Custom Properties category. This makes the symbol flexible and reusable: a consumer can adjust its behaviour — changing button labels, default setpoints, or connection strings — without ever modifying the symbol's internal graphics or animations.

A custom property is more than a simple text field. It can hold a literal value such as True or "Open", but it can also reference an automation object attribute like Tank1.InletValve.PV, or a relative reference such as Me.Cmd that resolves to the hosting object's command attribute at runtime. You can even chain custom properties together: one symbol's property can forward its value to a custom property on an embedded symbol, creating a cascade of references that automatically adapts to the host object. Script functions and expressions are also valid, giving you considerable power in a small configuration surface.

The Edit Custom Properties dialog supports eight data types, each represented by a distinct icon: Boolean for True/False flags, Double and Float for numeric values (with differing precision), Integer for whole numbers, String for text, Time and ElapsedTime for temporal values, and HistorySummary for aggregated historical data. Choosing the right data type matters because it determines what the property can hold and how it interacts with animations and scripts at runtime.

Creating and Configuring Custom Properties

You create and edit custom properties through the Edit Custom Properties dialog. To open it, right-click the canvas in the Graphic Editor and choose Custom Properties, or simply press Ctrl+M. The dialog presents a property grid on the left and a configuration panel on the right. Click the Add custom property button (the + icon) to create a new entry, then configure it using four fields.

The Data Type drop-down determines what the property can hold. The Default Value field accepts a literal value, a tag reference, or an expression — and for String, Time, and ElapsedTime types, a toggle button to the left lets you switch between Static Text mode (the field is treated as a plain string) and Expression or Reference mode (the field is evaluated as a tag reference or mathematical expression at runtime). The Visibility field controls whether the property is Public (visible and editable on the Properties tab when the symbol is embedded) or Private (hidden from consumers, protecting internal configuration). Finally, the Description field provides a brief note explaining the property's purpose, which is especially helpful when symbols come from built-in libraries.

The Edit Custom Properties dialog showing the + Add custom property button, an empty property grid, and the right-side panel with Data Type, Default Value, Visibility, and Description fields.
Edit Custom Properties dialog for CommandPanel showing the Cmd property: Data Type = Boolean, Default Value = Me.Cmd, Visibility = Public.
Warning: If you switch a String property's Default Value to "Expression or Reference" mode without providing a valid expression or reference, a red border and warning icon appear. The dialog will refuse to close until a valid value is entered. This is a deliberate safety check — an empty reference at runtime would produce unpredictable results.
Tip: When designing symbols for reuse, set the most commonly needed properties to Public and leave internal bookkeeping properties as Private. This keeps the Properties tab clean for consumers while preserving the symbol's internal logic.

The Button Tool and Pushbutton Animation

InTouch's Graphic Editor includes a Button tool that lets you place interactive push buttons directly on the canvas. To create a button, select the Button tool from the Tools pane, click and drag diagonally to define its size, then type a label and press Enter. The button appears with default styling that you can refine through the Properties tab — adjusting the Text, Width, Height, and ElementStyle properties to match your design.

A button by itself is just a visual element. To make it perform an action at runtime, you apply a Pushbutton animation. This animation writes a value to a Boolean, analog, or string reference when the operator presses or releases the button. For example, setting the Boolean Reference to Cmd and the Action to Set will write True to the command attribute whenever the button is pressed — exactly what you need for an "Open" or "Start" command. Conversely, the Reset action writes False, making it suitable for "Close" or "Stop" buttons.

Show Symbol and Hide Symbol Animations

Building interactive popups — command panels, detail views, and configuration dialogs that appear on demand — is a fundamental requirement in most HMI applications. InTouch provides the Show Symbol animation for exactly this purpose. When a user clicks an element at runtime, the specified graphic appears as a popup window. You control the popup's behaviour through several options: which graphic appears (selected by name or browsed via the Galaxy Browser), whether the window has a title bar and close button, whether it is modal (blocking interaction with other windows) or modeless, whether it can be resized, and its initial position and size. A shortcut key can also be assigned for keyboard-driven access.

The complementary Hide Symbol animation closes a popup that was previously shown. Together, Show Symbol and Hide Symbol give you complete control over the popup lifecycle without writing any script code.

The TreatAsIcon Property

There is one critical detail that trips up many developers when they first attempt to add interaction animations to embedded graphics. By default, when a graphic is embedded inside another symbol, its own internal animations take precedence over any animations defined on the parent. This means that a Show Symbol animation you add to the embedded graphic will not work until you change a single Boolean setting: TreatAsIcon.

TreatAsIcon is a Boolean property on embedded graphics. When set to True, it enables interaction animations — such as Show Symbol and Pushbutton — defined on the embedded graphic, allowing it to respond to user clicks at runtime. When set to False (the default), the embedded graphic's own animations take precedence and parent-level interactions are blocked. You must set TreatAsIcon to True before adding a Show Symbol animation, or the animation may not activate properly.

Think of TreatAsIcon as a gatekeeper. In its default False state, the embedded graphic is treated as a passive, display-only element — its internal animations run, but no new interaction animations can be layered on top. Flip it to True, and the graphic becomes an interactive element that can respond to user clicks, open popups, and display dynamic labels. Without TreatAsIcon set to True, a valve symbol embedded in a process display would remain inert: the operator could see the valve's status but could not click it to open a command panel.

Properties panel for SA_Agitator_Settler showing the Runtime Behavior section with TreatAsIcon = True highlighted.
Warning: Always set TreatAsIcon = True before adding a Show Symbol animation to an embedded graphic. If you add the animation first and then toggle TreatAsIcon, the animation may not register properly and will fail silently at runtime. The correct order of operations is: open the graphic for editing, select the embedded element, set TreatAsIcon = True, and only then add the Show Symbol animation.

Quiz — Test Your Understanding

Q1: What is the difference between Public and Private custom properties?

A Public custom property is visible and editable on the Properties tab when the symbol is embedded in another graphic — this is what consumers use to configure the symbol's behaviour. A Private custom property is hidden from consumers, which protects internal configurations from accidental changes.

Q2: When would you use "Expression or Reference" mode instead of "Static Text" mode for a String custom property?

Use "Expression or Reference" mode when the default value should be a tag reference (like Me.Cmd.OffMsg) or a runtime expression. "Static Text" mode treats the field as a fixed string literal that never changes. The choice depends on whether the label needs to adapt at runtime or remain constant.

Q3: Why must TreatAsIcon be set to True before adding a Show Symbol animation?

When TreatAsIcon is False (the default), the embedded graphic's internal animations take precedence and new interaction animations are blocked. Setting it to True opens the gate, enabling the Show Symbol animation to respond to user clicks and open popup windows. The setting must be applied first so the animation registers correctly.

Q4: What does the Pushbutton animation's "Set" action do compared to "Reset"?

Set writes True (or 1) to the referenced Boolean tag when the button is pressed, signalling an "on" command. Reset writes False (or 0), signalling an "off" command. Together, Set and Reset implement Start/Stop or Open/Close button pairs.

Q5: Can a custom property hold a reference to another custom property on an embedded symbol? What is this technique called?

Yes. A custom property can reference a custom property on an embedded symbol, creating a chain of property forwarding. When the outer symbol's property value changes, the embedded symbol's property automatically receives the forwarded value. This technique makes symbols composable — complex panels can be assembled from simpler, independently configurable building blocks.
