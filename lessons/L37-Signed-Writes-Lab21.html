37.1 Introduction

This lesson covers two connected topics. First, you will configure WindowViewer's inactivity timeout settings to automatically log off inactive operators. Then, you will create an InactivityMessage symbol that displays a blinking warning when inactivity is detected, and an Inactivity window to host that message. These elements form a critical part of your application's security posture — preventing unauthorized access when operators leave their workstations unattended.

Objectives — Upon completion, you will be able to:
  • Configure WindowViewer inactivity warning and timeout settings
  • Create a symbol with Predefined and Named scripts for inactivity messaging
  • Add a Blink animation to draw attention to warnings
  • Create a popup window to host the inactivity message
  • Use Condition scripts to open and close the Inactivity window

37.2 Configuring Inactivity Timeouts

WindowViewer can automatically log off an inactive operator, requiring them to re-authenticate before continuing. This prevents unauthorized access when workstations are left unattended.

How the timer works: A timer measures the period the operator has not interacted with the running application. It resets only when the operator clicks the mouse or presses a key — simply moving the mouse does not reset the timer.

In WindowMaker, navigate to File > Configure > WindowViewer. On the Preferences tab, set the Inactivity Warning to 15 seconds and the Timeout to 30 seconds. Click Save — a notification confirms the changes will take effect the next time WindowViewer starts. Close WindowViewer.

WindowViewer Preferences tab showing inactivity timeout settings: Warning at 15 seconds and Timeout at 30 seconds
WindowMaker notification confirming WindowViewer configuration was updated and must be restarted

On the canvas, add a Text element with the text "Inactivity Message" and name it Message1. Below that, add a Text Box element with the text "Action Message" and name it Message2.

Right-click the canvas and select Scripts. In the Edit Scripts window, configure a Predefined Script with the trigger set to On Show (the default). The script body sets both message texts to display an inactivity warning when the symbol is shown:

When the inactivity warning period expires, this script changes the displayed text to "Inactivity Warning" and "Click mouse or use keyboard to prevent system log off."

37.3.2 Adding a Named Script for Timeout

Add a Named Script called TimeoutMessage with the expression InTouch:$InactivityTimeout and trigger set to OnTrue. This script fires when the inactivity timeout period expires, changing the messages to "Inactivity Timeout" and "You have been logged off!" Click OK to close Edit Scripts.

Edit Scripts window showing Named Script "TimeoutMessage" with Expression InTouch:$InactivityTimeout and trigger OnTrue

37.3.3 Styling and Adding the Blink Animation

Select both text elements, align them to the Centers using the toolbar, and set the Element Style to Title. Use Align Center to center both elements horizontally.

WindowMaker canvas with Align Center tooltip — two selected text elements ready for center alignment

Double-click the Action Message text box and add a Blink animation. In the Blink animation dialog, set the Boolean state expression to true, configure it to blink when the expression is True, select Medium speed, and set the Text Color to yellow. This makes the action message flash to draw the operator's attention.

Edit Animations dialog configuring Blink animation with Boolean state true, Medium speed, and yellow Text Color

Save, close, and check in the graphic.

Why a Blink animation? In industrial environments, a flashing message immediately draws the operator's eye to the warning, ensuring they notice the inactivity alert before timeout occurs.

37.6 Quiz — Test Your Understanding

Quiz

Q1: What action resets the inactivity timer in WindowViewer?

Only clicking the mouse or pressing a key resets the timer. Simply moving the mouse does not count as interaction — a click is required.

Q2: What is the difference between the Predefined Script and the Named Script in the InactivityMessage symbol?

The Predefined Script triggers on "On Show" and displays the inactivity warning message. The Named Script monitors the $InactivityTimeout tag and triggers on "OnTrue" to display the timeout message when the user is logged off.

Q3: Why does the Condition script check $Operator before showing the Inactivity window?

The check prevents the warning from appearing when no user is logged in. If no one is logged on, there is nobody to warn about inactivity, so showing the window would be unnecessary and confusing.

Q4: What happens to the Inactivity window when the user interacts with the application after seeing the warning?

The $InactivityWarning tag is set to 0, triggering the On False condition. The script then hides the Inactivity window, returning the operator to their normal view.

Q5: Why is the Inactivity window configured as a Popup without a title bar?

A Popup window appears on top of other windows, ensuring the warning is visible. Omitting the title bar prevents the operator from accidentally closing the warning without interacting with the application, which would defeat the purpose of the inactivity detection.
