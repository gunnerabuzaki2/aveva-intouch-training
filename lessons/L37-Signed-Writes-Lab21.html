Lesson 37: Lab 22 – Locking Down Your Application

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 7, Lab 22 (pages 501–510)

Continuing Lab 22 — configuring inactivity timeouts, creating the InactivityMessage symbol with blinking animations, building the Inactivity warning window, and writing condition scripts to automate window display based on inactivity state.

37.1 Configuring Inactivity Timeouts

Reference: Module 7, Lab 22 (pages 501)

Lab 22 — Part 1: The first task is to configure WindowViewer's inactivity settings so that the system automatically warns users and then logs them off after a period of inactivity.

In WindowMaker, go to File > Configure > WindowViewer. On the Preferences tab, find the Inactivity in secs section. Set Warning to 15 seconds and Timeout to 30 seconds. Click Save. A notification confirms the configuration was updated — you will need to restart WindowViewer for the changes to take effect. Close WindowViewer.

Inactivity settings — Warning at 15 seconds, Timeout at 30 seconds (page 501)
The confirmation dialog — WindowViewer must be restarted for changes to take effect (page 501)
Key Concept: The inactivity timer resets whenever the operator uses the mouse (a click, not just movement) or keyboard. Simply moving the mouse does not reset the timer — this prevents operators from avoiding logoff by jiggling the mouse.

37.2 Creating the InactivityMessage Symbol

Reference: Module 7, Lab 22 (pages 502–506)

Next, you will create a symbol that displays a blinking warning message when the inactivity period is triggered, and changes its text when the timeout occurs.

Steps 6–10. Create a new symbol named InactivityMessage and open it for editing. Add a Text element with the text "Inactivity Message" and name it Message1. Below it, add a Text Box element with the text "Action Message" and name it Message2. Right-click the canvas and select Scripts.

Message1 text element placed on the canvas (page 502)
Message2 text box added below Message1 — both elements on the canvas (page 502)

Configure Predefined Scripts

Step 12. In the Edit Scripts dialog, configure a Predefined Script with the On Show trigger. This script changes the displayed text when the inactivity warning period expires:

'Change displayed text when the inactivity warning period expires.
Message1.Text = "Inactivity Warning";
Message2.Text = "Click mouse or use keyboard to prevent system log off";
The Predefined Script — On Show trigger changes text to the inactivity warning (page 503)

Configure Named Script for Timeout

Steps 13–14. Add a Named Script called TimeoutMessage with expression InTouch:$InactivityTimeout and trigger OnTrue. This script runs when the timeout period expires:

'Change displayed text when the inactivity timeout period expires.
Message1.Text = "Inactivity Timeout";
Message2.Text = "You have been logged off!";
The TimeoutMessage named script — triggered when $InactivityTimeout becomes true (page 504)

Style and Animate the Symbol

Steps 15–19. Select both text elements and use the toolbar to align them. Configure the Element Style as Title and center-align both elements. Double-click the Action Message text box (Message2) and add a Blink animation.

Aligning text elements using the toolbar (page 505)
Center alignment applied to both text elements (page 505)
Adding a Blink animation from the visualization options (page 505)

Steps 20–22. Configure the Blink animation with Boolean state, expression true, Blink When True, Medium speed, and Text Color set to yellow. Click OK and save/check in the graphic.

Blink animation — text will blink in yellow when the warning is active (page 506)

37.4 Creating Inactivity Trigger Scripts

Reference: Module 7, Lab 22 (page 510)

Now you will create two InTouch Condition scripts. The first script opens the Inactivity window when the inactivity warning triggers. The second script closes it when the warning clears.

Step 30. In the Scripts pane, double-click Condition to open the Condition Scripts editor.

The Condition Scripts editor — ready for the $InactivityWarning trigger (page 510)
Key Concept: Condition scripts in InTouch monitor a tag's state and execute when the condition becomes true or false. By using $InactivityWarning as the condition, you can show or hide the warning window automatically based on the system's inactivity detection.

The first condition script will use $InactivityWarning with trigger On True to show the Inactivity window when the warning activates. The second will use the same tag with trigger On False to hide it when the user interacts with the application. The full script configuration will be completed in Lesson 38.

Knowledge Check

1. What is the difference between the Predefined Script (On Show) and the Named Script (TimeoutMessage) in the InactivityMessage symbol?

Click to reveal answer

The Predefined Script uses the On Show trigger — it runs when the symbol becomes visible, changing the text to "Inactivity Warning." The Named Script (TimeoutMessage) monitors the $InactivityTimeout tag and triggers OnTrue — it runs when the timeout period expires, changing the text to "Inactivity Timeout" and "You have been logged off!"

2. Why must WindowViewer be restarted after changing the inactivity timeout settings?

Click to reveal answer

The inactivity settings are loaded when WindowViewer starts. Changes made in the configuration dialog are saved to the configuration file, but the running WindowViewer instance continues using the old values. A restart is required to load the new settings into memory.

3. What does the Blink animation's yellow Text Color achieve in the InactivityMessage symbol?

Click to reveal answer

The yellow blinking text draws the operator's attention to the inactivity warning message. Since the Inactivity window has a red background, the yellow text provides high contrast and makes the warning message stand out, alerting the operator that they need to interact with the application to prevent automatic logoff.
