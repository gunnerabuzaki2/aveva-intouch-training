Lesson 34: Historian Client Trend + Lab 19

Module 6 — Trend Visualization | Pages 431–438

In Lab 18, you built the TrendHistDisplay symbol and connected it to the Historian server using AddServerEx(). Now it is time to complete the integration: you will add the server initialization property, wire up the script, add a navigation button so operators can reach the trend from anywhere in the application, and then thoroughly test it at runtime by comparing mixer levels and experimenting with live mode.

34.1 Configuring the Historian Connection

The TrendHistDisplay symbol needs to know which Historian server to connect to. Rather than hard-coding a server name into a script, you store it in a custom property. This makes the symbol portable — you can deploy it to a different server simply by changing the default value, without touching any scripts.

Adding the HistServer Custom Property

Open TrendHistDisplay in the Graphic Editor. Right-click the canvas and select Custom Properties. Add a new property with these settings:

PropertySetting
NameHistServer
Data TypeString
Default ValueYour Historian node name (your instructor will provide this)
VisibilityPrivate
The HistServer property stores the Historian node name. Private visibility keeps it hidden from operators at runtime.

Click OK to close the dialog. The Private visibility ensures that when TrendHistDisplay is embedded as a component inside another graphic, operators cannot see or change the server name — only developers can modify it at design time.

Writing the Server Initialization Script

Next, you need to tell the aaTrendControl which server to use. Right-click the canvas and open Scripts. In the Predefined Scripts section, confirm that the Trigger type is set to On Show. This is critical: the script runs every time the symbol becomes visible, which means the server connection is established fresh each time the trend window opens.

Enter the following script body:

Trend.AddServerEx(HistServer, "", "", false);

The On Show script calls AddServerEx to connect the trend to the Historian server when the symbol is displayed.
Key Concept: AddServerEx() vs. AddServer(). The AddServerEx() function takes additional parameters for username, password, and an integrated-security flag. By passing false for the last parameter, you tell the control to use Windows integrated authentication — no separate credentials are needed when the operator is already logged into a domain-joined workstation.

Click OK to close Edit Scripts. Save, close, and check in the graphic.

34.2 Adding Navigation

An operator should not have to hunt through a file system to find the trend display. You will add a button to the SystemNavigation symbol so the Historian Client Trend is accessible from the main navigation bar — just like Mixers, Alarm Overview, and the other views.

Duplicating a Navigation Button

Open the SystemNavigation symbol for editing. Find the Historical Alarms button, duplicate it, and move the copy to the right. Use Substitute Strings to change the button text to "Historian Client Trend".

The navigation bar now includes six buttons, with the new Historian Client Trend button on the far right.

Open the custom properties for the new button and change the GraphicName default value to TrendHistDisplay. This tells the navigation button which graphic to display when clicked.

Setting GraphicName to TrendHistDisplay links the navigation button to the trend display symbol.

Save, close, and check in the graphic. The navigation is now complete.

34.3 Testing at Runtime

Now the real test begins. You will exercise every feature of the Historian Client Trend: single-tag viewing, multi-tag comparison, preset time ranges, and live mode.

Single-Tag Trend View

Switch to Runtime and click the Historian Client Trend button in the navigation. After a moment, the trend appears with the Tag Picker panel showing the S01ENG server. In the Tags list, select Level_001.PV and drag it onto the chart. The chart immediately displays the mixer level with alarm limits highlighted — orange for HiHi and LoLo, yellow for Hi and Lo.

A single mixer level trend showing the characteristic fill-and-drain cycle with alarm limit bands.
Key Concept: Drag-and-drop tag selection. The Historian Client Trend does not require you to pre-configure which tags to display. Operators pick any tag from the Historian database and drag it onto the chart at runtime. This makes the trend a powerful diagnostic tool for troubleshooting unexpected process behavior.

Multi-Tag Comparison

The real power of this trend is comparing multiple mixers side by side. In the Tags panel, select Level_002.PV through Level_008.PV — hold Ctrl or Shift for multi-select — and drag them all onto the chart. Each tag appears as a different colored line.

Multiple mixer levels displayed as overlapping trend lines reveal the time offsets and latency between different mixers.

The overlapping lines tell a story: each mixer fills and drains at slightly different times, and the horizontal offset between lines reveals the latency. You can also drag other tags — temperature, flow, speed — to see how different process variables correlate.

Time Range and Live Mode

Use the time range dropdown above the chart to select "1 minute". The chart zooms into that window, showing higher resolution. Then click the Enable/Disable live mode button — the chart begins scrolling continuously, updating in real time as new data arrives from the Historian.

The preset time range menu offers quick access to common analysis windows.
In live mode, the chart scrolls continuously, giving operators a real-time window into the process.
Tip: Live mode is especially useful during commissioning or when an operator suspects an intermittent problem. The scrolling display makes it easy to spot transient spikes or dips that might be missed in a static historical view.

Close the TrendHistDisplay window and click Development to return to WindowMaker. <End of Lab>

Watch Out: If the trend shows "No Data" after dragging a tag, the Historian may not be archiving that tag. Verify in the Historian configuration that the tag is included in an archive and that data collection is active. Also check that the time range you selected actually contains data for that tag.

Quiz — Test Your Understanding

Q1: Why is the HistServer custom property set to Private visibility instead of Public?

Private visibility hides the property when TrendHistDisplay is embedded as a component in another graphic. Since the server name is an internal configuration detail, operators should not see or modify it at runtime. Only developers need access during design time.

Q2: What does the third parameter (false) in AddServerEx(HistServer, "", "", false) control?

The fourth parameter controls whether integrated (Windows) security is used. When set to false, the control uses Windows integrated authentication — the operator's domain credentials are passed automatically without requiring a separate username and password prompt.

Q3: How do you select multiple tags in the Tag Picker to compare them on the same trend chart?

Hold Ctrl (for individual multi-select) or Shift (for range select) while clicking tags in the Tags panel. Then drag the entire selection onto the chart. Each tag appears as a different colored line, making it easy to compare values across multiple mixers or process variables.

Q4: What happens when you click the Enable/Disable live mode button on the trend chart?

The chart switches from a static historical view to a continuously scrolling display. New data points are added in real time as they arrive from the Historian, giving operators a live window into current process behavior. This is particularly useful during commissioning or troubleshooting transient events.

Q5: Why must the AddServerEx script use the On Show trigger rather than On Initialize?

The On Show trigger fires every time the symbol becomes visible, which means the server connection is re-established each time the trend window opens. This ensures a fresh, valid connection. On Initialize only runs once when the symbol is first loaded, so if the connection drops or the window is closed and reopened, the trend would not reconnect.
