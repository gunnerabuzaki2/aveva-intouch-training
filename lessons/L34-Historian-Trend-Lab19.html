Lesson 34: Historian Client Trend + Lab 19

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 6, Lab 19 (pages 429–438)

In this lab you will embed the Historian Client Trend .NET control into a symbol, initialize it with a script, and use the Tag Picker to compare level data from multiple mixers at runtime.

34.1 Lab 19 Overview

Reference: Module 6, Lab 19 (pages 429–430)

Lab Objective: Embed the Historian Client Trend .NET control in a symbol, use the AddServerEx() script function to initialize it, and use the Tag Picker to add tags at runtime.

The Historian Client Trend (aaTrendControl) is a powerful .NET control that queries tags from a Historian database and plots them on a graphical display. Unlike the built-in Multi Pens Trend, this control supports features like panning, zooming, XY scatter plots, and alarm limit highlighting — making it ideal for comparing data across multiple assets.

The completed Historian Client Trend display — multiple mixer levels plotted with alarm limit highlighting (page 429)

34.2 Create the TrendHistDisplay Symbol

Reference: Module 6, Lab 19 (pages 430–431)

On the Graphics tab in the Training toolset, create a symbol named TrendHistDisplay. From the Toolbox, search for aatrend and drag aaTrendControl onto the canvas. Name the embedded graphic Trend.

Right-click the canvas and select Custom Properties. Add a custom property named HistServer with the following configuration:

PropertyValue
Data TypeString
Default Value<HistorianNodeName> (your instructor will provide this)
VisibilityPrivate
Key Concept: The HistServer custom property stores the Historian node name as a private string. This makes it easy to update the server address without editing scripts — just change the property value.

34.3 Add the Initialization Script

Reference: Module 6, Lab 19 (pages 432–433)

Right-click the canvas and select Scripts. In the Predefined Scripts section, verify the trigger type is set to On Show. Enter the following script:

'Add a server using integrated security.
Trend.AddServerEx(HistServer,"","",false);

This script runs every time the symbol is displayed, connecting the Trend control to the Historian server using Windows integrated security. Click OK, save, and check in the graphic.

On Show script: Trend.AddServerEx connects to the Historian using integrated security (page 432)

34.4 Add a Navigation Button

Reference: Module 6, Lab 19 (pages 432–433)

Open SystemNavigation for editing. Duplicate the Historical Alarms button, place it to the right, and use Substitute Strings to change the text to "Historian Client Trend". Open the duplicate's Custom Properties and set the GraphicName default value to TrendHistDisplay. Save and check in.

SystemNavigation updated with the Historian Client Trend navigation button (page 432)

34.5 Test in Runtime

Reference: Module 6, Lab 19 (pages 434–437)

Switch to Runtime and click the Historian Client Trend navigation button. After a moment the trend display appears. In the Tags panel, select Level_001.PV and drag it to the chart. The chart displays the level data with alarm limit highlighting — orange for HiHi and LoLo, yellow for Hi and Lo.

Now add additional tags to compare mixer performance. Select and drag the following tags to the chart (hold Ctrl or Shift to select multiple):

  • Level_002.PV through Level_008.PV

The overlapping trend lines reveal the time differences for filling and draining each tank, including any latency in the process. You can drag additional tags, change the time range using the drop-down list (e.g., 1 minute), and toggle live/replay mode to see the chart scroll continuously.

Single tag trend: Level_001.PV with orange HiHi/LoLo and yellow Hi/Lo alarm highlights (page 434)
Multi-tag comparison: Level_002 through Level_008 overlapping trend lines reveal timing differences between mixers (page 435)
Time range selector with preset options from 1 minute to Previous Hour (page 436)
Live mode: the chart scrolls continuously, showing real-time mixer level data (page 436)

34.6 Knowledge Check

Knowledge Check

1. What script function initializes the aaTrendControl's connection to a Historian server, and what trigger type should be used?

Click to reveal answer

Trend.AddServerEx(HistServer,"","",false) with an On Show trigger. This runs the connection script every time the symbol is displayed at runtime.

2. Why is the HistServer custom property set to Private visibility?

Click to reveal answer

Private visibility means the property is hidden when the symbol is embedded in other graphics. Only the internal script needs to access the server name — operators don't need to see or change it.

3. What colors does the Historian Client Trend use to highlight alarm limits, and which alarms do they represent?

Click to reveal answer

Orange represents HiHi and LoLo alarm limits. Yellow represents Hi and Lo alarm limits. These highlights appear directly on the trend chart background.

4. How do you add multiple tags to the Historian Client Trend chart at runtime?

Click to reveal answer

In the Tags panel, select multiple tags by holding Ctrl or Shift, then drag them to the chart. The overlapping trend lines allow direct comparison of multiple assets.
