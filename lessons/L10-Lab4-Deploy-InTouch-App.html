Do Not Copy Lab 4 – Deploying an InTouch Application 2-47 AVEVA™ InTouch for System Platform 2023 Lab 4 – Deploying an InTouch Application Introduction In this lab, you will deploy a Managed InTouch application, and then view the application in runtime. Objectives Upon completion of this lab, you will be able to:  Create and configure a ViewEngine  Select Home windows for InTouch application start up  Deploy a Managed InTouch application  Open a deployed InTouch application in runtime

Do Not Copy 2-48 Module 2 – Getting Started AVEVA™ Training Create a ViewEngine First, you will create an instance of the $ViewEngine template. 1. In the IDE, on the Templates tab, in the System template toolset, right-click the $ViewEngine template and select New | Derived Template. 2. Name the derived template as $gViewEngine.

Do Not Copy Lab 4 – Deploying an InTouch Application 2-49 AVEVA™ InTouch for System Platform 2023 3. Drag $gViewEngine to the Training\Global template toolset. 4. In the Global template toolset, right-click $gViewEngine and select New | Instance. In the Deployment view, a new instance is created with the default name of gViewEngine_001.

Do Not Copy 2-50 Module 2 – Getting Started AVEVA™ Training 5. In the Deployment view, name gViewEngine_001 as ViewEngine1. 6. Assign ViewEngine1 to PROD_Platform1.

Do Not Copy Lab 4 – Deploying an InTouch Application 2-51 AVEVA™ InTouch for System Platform 2023 Create an Instance of the InTouch Application Now, you will create a new instance of the $MixerView template. 7. On the Templates tab, in the Training\ViewApps template toolset, create an instance of the $MixerView template. In the Deployment view, a new instance is created with the default name of MixerView_001. 8. Leave the default name, and then assign MixerView_001 to ViewEngine1.

Do Not Copy 2-52 Module 2 – Getting Started AVEVA™ Training 9. In the Model view, assign MixerView_001 and ViewEngine1 to the Sys area. Configure WindowViewer Home Windows Now, you will configure WindowViewer settings to set the home windows. At runtime, the home windows will open automatically on startup in WindowViewer. 10. On the Templates tab, in the Training\ViewApps toolset, double-click $MixerView to open WindowMaker. WindowMaker opens and displays Select windows to open. You will not open any windows at this time.

Do Not Copy Lab 4 – Deploying an InTouch Application 2-53 AVEVA™ InTouch for System Platform 2023 11. On the backstage menu, select Configure | WindowViewer. The WindowViewer settings appear. 12. In WindowViewer settings, select the Startup tab. Home windows appears.

Do Not Copy 2-54 Module 2 – Getting Started AVEVA™ Training 13. In Home windows, select the following windows:  AlarmDisplay  ContentDisplay  Logon  Navigation  OverviewDisplay 14. Click Save. A WindowMaker notification message appears, indicating WindowViewer configuration successfully updated.

Do Not Copy Lab 4 – Deploying an InTouch Application 2-55 AVEVA™ InTouch for System Platform 2023 15. On the backstage menu, select Exit to close WindowMaker and then check in $MixerView. Deploy the Managed InTouch Application Now, you will deploy ViewEngine1 and MixerView_001 to the production node. 16. In the IDE, in the Deployment view, right-click ViewEngine1 and select Deploy.

Do Not Copy 2-56 Module 2 – Getting Started AVEVA™ Training The Deploy dialog box appears. 17. Leave all default settings and click Deploy. The Deploying Objects dialog box shows progress of the deployment process. 18. When the progress shows Deploy complete, click Close.

Do Not Copy Lab 4 – Deploying an InTouch Application 2-57 AVEVA™ InTouch for System Platform 2023 As soon as the deployment completes, the icon for the MixerView instance displays an orange synchronization icon, which indicates the final synchronization of the application on the production node. Once this process completes, the icon changes to fully deployed. View the Deployed Application in WindowViewer Finally, you will navigate to the production node and open the deployed $MixerView application in runtime. 19. Switch to the production node. Note: Your instructor will provide the name of your production node. 20. From the Windows Start menu, open Application Manager. The AVEVA Application Manager opens.

Do Not Copy 2-58 Module 2 – Getting Started AVEVA™ Training 21. On the InTouch tab, in the $MixerView tile, click the WindowViewer button to open the application in runtime. The Application Manager closes and after a few moments, InTouch WindowViewer opens and the $MixerView application displays in runtime. Notice the home windows are opened and the animated clock appears in the Logon window. 22. Leave the application running on the production node and switch back to the engineering node.
