Lesson 10: Lab 4 – Deploying an InTouch Application

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 2, Lab 4 (pages 87–98)

In this lab, you will deploy a Managed InTouch application by creating and configuring a ViewEngine, selecting startup windows, deploying to a production node, and launching in runtime.

Lab Objectives:
  • Create and configure a ViewEngine
  • Select home windows for InTouch application startup
  • Deploy a Managed InTouch application
  • Open a deployed InTouch application in runtime

10.1 Create a ViewEngine

Reference: Module 2, Lab 4 (pages 88–90)

The ViewEngine is the object that hosts and executes InTouch applications. You need to create one before you can deploy and run your Managed InTouch application. First, derive a project-specific template from the base $ViewEngine template.

  1. In the IDE, on the Templates tab, expand TrainingGalaxy > System. Right-click the $ViewEngine template and select New > Derived Template.
  2. Name the derived template $gViewEngine.
Right-click $ViewEngine in the System toolset and select New > Derived Template (page 88)
The $gViewEngine derived template now appears under the System toolset (page 88)
  1. Drag $gViewEngine to the Training\Global template toolset.
  2. Right-click $gViewEngine in the Global toolset and select New > Instance. A new instance named gViewEngine_001 is created in the Deployment view.
Drag $gViewEngine to the Training > Global toolset for organization (page 89)
Create an instance of $gViewEngine — this creates gViewEngine_001 in the Deployment view (page 89)
  1. In the Deployment view, rename gViewEngine_001 to ViewEngine1.
  2. Drag ViewEngine1 to assign it to PROD_Platform1.
Rename the instance to ViewEngine1 in the Deployment view (page 90)
ViewEngine1 is now assigned to PROD_Platform1 in the deployment hierarchy (page 90)
Key Concept: The ViewEngine must be assigned to a WinPlatform object (like PROD_Platform1) before it can host and execute InTouchViewApp objects. The ViewEngine's scan rate determines the scan rate of all hosted InTouchViewApp objects.

10.2 Create an InTouch Application Instance

Reference: Module 2, Lab 4 (pages 91)

Now create an instance of your $MixerView template and assign it to the ViewEngine you just created.

  1. On the Templates tab, in the Training\ViewApps toolset, right-click $MixerView and select New > Instance. A new instance named MixerView_001 is created.
  2. Leave the default name and assign MixerView_001 to ViewEngine1 in the Deployment view.
Create an instance of $MixerView from the Training > ViewApps toolset (page 91)
Assign MixerView_001 to ViewEngine1 — drag from Unassigned Host to ViewEngine1 in the deployment tree (page 91)

10.3 Assign to Area and Configure Home Windows

Reference: Module 2, Lab 4 (pages 92–94)

Assign the objects to the appropriate area in the Galaxy Model view, then configure which windows open automatically when the application starts in WindowViewer.

  1. In the Model view, assign MixerView_001 and ViewEngine1 to the Sys area.
Before assignment — MixerView_001 and ViewEngine1 appear under Unassigned Area (page 92)
After assignment — both objects are now under the Sys area in the Model view (page 92)
  1. On the Templates tab, double-click $MixerView to open WindowMaker. You will not open any windows at this time — exit the backstage view.
WindowMaker opens to the backstage — you will not open any windows at this time (page 92)
  1. On the backstage menu, select Configure > WindowViewer.
  2. In WindowViewer settings, select the Startup tab.
Navigate to WindowViewer configuration and select the Startup tab (page 93)
  1. In the Home windows section, select all five windows: AlarmDisplay, ContentDisplay, Logon, Navigation, and OverviewDisplay.
  2. Click Save. A notification confirms the configuration was updated successfully.
Select all five windows as home windows in the Startup tab and click Save (page 94)
WindowMaker confirms that the WindowViewer configuration was successfully updated (page 94)
Key Concept: Home windows are the windows that open automatically when WindowViewer starts in production. You must configure these before deploying — otherwise, no windows will appear at runtime.

10.4 Deploy the Application

Reference: Module 2, Lab 4 (pages 95–96)

Exit WindowMaker, check in your changes, then deploy the ViewEngine and InTouchViewApp to the production node.

  1. On the backstage menu, select Exit to close WindowMaker. Check in $MixerView when prompted.
Exit WindowMaker from the backstage menu — check in $MixerView when prompted (page 95)
  1. In the IDE, in the Deployment view, right-click ViewEngine1 and select Deploy.
Right-click ViewEngine1 in the Deployment view and select Deploy (page 95)
  1. In the Deploy dialog, leave all default settings and click Deploy.
  2. When the progress shows "Deploy complete," click Close.
The Deploy dialog — leave defaults and click Deploy to deploy both ViewEngine1 and MixerView_001 (page 96)
Deployment progress dialog — wait for "Deploy complete" and click Close (page 96)
Tip: After deployment completes, the MixerView instance shows an orange synchronization icon indicating final sync on the production node. Once syncing finishes, the icon changes to the fully deployed state.

10.5 Launch in Runtime

Reference: Module 2, Lab 4 (pages 97–98)

Finally, navigate to the production node and open the deployed application in WindowViewer to verify it runs correctly.

  1. Switch to the production node (your instructor will provide the node name).
  2. From the Windows Start menu, open Application Manager.
The sync icon progression: orange (synchronizing) to blue (fully deployed) (page 97)
Application Manager shows the $MixerView application tile on the InTouch tab (page 97)
  1. On the InTouch tab, click the WindowViewer button on the $MixerView tile to open the application in runtime.

The Application Manager closes and after a few moments, InTouch WindowViewer opens with the $MixerView application in runtime. The home windows open automatically, and the animated clock appears in the Logon window — confirming that the deployment was successful.

Click the WindowViewer button on the $MixerView tile to launch the application in runtime (page 98)
The deployed $MixerView application running in WindowViewer with all five home windows displayed and the clock animating in the Logon window (page 98)
  1. Leave the application running on the production node and switch back to the engineering node.
Key Concept: The deployment process is now complete. The $MixerView application is running on the production node via the ViewEngine, displaying all configured home windows with live data.

Knowledge Check

1. Why do you create a derived template from $ViewEngine instead of using the base template directly?

Click to reveal answer

Creating a derived template (like $gViewEngine) allows you to customize ViewEngine settings for your specific project while preserving the base template. It follows the same derivation pattern used for all Galaxy objects — you derive templates for project-specific configurations.

2. What is the relationship between ViewEngine1 and MixerView_001 in the deployment hierarchy?

Click to reveal answer

MixerView_001 (the InTouchViewApp instance) is assigned to ViewEngine1 (the ViewEngine instance). The ViewEngine hosts and executes the InTouch application — its scan rate determines the scan rate of the InTouchViewApp.

3. Why is it important to configure home windows before deploying?

Click to reveal answer

Home windows are the windows that open automatically when WindowViewer starts in production. If you don't configure them, no windows will appear when the application launches on the production node — operators would see an empty runtime.

4. What do the different deployment icon states (orange vs. blue) indicate?

Click to reveal answer

The orange icon indicates that final synchronization of the application on the production node is still in progress. The blue icon indicates that deployment is fully complete and the application is ready to run.

5. How do you launch a deployed Managed InTouch application for the first time?

Click to reveal answer

On the production node, open AVEVA Application Manager from the Windows Start menu, go to the InTouch tab, and click the WindowViewer button on the application tile. This only needs to be done the first time — subsequent starts launch WindowViewer automatically.
