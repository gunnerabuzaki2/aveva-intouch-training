Lesson 10: Lab4 Deploy InTouch App

Module 2 — Getting Started | Pages 87–98

Introduction

In this lab, you will deploy a Managed InTouch application, and then view the application in

runtime.

Objectives

Upon completion of this lab, you will be able to:



Create and configure a ViewEngine



Select Home windows for InTouch application start up



Deploy a Managed InTouch application



Open a deployed InTouch application in runtime

Create a ViewEngine

First, you will create an instance of the $ViewEngine template.

1.

In the IDE, on the Templates tab, in the System template toolset, right-click the $ViewEngine

template and select New | Derived Template.

2.

Name the derived template as $gViewEngine.

System Platform IDE Templates tree with right-click context menu on $ViewEngine showing New submenu with Derived Template option
Templates tree showing the newly created $gViewEngine derived template listed under the System folder, highlighted in blue

3.

Drag $gViewEngine to the Training\Global template toolset.

4.

In the Global template toolset, right-click $gViewEngine and select New | Instance.

In the Deployment view, a new instance is created with the default name of

gViewEngine_001.

Templates tree expanded to show Training folder contents, with right-click context menu on $gViewEngine showing New > Instance option
Deployment view showing the newly created gViewEngine_001 instance under Unassigned Host, alongside GRPlatform and AppEngine1 under PROD_Platform1

5.

In the Deployment view, name gViewEngine_001 as ViewEngine1.

6.

Assign ViewEngine1 to PROD_Platform1.

Deployment view showing ViewEngine1 renamed and GRPlatform with AppEngine1 and all deployed objects under PROD_Platform1
Deployment tree with ViewEngine1 expanded under PROD_Platform1 showing all deployed objects including Line1, Packaging, Plant, ProdPLC1, and Sys

Create an Instance of the InTouch Application

Now, you will create a new instance of the $MixerView template.

7.

On the Templates tab, in the Training\ViewApps template toolset, create an instance of the

$MixerView template.

In the Deployment view, a new instance is created with the default name of MixerView_001.

8.

Leave the default name, and then assign MixerView_001 to ViewEngine1.

Right-click context menu on $MixerView template in the Training ViewApps folder showing options including New > Instance and Derived Template
Side-by-side deployment views showing MixerView_001 moved from Unassigned Host to ViewEngine1 under PROD_Platform1 (before and after)

9.

In the Model view, assign MixerView_001 and ViewEngine1 to the Sys area.

Configure WindowViewer Home Windows

Now, you will configure WindowViewer settings to set the home windows. At runtime, the home

windows will open automatically on startup in WindowViewer.

10. On the Templates tab, in the Training\ViewApps toolset, double-click $MixerView to open

WindowMaker.

WindowMaker opens and displays Select windows to open.

You will not open any windows at this time.

Model view showing MixerView_001 and ViewEngine1 under Unassigned Area, before assignment to the Sys area
InTouch WindowMaker Open dialog displaying available windows as thumbnails: AlarmDisplay, ContentDisplay, Logon, Navigation, and OverviewDisplay

11. On the backstage menu, select Configure | WindowViewer.

The WindowViewer settings appear.

12. In WindowViewer settings, select the Startup tab.

Home windows appears.

WindowViewer configuration with Preferences tab active, showing Startup settings including minimum access level 9999 and inactivity timeout fields
WindowViewer Startup tab displaying Home windows section with five window thumbnails and Save button in bottom right

13. In Home windows, select the following windows:



AlarmDisplay



ContentDisplay



Logon



Navigation



OverviewDisplay

14. Click Save.

A WindowMaker notification message appears, indicating WindowViewer configuration

successfully updated.

WindowViewer Startup tab with all five home windows (AlarmDisplay, ContentDisplay, Logon, Navigation, OverviewDisplay) selected with Save button
WindowMaker notification dialog confirming WindowViewer configuration successfully updated with information icon

15. On the backstage menu, select Exit to close WindowMaker and then check in $MixerView.

Deploy the Managed InTouch Application

Now, you will deploy ViewEngine1 and MixerView_001 to the production node.

16. In the IDE, in the Deployment view, right-click ViewEngine1 and select Deploy.

Configure backstage menu with WindowViewer option highlighted among other configuration categories
Deployment view with $MixerView selected and right-click context menu showing Deploy option highlighted

The Deploy dialog box appears.

17. Leave all default settings and click Deploy.

The Deploying Objects dialog box shows progress of the deployment process.

18. When the progress shows Deploy complete, click Close.

Deploy 2 objects dialog box with General tab showing Cascade deploy checked, Skip selected for deployed objects, and On scan for initial state
Deploying Objects dialog showing Deploy complete status with log details of ViewEngine1 and MixerView_01 deployment to PROD_Platform1

As soon as the deployment completes, the icon for the MixerView instance displays an orange

synchronization icon, which indicates the final synchronization of the application on the

production node. Once this process completes, the icon changes to fully deployed.

View the Deployed Application in WindowViewer

Finally, you will navigate to the production node and open the deployed $MixerView application in

runtime.

19. Switch to the production node.

Note:

Your instructor will provide the name of your production node.

20. From the Windows Start menu, open Application Manager.

The AVEVA Application Manager opens.

Deployment view showing ViewEngine1 expanded with MixerView_001 displaying an orange synchronization icon during final deployment sync
AVEVA Application Manager showing the InTouch tab with $MixerView application tile displaying Managed 1920x975 status

21. On the InTouch tab, in the $MixerView tile, click the WindowViewer button to open the

application in runtime.

The Application Manager closes and after a few moments, InTouch WindowViewer opens and

the $MixerView application displays in runtime. Notice the home windows are opened and the

animated clock appears in the Logon window.

22. Leave the application running on the production node and switch back to the engineering

node.

AVEVA Application Manager with $MixerView selected and WindowViewer button highlighted in the toolbar ribbon
InTouch WindowViewer runtime displaying the deployed $MixerView application with Logon window, analog clock, Navigation panel, and ContentDisplay area
