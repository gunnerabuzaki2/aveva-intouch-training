Lesson 25: Lab 13 – Building an Alarm Aggregation Overview

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 5 – Alarms and Events Visualization, Lab 13 (pages 321–340)

In this lab, you will build an overview graphic that displays alarm aggregation for the Production area using SA_AlarmSeverity symbols. You will also add alarm severity indicators to navigation buttons and learn how to control alarm modes from the aggregation panel.

25.1 Lab Overview

Lab Objectives:
  • Use the SA_AlarmSeverity symbol to display alarm information and change alarm modes
  • Add alarm aggregation information to navigation buttons
  • Build a hierarchical alarm aggregation topology (Production → Lines → Mixers)

In the previous lesson, you created AlarmSeverityLabels and AlarmSeverityPanel symbols in the $gArea and $Mixer templates. In this lab, you will use those symbols to build a complete alarm aggregation overview that shows a hierarchy of alarm severity counts across the entire plant.

25.2 Repositioning and Resizing the Canvas

Reference: Module 5, Lab 13 (pages 321–322)

First, you will clean up the alarm aggregation canvas by repositioning all elements and setting a fixed canvas size.

  1. On a blank area of the canvas, right-click and select Select All.
  2. Move the selected elements to the left side of the canvas.
  3. Click the canvas to deselect all elements and show canvas properties.
  4. On the Properties tab, set Size to Fixed. The canvas now displays selection handles for resizing.
  5. Resize the canvas to fit the graphics, ensuring white space on all sides.
  6. Save and close, then check in the graphic.
The alarm aggregation hierarchy with all SA_AlarmSeverity panels connected by lines, shown in the Elements panel (page 321)
Canvas resized to fixed size, fitting all aggregation panels with appropriate white space (page 322)

25.3 Adding a Navigation Button for Alarm Overview

Reference: Module 5, Lab 13 (pages 323–325)

Next, you will create a navigation button that opens the AlarmAggregationOverview graphic in a popup window.

  1. On the Graphics tab, duplicate NavigationButton, name the duplicate NavigationButton2, and open it for editing.
  2. Right-click the canvas and select Custom Properties.
  3. Change the WindowModal and WindowResizable custom properties to True.
  4. Click OK to close Edit Custom Properties.
  5. Double-click the button to open Edit Animations.
  6. In Animations, ensure Element Style is selected, then click Remove Animation. Confirm with Yes.
  7. Remove the Disable animation as well. Only Action Scripts should remain.
  8. Click OK to close Edit Animations.
  9. Save and close, then check in the graphic.
Configuring NavigationButton2 with WindowModal and WindowResizable set to True (page 323)
Removing the Element Style animation from NavigationButton2 using the minus button (page 323)

Add the Button to SystemNavigation

  1. Open SystemNavigation for editing.
  2. On the Toolbox tab, search for "button2" and drag NavigationButton2 to the right of the Line 2 button.
  3. Change the button text to Alarm Overview.
  4. Right-click the Alarm Overview button and select Custom Properties.
  5. For GraphicName, enter AlarmAggregationOverview in Default Value. Leave WindowName blank — this makes the graphic appear in a popup window.
  6. Click OK to close Edit Custom Properties.
The Alarm Overview button added to the SystemNavigation bar alongside the existing navigation buttons (page 324)

25.4 Enhancing Navigation Buttons with Alarm Severity

Reference: Module 5, Lab 13 (page 325)

To give operators an at-a-glance view of alarm counts on the navigation buttons, you will embed AlarmSeverityLabels into the Line 1 and Line 2 buttons.

  1. From Instances, embed Line1\AlarmSeverityLabels at the top and center of the Line 1 button. Adjust the size to fit the button width.
  2. Similarly, embed Line2\AlarmSeverityLabels at the top and center of the Line 2 button.
  3. Save and close, then check in the graphic.
Enhanced navigation buttons showing embedded alarm severity indicators for Line 1 and Line 2 (page 325)

25.5 Testing at Runtime

Reference: Module 5, Lab 13 (pages 326–329)

  1. Click Runtime.
  2. In the Navigation window, observe that the Line 1 and Line 2 buttons now display alarm counts for each severity level (Critical, High, Medium).
  3. Click the Alarm Overview button to open the AlarmAggregationOverview popup window.
Runtime navigation bar showing live alarm severity counts on the Line 1 and Line 2 buttons (page 326)
The AlarmAggregationOverview popup showing the complete hierarchical alarm topology with rolled-up severity counts (page 326)
Key Concept: Alarm counts roll up from children to parents — for example, Line1 Critical 12 + Line2 Critical 12 = Production Critical 24. This hierarchical aggregation allows operators to quickly identify which area has the most severe alarm conditions.

Controlling Alarm Modes from the Aggregation Panel

  1. Behind the popup window, try clicking the Mixers button — it will not respond because the window is modal and must be closed first.
  2. In the Production alarm panel, click the up-arrow Command Panel button next to the Mode label.
  3. Select Disable from the dropdown, then click Apply.
The Production alarm panel showing the Command Panel control for changing alarm modes (page 327)
The mode dropdown showing all available options: Enable, Disable, Silence, and Inhibit (page 327)
  1. Observe that all alarm panels in the popup window now indicate Disable. Disabling alarms for the Production area automatically disables alarms for all child objects — this is the inheritance behavior of alarm aggregation.
All alarm panels show Disable mode — changing the parent Production area's mode cascades to all child objects (page 328)
Warning: Disabling alarms at a parent area level will disable alarms for all objects in the hierarchy beneath it. Only use this feature when you intend to suppress alarms for an entire production area.
  1. In the Production alarm panel, click the up-arrow next to Mode, select Enable, and click Apply.
  2. Observe that all alarm panels are now re-enabled.
  3. Close the AlarmAggregationOverview window.
  4. Click Development! to return to WindowMaker.

Knowledge Check

1. Why must the WindowName custom property be left blank on the Alarm Overview navigation button?

Click to reveal answer

Leaving WindowName blank causes the graphic specified in GraphicName to appear in a popup window rather than replacing the current window. This is how popup windows are created in InTouch navigation.

2. What happens when you disable alarms at the Production area level in the aggregation panel?

Click to reveal answer

Disabling alarms at the Production level automatically disables alarms for all child objects in the hierarchy (Line1, Line2, and all individual mixers). This is the hierarchical inheritance behavior of alarm aggregation.

3. What two animations must you remove from NavigationButton2 before using it as the Alarm Overview button?

Click to reveal answer

You must remove the Element Style animation and the Disable animation, leaving only the Action Scripts animation. This ensures the button simply opens the popup window without additional visual state changes.

4. How are alarm severity counts aggregated in the hierarchical overview?

Click to reveal answer

Counts roll up from children to parents — for example, Line1's Critical count plus Line2's Critical count equals the Production-level Critical count. This allows operators to see at a glance where the most severe alarms are concentrated.
