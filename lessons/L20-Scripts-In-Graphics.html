Lesson 20: Scripts in Graphics

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 3 — Symbols, Section 7 (pages 249–254)

This lesson explains how scripts work within InTouch graphics, covering execution types, triggers, the ShowGraphic() function, and how to associate scripts with symbols. It also introduces Lab 10 — Creating a Navigation Symbol.

20.1 Associating Scripts with Graphics

Reference: Module 3, Section 7 (pages 249–250)

In the previous labs you used animation links to connect symbols to tag values. Scripts take this further: they let you write programmatic instructions that run when specific events occur or conditions change. A script associated with a graphic can animate the graphic, modify its elements, or perform complex logic while the application is running.

To associate a script with a graphic, you select the graphic in the Graphic Editor and then choose the expression or reference whose changes will trigger the script. There are two categories of symbol scripts:

Script TypeBehaviorExample Triggers
PredefinedRuns based on the symbol's lifecycle statusOn Show, While Showing, On Hide
NamedRuns when an expression or reference changes stateDataChange on a tag value, true/false transitions
Key Concept: Predefined scripts are analogous to window scripts — they respond to the symbol's visibility state. Named scripts are more flexible: they fire whenever their trigger expression changes, including transitions between true and false, or changes in data quality.

20.2 Execution Types and Triggers

Reference: Module 3, Section 7 (pages 249–250)

Scripts execute in one of two modes: synchronous or asynchronous. In synchronous mode (the default), animation and tag value updating pauses while the script runs, then resumes afterward. In asynchronous mode, animation continues during script execution — but asynchronous scripts only support the Execute trigger type.

The available execution type triggers are:

TriggerWhen It Fires
StartupOnce when the object enters scan
On ScanEach AppEngine scan cycle while the object is on scan
ExecuteOn demand (used for asynchronous scripts)
Off ScanOnce when the object leaves scan
ShutdownOnce when the object shuts down

An important constraint: scripts cannot trigger faster than the AppEngine scan period. If your script is associated with an AppEngine that scans every 500 ms, the script cannot run more frequently than that. Also, the execution order of scripts is locked at each stage of the template hierarchy — a derived template inherits its parent's script block as an ordered unit.

Key Concept: Script text has no length limit. You can save a script with syntax errors, but the object cannot be deployed until all errors are corrected. Use the built-in validation features to catch issues before deployment.

20.3 The ShowGraphic() Function

Reference: Module 3, Section 7 (pages 250–252)

The ShowGraphic() function is one of the most powerful scripting tools in InTouch. It allows you to display any Galaxy symbol as a popup window from within a script, without having to embed the symbol in the application first. When all Galaxy graphics are associated with an InTouchViewApp template, the deployed application can handle ShowGraphic requests for any symbol in the Galaxy.

The basic syntax is:

Dim graphicInfo as aaGraphic.GraphicInfo;
graphicInfo.Identity = "<Identity>";
graphicInfo.GraphicName = "<SymbolName>";
ShowGraphic( graphicInfo );

To close a graphic displayed by ShowGraphic, use:

HideGraphic(string identity);

You can configure many properties of the popup window through the GraphicInfo object, including:

  • Whether the window has a title bar (HasTitleBar)
  • Whether it can be resized (Resizable)
  • Whether it is modal or modeless (WindowType)
  • Its position and size on screen
  • The OwningObject to pass for relative tag resolution
  • Custom property values for the target symbol
Important: ShowGraphic can be used in action scripts, named scripts, and predefined scripts. However, although the system allows placing it in server scripts (Startup, On Scan, Off Scan, Shutdown, Execute), the function cannot actually run at runtime from those contexts.

20.4 Runtime Behavior and Identity Management

Reference: Module 3, Section 7 (pages 251–252)

Graphics displayed by ShowGraphic behave like ShowSymbol animation popups — they float on top of all InTouch windows (except other InTouch popup windows). There is no parent/child relationship between the window that opened the graphic and the graphic itself. This means you can manage the popup from anywhere in the application.

Some key runtime behaviors to understand:

  • Same Identity: If you try to open a ShowGraphic popup with the same Identity as one that is already displayed, the request is silently blocked — no error or warning is logged.
  • Closing: You can close a ShowGraphic popup by calling HideGraphic or HideSelf, clicking the Close Window button (if configured), or closing WindowViewer. You cannot close it by closing the InTouch window that opened it.
  • Caching: Enabling in-memory graphics caching in WindowViewer keeps ShowGraphic popups cached. The system tracks popup age and removes the oldest when the user-defined memory limit is exceeded.
  • Modal behavior: If a graphic is modal, calling ShowGraphic again with the same identity cannot change it to modeless.
Key Concept: The OwningObject in a ShowGraphic script resolves only relative references — it does not affect absolute references. The relative reference is resolved by the object that hosts the script, not by the graphic definition.

20.5 Types of Scripts

Reference: Module 3, Section 7 (pages 248, 252)

InTouch provides seven categories of scripts, each defined by what causes it to run:

Script TypeRuns When
ApplicationWindowViewer starts, stops, or while running
WindowA window opens, closes, or remains open
KeyA key or key combination is pressed or released
ConditionA condition becomes true, false, or transitions
Data ChangeA tag value or expression changes
ActionAn operator clicks on a graphic object
ActiveX EventAn ActiveX event occurs (e.g., clicking a control)

QuickScript, the InTouch scripting language, supports conditional branching, loops, local variables, and a rich library of built-in functions (mathematical, trigonometric, string manipulation, and more). You can also use OLE objects and ActiveX controls within scripts for extended functionality.

Key Concept: A QuickFunction is a reusable function written in QuickScript and stored in the QuickFunction library. Once created, it can be called from any script or from animation link expressions, promoting code reuse across your application.

Knowledge Check

1. What is the difference between a predefined script and a named script on a graphic symbol?

Click to reveal answer

A predefined script runs based on the symbol's lifecycle status (On Show, While Showing, On Hide). A named script runs when an expression or reference associated with it changes state, such as a tag value changing or a boolean transitioning between true and false.

2. Why can't scripts trigger faster than the AppEngine scan period?

Click to reveal answer

Scripts execute within the AppEngine's scan cycle. The AppEngine can only evaluate and execute scripts during its scan, so a script cannot run more frequently than the scan interval that governs it.

3. What happens if you call ShowGraphic() with an Identity that already has a displayed popup?

Click to reveal answer

The ShowGraphic request is silently blocked. No error or warning message appears in the logger. The existing popup remains displayed unchanged.

4. Can the ShowGraphic() function be used in a Startup server script?

Click to reveal answer

While the system allows you to place ShowGraphic() in a Startup script without reporting an error, the function cannot actually execute at runtime from server scripts like Startup, On Scan, Off Scan, Shutdown, or Execute.
