Lesson 12: Building a Process Overview

Module 3 — Symbols | Pages 113–144

Lab Objectives: By the end of this lab you will be able to create graphic toolsets for organizing items on the Graphics tab, build a new symbol in the Graphic Editor, embed Situational Awareness dashboard graphics, configure wizard options and custom properties, work with string substitution, and set a fixed canvas size.

In a typical InTouch project the Graphics tab can quickly become crowded with a mix of out-of-the-box library graphics, third-party controls, and custom symbols you create during development. Without a clear organizational strategy, finding the right graphic becomes a time-consuming chore. This lab teaches you how to impose order on that chaos by creating graphic toolsets — named folders that group related graphics together — and then walks you through building a Process Overview symbol that consumes materials and produced products data for eight mixers using dashboard charts from the Situational Awareness library.

The final result: consumed materials and produced products bar charts displaying live batch production data for mixers M100 through M800.

Organizing Graphics with Toolsets

Before you create any new symbols, the first step is to establish a clean folder structure on the Graphics tab. In the IDE, right-click TrainingGalaxy on the Graphics tab and choose New > Graphic Toolset. Create two toolsets to house out-of-the-box items: one named Apps and another named .NET Controls.

Right-clicking TrainingGalaxy reveals the New submenu where you can create a Graphic Toolset.
After creating toolsets, the Graphics panel tree shows a clean hierarchy under TrainingGalaxy.
The .NET Controls toolset is ready to receive client control graphics.

Now drag the following application graphics into the Apps toolset: AlarmApp, ContentPresenterApp, DocViewerApp, GraphicRepeaterApp, HamburgerApp, HistoricalTrendApp, ImageViewerApp, InSightApp, MapApp, NavigationApp, PDFViewerApp, SpreadsheetViewerApp, TitleBarApp, and WWWebAppControls. These are pre-built InTouch application components that provide ready-made functionality like alarm displays, document viewers, and navigation controls.

Both toolsets are populated — Apps holds application components, while .NET Controls holds client-side trend and alarm controls.

Next, drag the aaTrendControl, AlarmClient, and TrendClient .NET client controls into the .NET Controls toolset. Collapse both toolsets when you are finished, and then create a third toolset named Training — this is where your custom course symbols will live.

The .NET Controls toolset with its three client controls.
The Training toolset is created and ready for your custom symbols.
What is a Graphic Toolset? A graphic toolset is a logical folder on the Graphics tab that groups symbols and library graphics together. Toolsets do not affect runtime behavior — they exist purely for developer organization. Think of them as labeled binders that keep your graphics tidy as the project grows.

Creating the Process Overview Symbol

With the toolsets in place, you now create the symbol that will become your process overview. Right-click the Training toolset and choose New > Symbol. Name it ProcessOverview and double-click it to open it in the Graphic Editor.

Right-clicking the Training toolset to create a new symbol.
The ProcessOverview symbol now appears under the Training toolset.

When the Graphic Editor opens, maximize it to give yourself enough room for the two dashboard charts you are about to embed. The canvas starts empty — you will populate it with a stacked column chart and a horizontal bar chart from the Situational Awareness library.

The Graphic Editor opens with a blank canvas, ready for the dashboard graphics.

Embedding the SA_StackedColumnChart

The Situational Awareness library includes a collection of pre-built dashboard tools — charts, gauges, and meters — that you can embed directly into your symbols. On the right side of the Graphic Editor, select the Toolbox tab to reveal the same toolset hierarchy you see on the Graphics tab in the IDE.

The Toolbox tab mirrors the Graphics tab hierarchy, providing drag-and-drop access to library graphics.

Expand Situational Awareness Library and scroll down to Dashboard Tools. The preview area at the bottom shows the available chart graphics — SA_BarChart, SA_BoxChart, SA_BulletGraph, SA_ColumnChart, SA_PieChart, and SA_StackedColumnChart. Select SA_StackedColumnChart and drag it onto the canvas. This chart will display the consumed materials for each of your eight mixers.

Configuring Wizard Options and Custom Properties

Once the chart is on the canvas, you need to configure it to match your plant layout. On the Properties tab, change the Name to IngredientsConsumed. Then, under Wizard Options, set NumberOfBars to Eight — one bar per mixer. The other wizard options (ShowTitle, ShowAxisDescription, ShowValueLabel, ShowLegend) should all remain set to True.

The Properties panel shows the name and wizard options for the stacked column chart.
NumberOfBars is changed from Ten to Eight to match the eight mixers in the plant.

Next, right-click the IngredientsConsumed graphic on the canvas and select Custom Properties. The Edit Custom Properties dialog appears with a list of bar data points (Bar01A through Bar08D). Each bar has up to four series (A, B, C, D), but you only need the A and B series — the C and D entries can be left at their default of 0.

Right-clicking the chart reveals the Substitute submenu for string and property editing.
Pro Tip: Instead of manually typing each reference, double-click the Default Value field to open the Galaxy Browser. This is faster than clicking the Browse button and ensures you select the correct attribute without typos.

For each bar, you map the data point to a mixer's totalizer attribute. For example, Bar01A maps to Mixer100.Totalizer1.PV and Bar01B maps to Mixer100.Totalizer2.PV. In the Galaxy Browser, enable Show Hierarchical Name to see the full object path, then navigate to the mixer instance and select the desired attribute.

The Edit Custom Properties dialog lets you map each bar data point to a live attribute.

Substituting Strings for Meaningful Labels

The SA_StackedColumnChart ships with generic placeholder labels — "A", "B1", "Chart Heading", and so on. You need to replace these with meaningful names that reflect your plant's mixers and ingredients. Right-click the IngredientsConsumed graphic and select Substitute > Substitute Strings.

The Substitute Strings option lets you rename all placeholder text in the embedded graphic.
Warning — Duplicate Strings: When substituting strings, make sure each replacement string is unique. If you enter two different original strings that map to the same new string, the Substitute Strings dialog will consolidate them. Once consolidated, you can never un-consolidate them — you would have to delete the entire graphic and lose all your wizard options, custom properties, size, position, graphic name, and strings.

Substitute the strings as follows: "A" becomes "Ingr1", "B" becomes "Ingr2", "B1" through "B8" become "M100" through "M800", "C" becomes "N/A-C", "D" becomes "N/A-D", "Chart Heading" becomes "Consumed Materials", "Horizontal Axis Description" becomes "Mixers", "Short Description" becomes "Consumed materials quantity of current batch", and "Vertical Axis Description" becomes "Materials Quantity".

The Substitute Strings dialog showing all 13 replacements for the consumed materials chart.
After substitution, the chart displays meaningful mixer names and axis labels.

Setting a Fixed Canvas Size

By default, a symbol's canvas can grow or shrink dynamically. For a dashboard-style overview like this one, you often want a fixed resolution so that the symbol occupies a predictable footprint on the window. Click the canvas to set focus on it, then on the Properties tab set Size to Fixed. Drag the bottom-right handle of the canvas to encompass both charts with some white space around them.

Why Fixed Canvas Size? Setting the canvas to a fixed size creates a bounding box. All white space outside this box is cropped at runtime, giving you precise control over the height and width of the graphic as it appears on an InTouch window. This is especially useful for dashboard symbols that need to fit into specific layout areas.

Now that your ProcessOverview symbol is complete, you need to place it inside an actual InTouch window. This opens WindowMaker, where you can select the windows you want to edit — choose AlarmDisplay, ContentDisplay, Logon, Navigation, and OverviewDisplay.

Testing in Runtime

Click Runtime to switch from development to runtime mode. You should see both charts displaying live data from the mixers. The Consumed Materials stacked column chart shows ingredient quantities, while the Produced Products horizontal bar chart shows output quantities.

Observe the timing: For any given mixer, you will not see any produced products until a few seconds after the consumed materials are at their highest level. This reflects the real-world batch processing sequence — ingredients are consumed first, then products are produced.
The Process Overview in runtime, showing live batch production data for all eight mixers.

When you are satisfied that the graphics display correctly, click Development! to return to WindowMaker.

Summary

In this lab, you learned how to organize your Graphics tab using toolsets, create a new symbol from scratch, embed Situational Awareness dashboard charts using two different workflows (Toolbox drag-and-drop and context-menu embedding), configure wizard options to customize chart appearance, bind custom properties to live Galaxy attributes, substitute strings to replace placeholder labels with meaningful names, and set a fixed canvas size for predictable runtime dimensions. These skills form the foundation for building dashboard-style process overviews that give operators a clear, at-a-glance view of plant status.

Quiz — Test Your Understanding

Q1: What is the purpose of a Graphic Toolset on the Graphics tab?

A Graphic Toolset is a logical folder that groups related symbols and library graphics together for developer organization. It has no effect on runtime behavior — it simply keeps the Graphics tab tidy as projects grow.

Q2: What are two different methods for embedding a library graphic into a symbol?

Method 1: Select the Toolbox tab in the Graphic Editor, navigate to the desired library graphic, and drag it onto the canvas. Method 2: Right-click a blank area on the canvas and select Embed Industrial Graphic, then browse the Galaxy Browser to find and select the graphic.

Q3: Why should you avoid entering duplicate replacement strings in the Substitute Strings dialog?

If two different original strings map to the same replacement string, the Substitute Strings dialog consolidates them. Once consolidated, you cannot undo this — you would have to delete the entire graphic and lose all wizard options, custom properties, size, position, and other settings.

Q4: What does setting the canvas Size to "Fixed" do?

Setting the canvas to Fixed creates a bounding box. All white space outside the bounding box is cropped at runtime, giving you precise control over the exact height and width of the graphic as it appears on an InTouch window.

Q5: In the Process Overview, why do produced products appear after consumed materials for a given mixer?

This reflects the real-world batch processing sequence: ingredients are consumed first during mixing, and only after the mixing process is underway or complete do produced products begin to accumulate. The timing in the graphic mirrors the actual plant process.
