Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 3 (pages 113–144)

In this lab you will create a ProcessOverview symbol using Situational Awareness dashboard charts, configure it with live data references, and embed it in the OverviewDisplay window.

Lab Objectives: Upon completion, you will be able to create graphic toolsets, create and edit a symbol in the Graphic Editor, embed SA Library graphics via the Toolbox tab and the Galaxy Browser, configure Wizard Options and Custom Properties, set up string and reference substitutions, and configure a fixed canvas size.

12.8 Linked Symbols and Relative References (Preview)

Reference: Module 3, Section 3 (pages 143–144)

Before moving to the next lesson, it is worth previewing a concept you will use extensively: the relationship between graphics and automation objects. When adding symbols to an automation object, you can either create symbols within the object and embed Graphics tab symbols, or use the linked symbols feature to associate external symbols directly.

From the Content pane on the Attributes tab in the Object Editor, the Add button creates a new internal symbol, while the Link Content button opens the Galaxy Browser to select a symbol from the Graphics tab. You can select a symbol from the list and click Edit Content to open it in the Graphic Editor.

Key Concept — Relative References: You can use relative references when configuring graphics. These are references that go up the object hierarchy to parent objects. Valid relative references include: Me, MyArea, MyContainer, MyEngine, MyHost, and MyPlatform. They are resolved at runtime when the graphic is displayed.
The Object Editor Content pane with Add and Link Content buttons for symbol management (page 144)

Knowledge Check

1. Why is it important to enter unique replacement strings in the Substitute Strings dialog rather than reusing the same string?

Click to reveal answer

If duplicate strings are entered, the Substitute Strings dialog consolidates them on subsequent openings, making individual strings uneditable. This would require deleting the graphic and losing all Wizard Options, Custom Properties, position, size, name, and string configurations.

2. What is the difference between embedding a graphic using the Toolbox tab drag-and-drop method versus the Galaxy Browser method?

Click to reveal answer

The Toolbox tab method involves finding the symbol in the Toolbox panel and dragging it onto the canvas. The Galaxy Browser method involves right-clicking the canvas and selecting Embed Industrial Graphic, which opens the Galaxy Browser for selection. Both achieve the same result — a linked copy of the original symbol.

3. Why do the custom properties for the IngredientsConsumed chart use Totalizer1 and Totalizer2 while ProducedProducts uses Totalizer3?

Click to reveal answer

Totalizer1 and Totalizer2 track the consumed ingredient quantities (the inputs), while Totalizer3 tracks the produced product quantity (the output) for each mixer. This separation reflects the actual data model where inputs and outputs are tracked by different totalizer attributes.

4. What happens at runtime when the canvas Size is set to "Fixed" versus "Auto"?

Click to reveal answer

Fixed uses the FixedWidth and FixedHeight values to define the exact pixel dimensions at runtime, with white space cropped to the bounding box. Auto dynamically sizes the graphic by cropping all white space outside runtime elements.

5. When you drag ProcessOverview into OverviewDisplay from the Training toolset, why must you select the Instance version rather than the Template version from the Galaxy Browser?

Click to reveal answer

Template versions use Me. relative references that are not resolved at runtime — they only work when displayed through their parent object. Instance versions have the Me. references replaced with actual I/O links to specific mixer objects, so they display live data at runtime.
