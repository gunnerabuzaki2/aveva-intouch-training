Lesson 23: Widgets Overview + Lab 11 – Creating a Carousel Dashboard

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 4 – Widgets (pages 281–292)

This lesson introduces HTML5 widgets in InTouch and walks you through building a Carousel dashboard that cycles through process graphics automatically at runtime.

23.1 What Are HTML5 Widgets?

Reference: Module 4, Section 1 – Widgets Overview (pages 283–284)

Widgets are small, self-contained code blocks built on standard web technologies — HTML5, CSS, and JavaScript — that extend InTouch's functionality without altering any of its core features. Think of them as miniature web applications that InTouch can host, both at design time in the Graphic Editor and at runtime in WindowViewer.

The power of this approach is that InTouch acts as a container for these widgets. Operators see a consistent user interface with predictable placement, regardless of the widget's internal complexity. Common use cases include embedding social media feeds, weather displays, RSS readers, podcasts, or custom plant-floor dashboards on large monitors throughout a facility.

Key Concept: Widgets are imported into InTouch from the IDE using Galaxy → Import → Visualization → Web Widgets. The file format is a Custom Widget Package (.cwp), which bundles HTML5, CSS, and JavaScript into a single portable package.

Default Widgets

InTouch ships with six default widgets, available under the Widgets folder on the Graphics tab:

WidgetPurpose
CarouselCycles through multiple Industrial Graphics automatically or via user navigation
DatagridDisplays tabular data in a grid format
MapAppEmbeds map-based visualizations
QRCode_ScannerScans QR codes from a camera input
TeamworkCollaboration and communication widget
Web_BrowserEmbeds a web browser within the HMI

After importing a widget, it becomes available in WindowViewer and its behavior can be controlled through design-time properties in the Graphic Editor.

23.2 The Carousel Widget

Reference: Module 4, Section 1 (page 284)

The Carousel widget is the most commonly used default widget. It allows you to cycle through a sequence of Industrial Graphics like a slideshow, without requiring any user input. Operators can also manually navigate forward and backward through the carousel. This makes it ideal for displaying dashboards, alerts, or alarm summaries on large plant-floor monitors where a single screen cannot show everything.

Carousel Properties

In addition to standard graphic properties, the Carousel exposes a set of widget-specific properties under Widget Properties in the Graphic Editor:

PropertyDescriptionDefault
AutoplayWhen True, the carousel starts cycling automatically on load. When False, the user must click to start.True
BackgroundColorThe background color of the widget area.#FFFFFF
GraphicNamesA comma-separated list of Industrial Graphics to display in the carousel.(empty)
IntervalTime delay in milliseconds between automatic transitions.5000
KeyboardWhen True, the carousel responds to keyboard arrow inputs for navigation.True
LoopWhen True, the carousel cycles continuously. When False, it stops after one pass.True
PauseWhen True, the carousel pauses when the mouse hovers over it or during a touch event. Cycling resumes when the pointer moves away.True
Key Concept: The GraphicNames property is the most critical setting — it determines which Industrial Graphics the carousel displays at runtime. Each name must match an existing graphic in your Galaxy.

23.3 Lab 11 – Creating a Carousel Dashboard

Reference: Module 4, Lab 11 (pages 285–292)

Lab Objectives:
  • Use widgets within InTouch graphics
  • Display multiple Industrial Graphics in the Carousel widget
  • Test carousel behavior at runtime (autoplay, pause, manual navigation)

Step 1: Create the Dashboard Symbol

First, you will create a Dashboard symbol and embed the Carousel widget into it.

  1. In the IDE, on the Graphics tab, in the Training toolset, create a new symbol named Dashboard and open it for editing.
  2. From the Toolbox tab, drag the Widgets\Carousel component onto the canvas.
The Carousel widget placed on the Dashboard symbol canvas, with the Widgets folder visible in the Toolbox panel (page 286)

Step 2: Configure Carousel Properties

Now you will tell the carousel which graphics to display and how to behave.

  1. In Properties, scroll down to locate the Widget Properties section.
  2. In GraphicNames, enter: Line1_MixingProcess,ClockAnalogWall
Tip: The graphic names you enter here must exactly match the names of existing Industrial Graphics in your Galaxy. The carousel will display these graphics in the order you specify, separated by commas.

The remaining properties — Autoplay (True), BackgroundColor (#FFFFFF), Interval (5000 ms), Keyboard (True), Loop (True), and Pause (True) — are left at their default values for this lab.

The Properties panel for the Carousel widget, showing standard graphic properties and the Widget Properties section at the bottom (page 287)
The GraphicNames property configured with the two process graphics to cycle through at runtime (page 287)
  1. Save and close, then check in the graphic.

Step 3: Create a New Window

Next, you will create a window in WindowMaker to host the carousel.

  1. In WindowMaker, click the New (+) button.
  2. Ensure Frame window is selected.
  3. Configure the window properties as follows and click Create:

Step 4: Embed the Dashboard Symbol

Now you will place the Dashboard symbol inside the Carousel window.

  1. In the Carousel window, embed the Dashboard symbol from the Industrial Graphics library.
  2. In Properties, uncheck Maintain aspect ratio. This allows the carousel to stretch and fill the entire window.

Step 5: Test at Runtime

Finally, you will verify that the carousel works correctly in WindowViewer.

  1. Click Runtime. After a few moments, the carousel begins automatically cycling through the configured graphics.
  2. When the mixer graphic appears, hover your mouse over the carousel to pause automatic scrolling.
  3. Move your cursor to the left or right edge of the carousel until a pointing-hand cursor appears, then click to advance manually.
  4. Click Development! to return to WindowMaker.
The Carousel in runtime, automatically cycling between the mixer process overview and the analog clock graphic (page 290)

Knowledge Check

1. What file format is used to import HTML5 widgets into InTouch, and where do you find the import command?

Click to reveal answer

Widgets are imported using the Custom Widget Package (.cwp) format. The import command is found at Galaxy → Import → Visualization → Web Widgets in the IDE.

2. What happens when the Carousel widget's Pause property is set to True and an operator hovers their mouse over the widget?

Click to reveal answer

The carousel automatically pauses its cycling when it detects a mouse hover or touch event, allowing the operator to examine the current graphic. Cycling resumes when the mouse moves away.

3. Why must you uncheck Maintain aspect ratio when embedding the Dashboard symbol into the Carousel window?

Click to reveal answer

Unchecking Maintain aspect ratio allows the embedded Dashboard graphic to stretch and fill the entire Carousel window, rather than being constrained to its original proportions. This ensures the carousel displays full-screen.

4. What determines the order in which graphics appear in the Carousel, and how do you configure it?

Click to reveal answer

The order is determined by the comma-separated list you enter in the GraphicNames widget property. The first graphic in the list appears first, and the carousel cycles through them in sequence.
