23.1 What Are HTML5 Widgets?

Modern industrial visualization demands more than static gauges and trend lines. Operators increasingly expect rich, interactive displays that rival consumer-grade web applications — dashboards with live data grids, embedded maps, rotating graphic displays, and even QR code scanners. AVEVA InTouch for System Platform meets this demand through its support for HTML5 Widgets: self-contained code blocks built with standard web technologies (HTML5, CSS, and JavaScript) that extend InTouch's native capabilities without modifying its core engine.

Think of a widget as a miniature web application that lives inside your HMI screen. InTouch acts as a container for these widgets at both design time and runtime, rendering them within the graphic canvas just like any other symbol. This architecture means that virtually any web-based UI component — whether it's a third-party charting library, a custom data visualization, or even a social media feed for a KPI display board — can be embedded directly into your industrial graphics.

Key Concept: Widgets as Web Containers
InTouch serves as a host container for HTML5 Widgets. At design time, you drag widgets from the Toolbox onto your canvas and configure their properties. At runtime, WindowViewer renders the widget's HTML5 content inside the graphic, giving it access to InTouch's tag data while leveraging the full power of web-based UI frameworks.

23.2 Built-In Widget Library

By default, InTouch ships with several pre-built widgets available under the Widgets folder in the Graphics tab. Each widget serves a distinct purpose in industrial visualization:

WidgetPurpose
CarouselCycles through multiple graphics automatically or via user control — ideal for rotating dashboards on plant-floor monitors
DatagridDisplays tabular data in a sortable, scrollable grid — perfect for listing alarms, production logs, or equipment status
MapAppShows geographic or plant map visualizations — useful for site-wide overviews or GPS-tracked assets
QRCode_ScannerScans QR codes for rapid asset identification and lookup in the field
TeamworkProvides collaborative or multi-user display features for shift handovers and team coordination
Web_BrowserEmbeds external web content directly in your HMI — dashboards, documentation, or third-party applications

Beyond these built-in options, you can import your own custom widgets through the IDE by navigating to Galaxy > Import > Visualization > Web Widgets. Custom widgets are packaged as .cwp files (Custom Widget Package), which bundle the HTML5, CSS, and JavaScript source files into a single importable archive. This packaging format ensures that all dependencies travel together and deploy cleanly to runtime environments.

Tip: If you need a widget that doesn't exist in the built-in library, check whether someone in the AVEVA community has published a custom widget package. You can also build your own using standard HTML5/CSS/JS and package it as a .cwp file.

23.3 The Carousel Widget

The Carousel widget is one of the most practical built-in widgets for plant-floor visualization. It allows you to cycle through multiple Industrial Graphics — like a rotating slideshow — without any operator input. It also supports manual navigation with previous/next buttons, making it ideal for displaying dashboards, alerts, or alarm summaries on large monitors throughout the facility.

The Carousel's behavior is controlled through its Widget Properties panel, which appears in the Properties inspector when the Carousel is selected. Understanding these properties is essential for configuring the widget correctly.

Key Concept: Essential Carousel Properties
The most important properties that control the Carousel's behavior are:
  • GraphicNames — A comma-separated list of Industrial Graphics to display (e.g., Line1_MixingProcess,ClockAnalogWall)
  • Autoplay — Whether the carousel starts cycling automatically when the window opens (True/False)
  • Interval — The delay in milliseconds between automatic transitions (default: 5000, meaning 5 seconds per graphic)
  • Loop — Whether the carousel cycles continuously or stops after completing one pass through all graphics
  • Pause — Whether hovering the mouse over the carousel pauses the automatic scrolling

Additional properties include Keyboard (whether the carousel responds to keyboard arrow input), BackgroundColor, and the standard graphics properties for sizing and positioning on the canvas.

Tip: Set the Pause property to True for plant-floor displays. This lets operators hover over a graphic of particular interest to study it at their own pace, then move their mouse away to resume automatic cycling — a simple but highly effective usability improvement.
Pitfall: The GraphicNames property must contain exact, case-sensitive graphic names as they appear in the Galaxy. If you misspell a name or use the wrong case, the Carousel will silently skip that graphic with no error message at runtime. Always copy-paste graphic names directly from the Galaxy Browser to avoid typos.

23.4 Lab 11 – Building a Carousel Dashboard

Lab Objectives:
  • Create a Dashboard symbol and embed the Carousel widget
  • Configure the Carousel to display multiple process graphics
  • Test the Carousel at runtime in WindowViewer

In this lab, you will build a rotating dashboard that automatically cycles between different process graphics — a real-world use case for monitoring multiple areas of a plant from a single screen.

Step 1: Create the Dashboard Symbol

In the IDE, navigate to the Graphics tab and create a new symbol named Dashboard in the Training toolset. Open it for editing. From the Toolbox tab, drag the Widgets\Carousel component onto the canvas. You will see the Carousel appear as a bordered region on the design canvas, ready for configuration.

Carousel widget placed on the canvas, with the Widgets toolbox panel visible on the right showing all available widgets.

Step 2: Configure Widget Properties

With the Carousel selected, scroll down in the Properties panel to find the Widget Properties section. This is where you define which graphics the Carousel will display and how it will behave. Set the GraphicNames property to:

Line1_MixingProcess,ClockAnalogWall

These two graphics will now cycle automatically in the Carousel at runtime. Leave the other properties at their defaults: Autoplay set to True, Interval at 5000 milliseconds, Loop enabled, and Pause enabled. When you are satisfied with the configuration, save, close, and check in the graphic.

Widget Properties panel with GraphicNames set to the two process graphics that will cycle in the Carousel.

Step 3: Create the Carousel Window

Create a new Frame window named Carousel with the comment "Display System Graphics" and the Title bar unchecked. This gives you a clean, borderless display area — exactly what you want for a full-screen dashboard on a plant-floor monitor.

Carousel window properties — note the Title bar is disabled for a clean full-screen appearance.

Now embed the Dashboard symbol in the Carousel window. In the Properties panel, uncheck Maintain aspect ratio so the widget stretches to fill the entire window area. This is important — without this setting, the Carousel would maintain its original proportions and leave empty space around the edges.

Dashboard symbol embedded in the Carousel window with Maintain aspect ratio unchecked.

Step 4: Test in Runtime

Click Runtime to launch WindowViewer. You should see the Carousel automatically begin cycling between your two graphics — first the Mixing Process display, then the Analog Clock, and back again. Hover your mouse over the Carousel to pause the scrolling, giving yourself time to study the current graphic. Then move your mouse to the left or right edge of the display — the cursor changes to a pointing hand, indicating you can click to advance manually in either direction. When you are done testing, click Development! to return to WindowMaker.

Runtime view of the Carousel cycling between the Mixer process graphic and the analog clock graphic.

Quiz — Test Your Understanding

Q1: What file format is used to import custom HTML5 Widgets into InTouch?

Custom Widget Package (.cwp) — this format bundles HTML5, CSS, and JavaScript files into a single importable package that deploys cleanly to runtime.

Q2: What happens when the Carousel's Autoplay property is set to False?

The carousel will not start cycling automatically on load. The user must manually click the next/previous buttons or use keyboard input to advance through the graphics.

Q3: How do you specify which graphics the Carousel displays at runtime?

Enter a comma-separated list of exact graphic names in the GraphicNames property (e.g., "Line1_MixingProcess,ClockAnalogWall"). The names must match the Galaxy graphic names exactly, including case sensitivity.

Q4: Why would you uncheck "Maintain aspect ratio" when embedding a widget symbol?

Unchecking Maintain aspect ratio allows the symbol to stretch and fill the entire available space. This is essential when you want the Carousel or widget to occupy the full window area rather than keeping its original proportions and leaving empty margins.

Q5: What is the effect of setting the Pause property to True on a plant-floor display?

Setting Pause to True allows operators to hover their mouse over the Carousel to pause the automatic cycling, giving them time to study a particular graphic. Moving the mouse away resumes the automatic rotation.
