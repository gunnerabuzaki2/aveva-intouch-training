Lesson 21: Lab 10 – Creating a Navigation Symbol

Source: AVEVA InTouch for System Platform 2023 Training Manual, Module 3 — Symbols (pages 255–276)

In this lab you will build a navigation system that lets operators switch between different mixer views at runtime. You will use the Select Alternate Instance feature, the ShowGraphic() script function, and alignment tools to create a polished, reusable navigation bar.

21.1 Creating the Line 1 Mixer Display (Embedded Approach)

Reference: Module 3 (pages 255–258)

The first approach to displaying multiple mixers is to embed the graphics directly on the canvas and use the Select Alternate Instance feature to swap each copy to a different mixer instance. This gives you a visual layout where all four mixers are visible simultaneously.

Lab Steps:
  1. Create a new symbol named Line1_MixingProcess and open it for editing.
  2. Right-click the canvas and select Embed Industrial Graphic. From Instances, embed Mixer100\MixingProcess_Detailed.
  3. Place it in the top-left corner. Zoom to 75% for a better view.
  4. Duplicate the graphic and place the copy to the right. Right-click the duplicate and select Industrial Graphic > Select Alternate Instance. Double-click Mixer200.
  5. Select all graphics, duplicate them, and place the copies below. Change the bottom-left to Mixer300 and the bottom-right to Mixer400.
  6. Use the Alignment toolbar to align the top pair (Align Top), bottom pair (Align Bottom), left pair (Align Left), and right pair (Align Right).
  7. Save, close, and check in.

21.2 Creating the Line 2 Mixer Display (Scripted Approach)

Reference: Module 3 (pages 259–263)

The second approach uses the ShowGraphic() script function to dynamically display mixer graphics as borderless popup windows. This is more flexible than embedding because the graphics are loaded at runtime and positioned programmatically.

Lab Steps:
  1. Create a new symbol named Line2_MixingProcess. Draw a rectangle on the canvas and name it MixerDisplay with Width = 1490, Height = 670. This rectangle defines the area where the popup graphics will be positioned.
  2. Add a custom property named ShowTrigger (Data Type: String). This will be the trigger for the named script.
  3. Open the Scripts editor. In the Predefined Scripts pane, select On Hide and enter the cleanup script:
' Hide all graphics before displaying new graphics.
dim i as integer;
for i = 5 to 8
  ' Hide all displayed graphics.
  ' Identities: View5, View6, View7, View8.
  HideGraphic( "View" + StringFromIntg(i, 1) );
next;

This On Hide script ensures that when the Line2_MixingProcess symbol is hidden (e.g., when the operator navigates away), all four popup windows are closed.

  1. Add a Named script called ShowMixer. Set Expression = ShowTrigger, Trigger = DataChange.
  2. Enter the ShowMixer script body (available in C:\Training\Lab 10 - Creating a Navigation Symbol.txt). The script loops from 5 to 8, creating a GraphicInfo object for each mixer, configuring its position within the MixerDisplay rectangle, and calling ShowGraphic().

Key Concept: The ShowGraphic() function positions each popup relative to the MixerDisplay rectangle. The script calculates X and Y coordinates based on the rectangle's position and dimensions, placing mixers in a 2×2 grid with no title bars or close buttons — creating the illusion that they are part of the main window.

21.3 Creating the Reusable Navigation Button

Reference: Module 3 (pages 264–269)

The navigation button is a self-contained symbol with custom properties that control what happens when it is clicked. Each button instance can target a different graphic or window, making the button truly reusable.

Lab Steps:
  1. Create a symbol named NavigationButton. Add five custom properties:
PropertyData TypePurpose
GetButtonTextStringTracks which button is currently active (shared with SystemNavigation)
GraphicNameStringThe target graphic or window to display
WindowModalBooleanWhether the popup is modal
WindowNameStringInTouch window name (if targeting a legacy window)
WindowResizableBooleanWhether the popup can be resized
  1. Add a Button to the canvas named Mixer_Btn with Text = Label, ElementStyle = Intensity2, Width = 100, Height = 50.
  2. Add an Action Scripts animation (On Left/Key/Touch Up) with this script logic:
dim graphicInfo as aaGraphic.GraphicInfo;
GetButtonText = Text;
' Identity is targeting an InTouch window.
if (WindowName <> "") then
  graphicInfo.Identity = "InTouch:" + WindowName;
else;
  graphicInfo.Identity = GraphicName;
endif;
graphicInfo.GraphicName = GraphicName;
' Check if window should be modal.
if (WindowModal) then
  graphicInfo.WindowType = aaGraphic.WindowType.Modal;
endif;
graphicInfo.Resizable = WindowResizable;
ShowGraphic( graphicInfo );
  1. Add a Disable animation: Boolean, Expression = GetButtonText == Text, Disabled When = True. This disables the button when it is the active one.
  2. Add an Element Style animation: Boolean, Expression = GetButtonText == Text, True → Intensity4, False → unchecked (default).
  3. Save, close, and check in.

21.4 Assembling the SystemNavigation Symbol

Reference: Module 3 (pages 270–271)

Now you will embed three NavigationButton instances into a single container symbol and configure each one to target a different view.

Lab Steps:
  1. Create a symbol named SystemNavigation. Add a custom property LastButton (String).
  2. Embed the NavigationButton graphic. Change its button text to Mixers. Configure its properties:
    • GetButtonText = LastButton
    • GraphicName = Mixer100.SwitchedMixingProcess
    • WindowName = ContentDisplay
  3. Duplicate the button. Change text to Line 1. Set GraphicName = Line1_MixingProcess.
  4. Duplicate again. Change text to Line 2. Set GraphicName = Line2_MixingProcess.
  5. Save, close, and check in.

21.5 Testing in Runtime

Reference: Module 3 (pages 271–273)

With the navigation symbol assembled, embed it in the Navigation window and switch to WindowViewer to test.

Lab Steps:
  1. In the Navigation window, embed the SystemNavigation symbol.
  2. Click Runtime to switch to WindowViewer.
  3. Click Line 1 — the button highlights and four mixer process displays appear (Mixer100–Mixer400).
  4. Click Line 2 — the Line 1 popups close (via the On Hide script) and four new popups appear (Mixer500–Mixer800).
  5. Click Mixers — the Line 2 popups close and the Mixer100 SwitchedMixingProcess window opens in ContentDisplay.
  6. Click Development! to return to WindowMaker.

Important: The mixers displayed by Line 2 are borderless popup windows with no title bar or close button. They float on top of WindowViewer. When you navigate away (e.g., click Line 1 or Mixers), the On Hide script on Line2_MixingProcess automatically closes these popups. This is why the cleanup script is essential — without it, orphaned popups would remain on screen.

Knowledge Check

1. What is the purpose of the ShowTrigger custom property on the Line2_MixingProcess symbol?

Click to reveal answer

ShowTrigger serves as the expression for the ShowMixer named script. When its value changes (DataChange trigger), the script fires and displays the four mixer popup windows using ShowGraphic().

2. Why does the On Hide script call HideGraphic() in a loop instead of hiding each popup individually?

Click to reveal answer

Using a loop with StringFromIntg() to build the identity names (View5 through View8) is more efficient and less error-prone than writing four separate HideGraphic calls. It also makes it easy to adjust the number of popups by changing the loop bounds.

3. What role does the GetButtonText custom property play in the navigation button?

Click to reveal answer

GetButtonText is a shared String property (mapped to LastButton in SystemNavigation) that tracks which button was last clicked. The Disable and Element Style animations use the expression GetButtonText == Text to determine if this button is the active one — disabling it and highlighting it when it is.

4. Why is the runtime resolution requirement of 1920 × 1080 important for this lab?

Click to reveal answer

The ShowGraphic() popup windows are positioned using absolute coordinates within the MixerDisplay area. Microsoft's window management prevents popups from appearing partially off-screen, so at non-1920×1080 resolutions, the system may reposition popups to fit, causing them to overlap or appear in unexpected locations.

5. How does the Select Alternate Instance feature differ from embedding a completely new graphic?

Click to reveal answer

Select Alternate Instance re-points an existing embedded graphic to a different Galaxy instance of the same symbol type. This preserves all the symbol's configuration and animations while changing only the underlying tag reference. Embedding a new graphic would require reconfiguring everything from scratch.
